## 1. 本轮结论

这轮已经完成的是“可配置、可测试、可继续扩展”的第一版系统：

- 已经把项目运行时画质分级接入到存档、CVar、Scalability、DeviceProfile 和玩家设置入口。
- 已经建立 PC / Steam Deck 15W / Switch 2 Candidate / Steam Deck 5W / Fallback Low 的目标分级映射。
- 已经实现材质批处理的离线工具链样板：扫描、审计、Texture2DArray 计划、PropertyTexture 计划、Mapping DataAsset、ProxyMesh、Batch Material 生成阶段。
- 已经建立 UE5.8 性能审计和 profiling 计划 commandlet。
- 已经生成一个小样本资产目录，用于验证 Texture2DArray、PropertyTexture、ProxyMesh、Batch Material 资产输出链路。
- 已经移除 `Docs/GeneratedReports/**` 的 Git 跟踪，并把它加入 `.gitignore`。后续报告仍可本地生成，但不会上传到 GitHub main。

还没有完成的是“全项目自动替换生产管线”：

- `MaterialBatchBuild -Apply` 全量替换仍故意禁用，避免未审查代理资产直接替换关卡。
- 当前 Batch Material 仍是桥接版本，最终运行时批处理母材质还需要完整读取 `UV7.x`、`PropertyTexture` 行和 Texture2DArray slice。
- 真实平台 GPU 数据还没有完成矩阵实测，尤其是 Steam Deck 15W / 5W 和 Switch 2 真机。

### 3.4 运行时实际应用了哪些 CVar

`UYogPerformanceSettingsLibrary::ApplyGraphicsSettings` 会应用：

```text

ReflectionQuality
PostProcessingQuality
TextureQuality
VisualEffectQuality
FoliageQuality
ShadingQuality

r.ReflectionMethod
r.Nanite
r.Shadow.Virtual.Enable
```

注意：

- 当前策略允许 Lumen Lite，但默认不启用 Nanite 和 VSM。

- `r.Yog.DynamicLightQuality`、`r.Yog.MaterialLightQuality`、`r.Yog.MaterialLight.MaxLightInfoCount` 是项目自定义预算 CVar，后续灯光系统和赛璐璐材质灯插件应读取这些值做降级。

### 6.3 Commandlet 当前能力

`MaterialBatchAudit`：

- 扫描 StaticMesh 资产或地图中的 StaticMeshComponent。
- 统计材质槽、LOD、标签、候选状态和拒绝原因。
- 只审计，不生成资产。

`MaterialBatchMaterialAudit`：

- 审计目标材质，如 `M_Env_Building`。
- 记录 Texture 参数、Scalar 参数、Vector 参数、Static Switch、Blend Mode、Shading Model。
- 只审计，不改材质。

`MaterialBatchBuild`：

- 支持从资产目录或 Map 扫描候选。
- 支持 `-RequireTag=<Prefix>` 只扫描带指定 Actor Tag 前缀的对象。
- 支持 dry-run 生成 plan / manifest。
- 支持分阶段 partial apply：
  - `-ApplyMappingOnly`
  - `-ApplyTextureArraysOnly`
  - `-ApplyPropertyTextureOnly`
  - `-ApplyProxyMeshOnly`
  - `-ApplyBatchMaterialOnly`
- `-Apply` 全量模式仍禁用，避免自动替换地图。

### 6.6 美术侧标记规则

Actor Tag 推荐：

| Tag | 用途 |
| --- | --- |
| `EnvBatch.Source.<GroupName>` | 原始静态环境物件，允许作为批处理输入 |
| `EnvBatch.Proxy.<GroupName>.Medium` | Medium 及以下可用代理 |
| `EnvBatch.Proxy.<GroupName>.Low` | Low 档代理 |
| `EnvBatch.Baked.Ground.Low` | Low 档地面烘焙替代 |
| `EnvBatch.Exclude` | 明确排除批处理 |

原则：

- 不要把可交互、可破坏、动态、独立碰撞、独立脚本事件对象打进批处理。
- 不要整关卡合并成一个大 Mesh，应按房间、走廊段、街区、可见性 cell 或 HLOD cell 分块。
- 重复实例优先考虑 ISM/HISM。
- 近景英雄资产保留独立材质和 LOD。
- 中远景静态建筑模块优先尝试 Geometry Merge + Texture2DArray。

## 7. Lumen Lite 与性能分级策略

### 7.1 当前默认判断

- PC：允许 Lumen High / Ultra。
- Steam Deck 15W：可以测试 Lumen Lite，但必须以实测 GPU ms 为准。
- Switch 2 Candidate：暂时复用掌机 15W 候选策略，不能当成真机结论。
- Steam Deck 5W：默认 Lumen Off。
- Nanite 和 VSM 当前不作为掌机方案依赖。

### 7.2 需要测试的矩阵

每个代表场景至少测：

1. Baseline：当前项目设置，Lumen Off，无 batch proxy。
2. Lumen Lite only。
3. Batch only。
4. Batch + Lumen Lite。
5. Low：Lumen Off + Batch Proxy / HLOD / Baked Replacement。
6. PC Ultra：最高画质路径。

记录：

- FPS
- Game ms
- GPU ms
- RHI ms
- Mesh Draw Calls
- Total Draw Calls
- Dynamic light count
- Shadow cost
- Lumen pass cost
- Material light sample cost
- Texture memory / streaming pool
- 视觉差异

### 16.5 Lumen Off 档位使用什么

Low / Steam Deck 5W 不建议依赖 Lumen。推荐替代方案是：

- 静态环境使用预计算/烘焙光照、低频补光、材质灯降采样。
- 装饰性动态灯转为静态烘焙贡献或材质贴图表现，只保留玩法关键动态灯。
- 地面、远景和高重复度建筑件使用 Batch Proxy / HLOD / Baked Replacement。
- 反射使用 SSR Low 或关闭，避免 Lumen Reflection。
- `r.Yog.MaterialLightQuality` 和 `r.Yog.MaterialLight.MaxLightInfoCount` 控制赛璐璐材质灯采样预算。

切换是否需要重启：

- 纯 CVar / Scalability 项，例如分辨率比例、阴影质量、材质灯预算、Lumen 开关，技术上可以运行时切换，但可能有 shader/streaming hitch。
- Source / Proxy / Baked 这种场景表现层切换，不建议强制重启游戏，但建议在“离开当前房间 / 重新加载当前关卡 / 下一房间”时生效。
- 玩家 UI 可以即时保存设置，并提示“部分场景替代资源将在下次房间加载时生效”。
