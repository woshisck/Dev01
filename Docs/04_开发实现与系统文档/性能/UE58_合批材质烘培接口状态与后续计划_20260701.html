UE5.8 合批与材质烘培接口状态及后续计划

日期：2026-07-01　项目：DevKit / UE5.8　目标平台：PC 端优先

当前项目已经具备静态环境合批和材质烘培/VT Atlas 方向的基础接口，但还没有完成正式生产替换闭环。

EpicHighMidLow

1. 当前结论

已有：EnvBatch.* Actor Tag、MaterialBatchBuild dry-run/report/manifest、Mapping DataAsset、PropertyTexture、VT Atlas、ProxyMesh、BatchMaterial 分阶段接口、模型资产合规检查、材质合规检查，以及 UE 原生 r.MaterialQualityLevel 接入。

未完成：正式打包前自动替换关卡资产、墙面/建筑复杂材质所见即所得烘培、真实关卡 VT Atlas 视觉验收、VT pool / streaming pool profiling，以及最终合批父材质的完整采样验收。

2. 三块材质处理方向

地面

使用一套 RVT 地面主材，不通过性能分级做分支。多层混合、贴花混合、Mesh Paint、顶点色等静态效果在 RVT 写入前完成；动态效果放在 RVT 后叠加。

墙面、建筑

保留 2 层混合、顶点色或 Mesh Paint、贴花、可能的 RVT 接收。该部分暂时搁置，后续单独做“编辑器复杂材质到出包 baked 材质”的转换链。

静态模型、物件、非交互道具

继续走合批主线：美术制作普通单体资产和 unique 贴图，场景中通过 EnvBatch.Source.<Group> 标记，正式打包链生成 VT/UDIM-style Atlas、Mapping、PropertyTexture、ProxyMesh、BatchMaterial 和替换计划。

3. 性能分级调用方式

项目四档面向玩家和美术仍为 Epic / High / Mid / Low。运行时材质分级主线使用 UE 原生 r.MaterialQualityLevel 和材质 Quality Switch。

项目档位UE Material Quality说明
EpicEpic = 3PC 高规格目标
HighHigh = 1UE 原生枚举中 High 为 1
MidMedium = 2项目 Mid 映射到 UE Medium
LowLow = 0最低运行时材质质量

项目自定义 CVar 如 r.Yog.VTAtlasQuality、r.Yog.BatchProxyPreference、r.Yog.MaterialLightQuality 仅作为增强预算和生产链参数，不替代 UE 原生 Scalability。

4. 已完成工作进展

  • 模型资产合规检查编辑器第一版完成，支持目录树、分类筛选、状态设置、模型预览、打开模型和定位 Content Browser。
  • 环境合批标记只负责 Actor Tag，不混入模型 LOD 配置或贴图规则。
  • 材质合规检查替代旧贴图规则入口，支持贴图命名、sRGB、尺寸、VT 建议和材质接口检查。
  • MaterialBatchBuild 保留 dry-run 和 partial apply 接口，全量替换仍不作为日常入口。
  • 项目四档性能设置已接入 UE 原生 r.MaterialQualityLevel。

5. 后续计划

阶段任务验收
P0保持当前模型合规、材质合规、EnvBatch 标记工具可用编译通过，工具入口可见
P1选择真实 Streaming Level 运行 MaterialBatchBuild dry-run输出 report / manifest，拒绝原因可复查
P2验证真实 unique 贴图到 VT/UDIM-style Atlas 的生成链确认尺寸、格式、mip、VT pool 和 streaming pool 风险
P3验证 ProxyMesh、PropertyTexture、batch index 和材质采样一致性视觉对比、draw call、GPU time、碰撞和遮挡表现通过
P4单独推进墙面/建筑材质烘培专项编辑器可编辑效果可转换为 baked 结果
P5纳入打包前 gate阻断项失败，警告项报告，PC 端性能基线可追踪

6. 注意事项

  • 地面 RVT 不需要强行做性能分级分支。
  • 墙面/建筑材质烘培暂时不要混入当前模型合规工具。
  • 交互物、角色、武器和运行时动态物件默认不进入静态环境合批。
  • 美术不需要手工维护 UDIM、SVT tile 或 Atlas。
  • Quality Switch 是 UE 原生材质分级接口，但不是所有材质都必须使用。
  • 当前合批/烘培接口已经存在，正式生产替换仍需要真实关卡验证后才能开放。
