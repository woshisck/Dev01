如何配置

  • 改平台默认画质：编辑 Config/DefaultDeviceProfiles.ini。
  • 改 Scalability 质量组：编辑 Config/DefaultScalability.ini。
  • 改玩家预设数值：编辑 UYogPerformanceSettingsLibrary::MakeGraphicsSettingsForProfile 和 MakeGraphicsSettingsForTargetTier。
  • 改 UI 控件：编辑 WBP_GraphicsSettingsWidget 和 YogGraphicsSettingsWidgetBase。
  • 改材质灯预算：接入 r.Yog.MaterialLightQuality 和 r.Yog.MaterialLight.MaxLightInfoCount。
  • 改批处理输出目录：使用 -OutputRoot=/Game/Generated/MaterialBatch。

风险

  • Lumen Lite 掌机成本必须实测，不能只看功能说明。
  • VT/SVT 解决 streaming / 显存压力，不等于 draw call 合批。
  • Draw Call 优化依赖 Geometry Merge、HLOD、ISM/HISM 或合批材质。
  • Switch 2 当前只是候选档，必须等待真机数据。
  • 生成报告不再上传，正式文档不要放进 Docs/GeneratedReports。

编辑器 Play Menu 与主菜单 Start

编辑器工具栏中的自定义 DevKitPlayFromMainMenu / Play Menu 按钮已撤销。该按钮通过 GlobalMapOverride 强行从入口地图启动 PIE，会触发 YogWorldSubsystem BeginPlay 时序 ensure。现在编辑器恢复使用 UE 默认 Play 行为。

主菜单 Start 保持直接调用 StartNormalRunFromFrontend。测试画质时先通过项目默认入口地图或正常 Play/Standalone 路径进入主菜单，再进入 Options 选择档位或自定义参数，点 Back 回主菜单，再点 Start 进入 MainGameMap。Continue 仍保留存档选择/继续流程。

当前已取消默认锁帧：Medium / Low / Steam Deck / Switch Candidate / Fallback Low 预设均使用 FrameRateLimit=0 和 t.MaxFPS=0，方便观察真实性能上限。

M_Env_Building 审计结论

目标材质：/Game/Art/Material/EnvMaterial/Main/M_Env_Building.M_Env_Building

  • 材质为 Masked / DefaultLit / No WPO / One-sided，适合静态环境批处理候选。
  • 已经包含 T_Array_A、T_Array_M、T_Array_N 三个 Texture2DArray 参数。
  • 仍有 Unique 贴图参数：T Unique Color Map、T Unique MRA Map、T Unique Normal Map、T Unique Mask Map。
  • Tex_LightInfo 是材质灯信息，当前不应并入普通 BaseColor / Normal / ORM Texture2DArray。
  • Static Switch 组合仍会拆批，后续批处理母材质需要把可运行时化的开关转成参数。

下一步材质专项应做 M_Env_Building_Batch：读取 TexCoord7.x 作为 material/slice index，读取 _PropTexture 行参数，并采样对应 Texture2DArray slice。

EnvBatch Tagger 编辑器窗口

入口：Tools -> DevKit Tools -> Performance Tools -> EnvBatch Tagger

  1. 在 Level Viewport 或 World Outliner 选择 Actor。
  2. 打开 EnvBatch Tagger。
  3. 输入 Group，例如 Corridor_01b 或 Building_Stone。
  4. 点击 Source.<Group> 写入 EnvBatch.Source.<Group>。
  5. 点击 Proxy.<Group>.Medium / Proxy.<Group>.Low 标记代理件。
  6. 点击 Baked.Ground.Low 标记低档烘焙替代件。
  7. 点击 Exclude 明确排除对象，或用 Remove EnvBatch Tags 清理选中对象。

工具只写 Actor Tags，不自动保存地图；打完标签后按 UE 正常流程保存对应关卡或子关卡。

Lumen Off 档位使用什么

  • Low / Steam Deck 5W 默认不依赖 Lumen，采用静态/烘焙光照、低频补光和材质灯降采样。
  • 装饰性动态灯转为静态烘焙贡献或材质贴图表现，只保留玩法关键动态灯。
  • 地面、远景和重复建筑件优先使用 Batch Proxy / HLOD / Baked Replacement。
  • 反射使用 SSR Low 或关闭，避免 Lumen Reflection。
  • 场景表现层切换建议在离开房间、重新加载当前关卡或下一房间时生效；纯 CVar 项可以即时保存并尝试运行时应用。

下一步建议

  1. 选一个真实、有 mesh 数据的代表关卡或房间 cluster。
  2. 给静态环境 Actor 打 EnvBatch.Source.<GroupName>。
  3. 跑 MaterialBatchBuild dry-run，确认候选和拒绝原因。
  4. 跑 partial apply，生成 Mapping、T2DA、PropertyTexture、ProxyMesh、BatchMaterial。
  5. 在测试关卡中切换 Source / Proxy。
  6. 做 Baseline、Lumen Lite、Batch、Batch + Lumen Lite 的 GPU profile。
  7. 根据数据决定 Medium 是否默认开 Lumen Lite。
