UE5.8 性能分级本次修改与进展报告

日期：2026-07-01 ｜ 目标分支：main ｜ 项目：DevKit UE5.8

本次结论

本次已经把性能分级基础从方案文档推进到可在编辑器里使用的生产入口、运行时四档设置和自动化检查阶段。

  • 只保留 Epic、High、Mid、Low 四个目标挡位。
  • 场景组织按传统 Streaming Level 处理，不启用 World Partition，不引入 Data Layer 作为前提。
  • Lumen 保持项目默认开启；VSM 不作为默认开启项。
  • 模型资产分级不再生成单独 DA_ModelPerformance_* 配置资产，模型自己的 LOD、材质槽、碰撞和导入设置直接在 StaticMesh 资产内维护。
  • 正式合批、VT Atlas、ProxyMesh 和替换计划放到打包/TA 自动化链执行。
  • 材质侧本次提供性能分级接口和管线契约，不直接制作最终复杂材质网络。

已完成修改

运行时性能分级

  • 整理四档画质枚举、保存数据、Blueprint 可调用接口和图形设置 UI。
  • 更新项目 Scalability、DeviceProfile 和 Engine 默认配置。
  • 补充性能设置自动化测试，覆盖档位映射、保存和运行时应用路径。

编辑器工具入口

  • 恢复“从主菜单运行”的编辑器白色播放箭头，读取 YogGameInstanceBase.FrontendMap。
  • 新增顶部工具栏入口：性能工具、模型合规、贴图规则。
  • 模型合规和贴图规则移动到 Tools -> DevKit 工具 -> 美术资产工具。

模型资产合规检查

  • 新增独立模型合规窗口，扫描 /Game/Art 下 StaticMesh。
  • 支持目录树、分类筛选、模型预览、富文本检查信息、手动状态设置和 Content Browser 定位。
  • 默认口径为 Epic/High=LOD0、Mid/Low=LOD1，静态环境模型至少需要 LOD0 和 LOD1。
  • 删除独立模型分级配置窗口，右侧 打开模型设置 直接打开 StaticMesh 资产编辑器。

环境合批标记

  • 只负责写 EnvBatch.* Actor Tag，不再混入模型 LOD 配置或贴图规则编辑。
  • 支持 Source、Proxy、Baked、Exclude 语义，供后续打包链扫描。
  • 合批组可以是不连续 Actor，蓝图和 Level Instance 边界对象可以按 Actor Tag 进入后续扫描。

贴图命名规则与材质接口

  • 新增 UYogMaterialTextureNamingConvention 和默认规则资产。
  • 新增贴图命名规则窗口，支持写入默认规则、打开规则资产和检查选中 Texture2D。
  • 材质主线保持 UDIM-style SVT / VT Atlas，不要求美术手工维护 atlas/UDIM。
  • 地面和墙面 Source 阶段允许复杂静态混合，低档结果通过 bake 后的 VT Atlas 或 baked texture 进入运行时。

注意事项

  • 模型合规工具的“排除”是资产分类和状态，不代表删除资产。
  • 合批 tag 只是后续自动化扫描依据，不直接执行合并和替换。
  • 正式合批必须在打包/TA 链执行，并先审核 dry-run 报告。
  • VT 和非 VT 混用需要继续关注普通 streaming pool 与 VT physical pool 的重复驻留风险。
  • 本次包含生成 .uasset 资源，后续如果对视觉内容有要求，需要在 Unreal Editor 中打开检查。
