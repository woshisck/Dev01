# UE5.8 性能分级美术工具使用与测试说明

更新时间：2026-06-30

本文只描述当前已经提供或需要美术使用的工具入口。全项目资产审计、场景灯光预算、最终性能验收自动化暂不在本阶段执行，放到项目完成前出包阶段统一做。

## 当前边界

- 正式性能挡位只有 `Epic / High / Mid / Low`。
- 当前场景组织使用传统 Streaming Level，不使用 World Partition，不使用 DataLayer 作为美术工作流或验收前提。
- Nanite 和 VSM 当前保持关闭；VSM 不作为本轮性能分级目标。
- 角色、交互物、VFX、运行时动态 gameplay actor 暂不进入静态环境合批，也不强制开启 VT。
- 环境静态资产优先做到一个材质槽；特殊资产通过配置处理，不要求美术维护 AuthorProxy。
- 材质侧本阶段提供性能分级接口和模板，不要求现在完成最终地面/墙面复杂母材。
- 所有合批、代理、VT Atlas、替换和写入生成资产命令统一后移到正式打包链执行；当前美术阶段只做资产合规检查和标记准备。

## 美术工具入口

### 性能工具快捷入口

入口：

```text
Level Editor 顶部工具栏 -> 性能工具
Tools -> DevKit 工具 -> 性能工具 -> 性能工具快捷入口
```

用途：

- 集中打开当前阶段常用的性能分级工具。
- 快速打开“模型资产合规检查”，检查 `/Game/Art` 下 StaticMesh 是否满足性能分级制作要求。
- 快速打开“环境合批标记”窗口。
- 快速执行“从主菜单运行”。

### 从主菜单运行

入口：

```text
Level Editor 顶部 Play 工具栏 -> 白色播放箭头
Tools -> DevKit 工具 -> 性能工具 -> 从主菜单运行
性能工具快捷入口 -> 从主菜单运行
```

说明：

- 该入口读取 `YogGameInstanceBase.FrontendMap`，当前默认是 `/Game/Maps/L_EntryMenu`。
- 启动流程是：先加载入口地图，再使用编辑器正常 `RequestPlaySession` 进入 PIE。
- 该入口不再使用旧版 `GlobalMapOverride`，避免旧实现触发的 BeginPlay 时序 ensure。
- 如果当前已经在 Play 或正在启动 Play，需要先停止当前会话。

### 环境合批标记

入口：

```text
Level Editor 顶部工具栏 -> 性能工具 -> 环境合批标记
Tools -> DevKit 工具 -> 性能工具 -> 环境合批标记
```

用途：

- 给选中的静态环境 Actor 写入 `EnvBatch.Source.<Group>`。
- 给生成或特殊代理对象写入 `EnvBatch.Proxy.<Group>.Mid` / `EnvBatch.Proxy.<Group>.Low`。
- 给 baked 地面写入 `EnvBatch.Baked.Ground.Mid` / `EnvBatch.Baked.Ground.Low`。
- 给 baked 墙面写入 `EnvBatch.Baked.Wall.Mid` / `EnvBatch.Baked.Wall.Low`。
- 给不参与合批的对象写入 `EnvBatch.Exclude`。
- 显示选中对象的 Source/Proxy readiness、StreamingLevel、LOD0/LOD1、材质槽和静态 Mobility 情况。

使用规则：

- 同一个 Actor 在 `EnvBatch.Source / Proxy / Baked` 中只能属于一种角色。
- 不确定是否能合批的对象先标记 `EnvBatch.Exclude`。
- 只对静态环境对象做 Source/Baked 标记；角色、门、机关、拾取物、VFX、动态玩法对象先排除。

### 模型资产合规检查

入口：

- 性能工具快捷入口按钮 `模型资产合规检查`
- `Tools -> DevKit 工具 -> 美术资产工具 -> 模型资产合规检查`

用途：

- 扫描 `/Game/Art` 下的 `StaticMesh` 资产。
- 左侧显示目录列表、模型类型筛选和缩略图。
- 中间显示选中模型的编辑器预览。
- 顶部显示当前模型的阻断、警告和说明日志。
- 右侧显示合规检查项和可忽略警告设置。
- 不执行合批、不生成代理、不写 VT Atlas、不替换关卡资产。

当前模型合规标准：

- 只检查静态环境候选资产：地面、墙面、静态道具、混合静态环境。
- 角色、敌人、交互物、拾取物、动态/物理物、VFX、UI、武器分类会显示为“已排除”，不进入静态环境性能分级检查。
- 默认四档映射为 `Epic=LOD0 / High=LOD0 / Mid=LOD1 / Low=LOD1`。
- 静态环境候选资产至少需要 `LOD0` 和 `LOD1`；缺少 `LOD1` 是阻断项。
- 推荐 1 个材质槽；超过 1 个材质槽只给警告，可在窗口右侧勾选忽略。
- 简单碰撞不是强制项；没有简单碰撞只给警告，可在窗口右侧勾选忽略。需要碰撞的物件可以在场景中手动处理。
- 三角面数量会显示 `LOD0 / LOD1` 数值，但当前不作为警告或阻断标准。
- 暂不检查 UV、命名、Pivot、Bounds 和顶点数据。

美术要求：

- 普通静态环境模型至少维护 `LOD0` 和 `LOD1`。
- 如果模型属于角色、交互物、动态物或 VFX，不需要为了静态环境合批去补 LOD 或改材质槽。
- 多材质槽模型可以保留，但需要确认这是有必要的；否则建议拆分或整理为单材质槽。
- 没有碰撞的模型可以保留，但需要确认场景侧不会依赖该资产的默认碰撞。

### 模型设置入口

独立的 `模型分级配置` 窗口已经移除，不再生成 `DA_ModelPerformance_*` 配置资产。当前模型性能分级的资产侧入口统一收敛到 `模型资产合规检查` 和 StaticMesh 资产编辑器。

入口：

- Level Editor 顶部工具栏 `模型合规`。
- `Tools -> DevKit 工具 -> 美术资产工具 -> 模型资产合规检查`。
- 在 `模型资产合规检查` 右侧点击 `打开模型设置`，或在左侧模型树右键选择 `打开模型` / `打开模型位置（Content）`。

定位：

- `模型资产合规检查` 只做检查、分类、预览、状态覆盖和定位，不写入生成资产。
- LOD、碰撞、材质槽、Nanite、导入设置等模型自身数据，全部回到 StaticMesh 资产编辑器维护。
- 后续正式合批、代理、VT Atlas、替换和写生成资产仍在正式打包链执行，不在美术日常窗口中直接执行。

美术要求：

- 普通静态环境模型至少提供 LOD0、LOD1；缺少 LOD1 在合规检查中视为阻断。
- 当前默认四档检查口径为 `Epic=LOD0 / High=LOD0 / Mid=LOD1 / Low=LOD1`。
- 环境模型优先一个材质槽；多材质槽资产需要说明原因或拆分。
- 没有简单碰撞可以保留，但需要确认场景侧不会依赖该资产默认碰撞。

### 贴图命名规则表

入口：

- 性能工具快捷入口按钮 `贴图命名规则与检查`
- `Tools -> DevKit 工具 -> 美术资产工具 -> 贴图命名规则与检查`
- 在窗口中点击 `写入默认命名规则表`、`打开命名规则表` 或 `检查选中贴图命名`

产物：

```text
/Game/Generated/Performance/DA_MaterialTextureNamingRules_Default
Class: UYogMaterialTextureNamingConvention
```

默认规则：

| 通道 | 推荐后缀 | 兼容后缀 | 说明 |
| --- | --- | --- | --- |
| BaseColor | `_BaseColor`, `_BC` | `_D`, `_Diffuse`, `_Albedo`, `_A` | 颜色/albedo |
| Normal | `_Normal`, `_N` | `_NM`, `_NRM` | 法线贴图，线性采样 |
| ORM | `_ORM` | `_MRA`, `_RMA`, `_Packed`, `_MaskMap` | 默认 R=AO, G=Roughness, B=Metallic |
| Height | `_Height`, `_H` | `_Disp`, `_Displacement` | Source/bake 输入，Low runtime 不依赖 |
| Mask | `_Mask`, `_MSK` | `_Masks`, `_BlendMask`, `_UniqueMask` | 项目自定义遮罩 |
| LightInfo | `_LightInfo` | `_LI`, `_FakeLight`, `_MatLight` | 材质灯/假光照数据，独立降级 |
| Emissive | `_Emissive`, `_E` | `_Emission`, `_Glow` | 可选自发光 |

美术要求：

- 美术继续按单体资产提供 unique 贴图，不手工合 atlas/UDIM。
- 后续工具负责收集贴图、重排 UV、生成 UDIM/SVT 或等价 VT Atlas、写 PropertyTexture 和 MappingData。
- 同一材质族内 A/B/C set 的命名要一致，方便工具识别。
- 新增或调整后缀时，直接在 `DA_MaterialTextureNamingRules_Default` 中编辑规则；点击 `写入贴图命名规则` 后会自动打开资产，也可以之后用 `打开贴图规则` 再打开。

## 材质分级接口

运行时代码入口：

```text
UYogPerformanceSettingsLibrary::GetMaterialPerformanceInterfaceForTargetTier
UYogPerformanceSettingsLibrary::GetMaterialPerformanceInterfaceForGraphicsSettings
FYogMaterialPerformanceTierInterface
```

接口字段控制：

| 字段 | 用途 |
| --- | --- |
| `MaterialQuality` | 材质分支、混合层数、复杂效果开关 |
| `TextureQuality` | 贴图分辨率、mip、streaming 预算 |
| `VTAtlasQuality` | VT Atlas cell、mip、通道质量 |
| `DynamicOverlayQuality` | RVT/动态贴花/overlay 数量和质量 |
| `MaterialLightQuality` | 材质灯/LightInfo 质量 |
| `MaterialLightMaxLightInfoCount` | 单材质最多可读取的 LightInfo 数量 |
| `MaxUniqueTextureSets` | Source 主材允许采样的 unique texture set 数 |
| `MaxRuntimeBlendLayers` | runtime 保留的材质混合层数 |
| `MaxRuntimeOverlayLayers` | runtime overlay 层数 |
| `bPreferSourceMasterMaterial` | 当前挡位是否优先 Source 主材 |
| `bPreferBakedMaterial` | 当前挡位是否优先 bake 后材质 |
| `bUseVTAtlas` | 环境合批是否默认使用 VT Atlas |
| `bAllowGroundRuntimeRVTOverlay` | 地面 bake 后是否允许动态 RVT/overlay |
| `bAllowWallRuntimeRVTOverlay` | 墙面是否默认允许 runtime RVT overlay，当前默认 false |
| `bBakeStaticDecalsIntoSurface` | 静态贴花是否进入 bake 结果 |

当前模板策略：

- Source 可编辑母材使用引擎材质节点，不使用项目 `.ush`。
- `M_Env_MasterA_Source` 用于美术可见的 Source 材质模板，当前已通过 UE MCP MaterialTools/ProgrammaticToolset 创建并保存。
- `MI_Env_MasterA_Source_Epic/High/Mid/Low` 表达四档默认参数。
- `M_Env_Baked_VTAtlas` 用于 GeneratedBatchProxy/Baked，合同是 `TexCoord7.x -> _PropTexture row -> VT_Atlas UVRect`。

生成命令：

```powershell
UnrealEditor-Cmd.exe X:\Project\Dev01\DevKit.uproject -run=MaterialPerformanceTemplateSetup -Apply -Force
UnrealEditor-Cmd.exe X:\Project\Dev01\DevKit.uproject -run=MaterialBatchParentMaterialSetup -Apply -Force
```

当前 Codex 会话已经连接 UE MCP；Source 母材和四个 Source MI 已通过 `MaterialTools`、`MaterialInstanceTools`、`ProgrammaticToolset` 创建并反查。`MaterialPerformanceTemplateSetup` commandlet 保留为自动化重建入口。

## 地面和墙面处理

地面：

- Source 阶段允许多层材质、高度混合、mesh paint、顶点色、静态贴花混合。
- 静态效果在打包/批处理阶段 bake 到 VT Atlas 或 baked texture。
- bake 后只允许动态残留效果继续走 RVT/overlay，例如脚印、湿痕、临时污染。
- Low 默认关闭非必要动态 overlay。

墙面：

- Source 阶段同样允许 mesh paint、顶点色、高度混合、静态贴花。
- 默认不推荐墙面全量 runtime RVT；优先做 wall static bake 到 VT Atlas 或 baked texture。
- 需要动态变化的墙面效果用局部 runtime decal/overlay，不进入静态 bake。

## 功能测试方式

本阶段不做最终验收，只提供功能如何测试。

### 测试环境合批标记

1. 打开一个测试 Streaming Level。
2. 选中几个静态环境 Actor。
3. 打开“环境合批标记”，填写 Group，例如 `Prison_S_01`。
4. 点击 `标记 Source.<Group>`。
5. 确认 Actor Tags 中出现 `EnvBatch.Source.Prison_S_01`。
6. 点击 `刷新选择` 或重新选择对象，确认窗口只显示 Source/Proxy/Baked/Exclude tag 统计，不再要求模型 LOD 或贴图规则配置。

### 测试模型资产合规检查

1. 打开 `模型资产合规检查` 窗口。
2. 点击 `刷新 /Game/Art`。
3. 在左侧用模型类型筛选查看静态环境候选、地面、墙面、静态道具、已排除分类、阻断、警告和通过资产。
4. 选中一个 StaticMesh，确认中间预览窗口显示模型缩略预览。
5. 确认右侧显示 LOD 数量、默认映射 `Epic/High -> LOD0，Mid/Low -> LOD1`、材质槽数量、碰撞状态和 LOD0/LOD1 三角面。
6. 对只有 LOD0 的静态环境模型，确认状态为“阻断”。
7. 对多材质槽模型，确认状态为“警告”；勾选 `忽略多材质槽警告` 后该警告不再阻断当前判断。
8. 对无简单碰撞模型，确认状态为“警告”；勾选 `忽略碰撞警告` 后该警告不再影响当前判断。
9. 角色、交互物、动态物、VFX、UI、武器目录下的模型应显示为“已排除”，不参与静态环境合规统计。

### 测试模型设置入口

1. 打开 `模型资产合规检查` 窗口。
2. 在左侧展开目录并选择一个 StaticMesh。
3. 点击右侧底部 `打开模型设置`。
4. 确认 Content Browser 定位到该 StaticMesh，并打开 StaticMesh 资产编辑器。
5. 在资产编辑器中确认或调整 LOD、碰撞、材质槽和导入设置。
6. 返回 `模型资产合规检查`，点击 `刷新 /Game/Art` 验证状态是否更新。

### 测试贴图命名规则表

1. 打开 `贴图命名规则与检查` 窗口。
2. 点击 `写入默认命名规则表`。
3. 确认工具打开或保存 `/Game/Generated/Performance/DA_MaterialTextureNamingRules_Default`。
4. 在 Content Browser 选中需要检查的 `Texture2D`。
5. 点击 `检查选中贴图命名`，确认 BaseColor、Normal、ORM、Height、Mask、LightInfo、Emissive 后缀和 sRGB 检查结果。
6. 后续项目新增命名习惯时，只扩展该表，不改工具硬编码。

### 测试材质模板生成

1. 如果需要重建，运行 `MaterialPerformanceTemplateSetup -Apply -Force`；如果只验证当前结果，直接检查 MCP 已创建的资产。
2. 检查是否存在 `M_Env_MasterA_Source` 和四个 Source MI。
3. 打开 `M_Env_MasterA_Source`，确认存在 `T_BaseColor_A/B/C`、`T_Normal_A`、`T_ORM_A`、`T_Height_B/C`、`T_LightInfo`、`TierMaterialQuality`、`MaxRuntimeBlendLayers` 等参数；也可以通过 MCP `MaterialInstanceTools.list_parameters` 反查。
4. 打开四个 MI，确认 Epic/High/Mid/Low 的 scalar 参数不同。
5. 运行 `MaterialBatchParentMaterialSetup -Apply -Force` 后，检查 `M_Env_Baked_VTAtlas` 是否有 `VT_Atlas`、`_PropTexture`、`BatchRowCount`、`TexCoord7.x` 采样链。

### 后续正式验收才需要做

- 全项目资产审计。
- 场景灯光预算统计。
- 地面/墙面最终 bake 视觉 parity。
- VT pool 与普通 streaming pool 对比。
- Source/Proxy/Baked 互斥加载验证。
- runtime profiling 和提交 gate。

## 2026-06-30 工具拆分更新

当前性能工具不再把所有能力塞进“环境合批标记”窗口，编辑器里按下面四个美术/TA 可见入口使用；真正合批命令移到正式打包链执行：

1. `性能工具快捷入口`
   - 入口：Level Editor 顶部工具栏 `性能工具`，或 `Tools -> DevKit 工具 -> 性能工具 -> 性能工具快捷入口`。
   - 用途：集中打开“从主菜单运行”和“环境合批标记”。
   - 窗口内有推荐顺序：先到美术资产工具检查模型和贴图 -> 环境打 tag -> 正式打包链合批。

2. `环境合批标记`
   - 只负责给选中的关卡 Actor 写 `EnvBatch.*` Actor Tag。
   - 可以标记普通 StaticMeshActor、带 StaticMeshComponent 的蓝图，或已经处理好的 Level Instance 边界对象。
   - 一个合批组可以是不连续的一组对象；是否一起合并由相同 `EnvBatch.Source.<Group>` / `Cluster` 决定。
   - 这个窗口不再写模型 LOD 配置，不再写贴图命名规则。

3. `模型资产合规检查`
   - 入口：Level Editor 顶部工具栏 `模型合规`，或 `Tools -> DevKit 工具 -> 美术资产工具 -> 模型资产合规检查`。
   - 用法：扫描 `/Game/Art` 下 `StaticMesh`，左侧筛选和缩略图，中间预览模型，右侧查看 LOD、材质槽、碰撞、三角面和分类排除。
   - 原则：美术日常只看这个窗口，默认 `Epic/High=LOD0`、`Mid/Low=LOD1`；材质槽和碰撞是可忽略警告，三角面只展示。
   - `打开模型设置` 会定位并打开 StaticMesh 资产编辑器；不再生成单独模型分级配置资产。
   - 合批命令、代理生成、VT Atlas 和关卡替换不在这个窗口执行。

4. `贴图命名规则与检查`
   - 入口：Level Editor 顶部工具栏 `贴图规则`，或 `Tools -> DevKit 工具 -> 美术资产工具 -> 贴图命名规则与检查`。
   - 用法：先点击 `写入默认命名规则表` 生成 `/Game/Generated/Performance/DA_MaterialTextureNamingRules_Default`；之后在 Content Browser 选中 `Texture2D`，点击 `检查选中贴图命名`。
   - 检查内容：贴图后缀是否命中 BaseColor、Normal、ORM、Height、Mask、LightInfo、Emissive 规则，以及 sRGB 是否符合该通道预期。
   - 美术仍然提交单体 unique 贴图，不手工维护 atlas/UDIM；后续自动合批工具负责 VT Atlas / baked 输出。

5. `关卡模型材质合批`
   - 当前不再作为美术快捷入口或 Tools 菜单入口暴露。
   - 该能力保留在正式打包链/TA 自动化里使用：根据 `EnvBatch.Source.<Group>`、模型合规结果和贴图命名规则生成 dry-run 报告、VT Atlas、Mapping、PropertyTexture、ProxyMesh、BatchMaterial 和替换计划。
   - 全量关卡替换仍必须经过正式打包链 gate 和报告审核，不能在美术日常窗口中直接执行。
