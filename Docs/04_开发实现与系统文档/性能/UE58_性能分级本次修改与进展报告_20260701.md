# UE5.8 性能分级本次修改与进展报告

日期：2026-07-01
目标分支：`main`
范围：Epic / High / Mid / Low 四档性能分级、模型资产合规检查、环境合批标记、贴图命名规则、材质/VT 合批接口、运行时画质设置和自动化验证。

## 1. 本次结论

本次已经把项目的性能分级基础从“方案文档”推进到“可在编辑器里使用的生产入口 + 运行时四档设置 + 自动化检查”的阶段。

当前正式口径如下：

- 只保留 `Epic`、`High`、`Mid`、`Low` 四个目标挡位。
- 当前场景组织按传统 Streaming Level 处理，不启用 World Partition，不引入 Data Layer 作为前提。
- Lumen 保持项目默认开启；VSM 不作为默认开启项，避免为非 Nanite 主线额外增加阴影成本。
- 模型资产分级不再生成单独的 `DA_ModelPerformance_*` 配置资产；模型自己的 LOD、材质槽、碰撞和导入设置直接在 StaticMesh 资产内维护。
- 合批执行不暴露给美术日常入口；正式关卡模型材质合批、VT Atlas、ProxyMesh、替换计划等放到打包/TA 自动化链执行。
- 材质侧本次只提供性能分级接口、Source/Baked/VT Atlas 管线契约和贴图命名规则，不直接替美术制作最终复杂材质网络。

## 2. 已完成修改

### 2.1 运行时性能分级

- 增加/整理 `Epic`、`High`、`Mid`、`Low` 四档画质枚举、保存数据和 Blueprint 可调用接口。
- 更新图形设置 UI，使四档设置可以保存、读取并应用到运行时。
- 更新 `DefaultScalability.ini`、`DefaultDeviceProfiles.ini`、`DefaultEngine.ini`，把项目默认画质、Lumen 和性能档位设置纳入配置。
- 增加性能设置相关自动化测试，覆盖档位映射、保存和运行时应用路径。

### 2.2 编辑器工具入口

- 恢复“从主菜单运行”的编辑器白色播放箭头，读取 `YogGameInstanceBase.FrontendMap`，当前默认进入 `/Game/Maps/L_EntryMenu`。
- 新增顶部工具栏入口：
  - `性能工具`
  - `模型合规`
  - `贴图规则`
- 新增菜单入口：
  - `Tools -> DevKit 工具 -> 性能工具 -> 性能工具快捷入口`
  - `Tools -> DevKit 工具 -> 性能工具 -> 环境合批标记`
  - `Tools -> DevKit 工具 -> 美术资产工具 -> 模型资产合规检查`
  - `Tools -> DevKit 工具 -> 美术资产工具 -> 贴图命名规则与检查`
- 工具显示文案已中文化。

### 2.3 模型资产合规检查

- 新增 `模型资产合规检查` 编辑器窗口。
- 扫描 `/Game/Art` 下 StaticMesh，按资产目录和分类进行预览。
- 支持左侧目录树展开/选择，右侧切换状态后尽量保留目录展开和选择位置。
- 支持模型分类筛选：
  - 交互物
  - 角色
  - 武器
  - 道具
  - 场景模型-物件
  - 场景模型-建筑
  - 场景模型-地面
  - 场景模型-其他
  - 贴花模型
  - 其他资产
- 支持状态设置切换：自动、通过、警告、阻断、排除。
- 默认静态环境模型合规口径为 `Epic/High=LOD0`、`Mid/Low=LOD1`，至少需要 LOD0 和 LOD1。
- 材质槽和碰撞作为可忽略警告；三角面数只展示，不作为当前阻断标准。
- 支持模型预览、富文本彩色检查信息、警告/阻断日志。
- 右侧底部 `打开模型设置` 会定位并打开选中 StaticMesh 资产编辑器。
- 左侧列表支持右键打开模型或在 Content Browser 中定位模型。
- 删除了独立 `模型分级配置` 窗口和相关生成资产逻辑，避免把单模型设置分散到额外 DataAsset。

### 2.4 环境合批标记

- `环境合批标记` 现在只负责写 Actor Tag，不再混入模型 LOD 配置或贴图规则编辑。
- 支持 `EnvBatch.Source.<Group>`、`EnvBatch.Proxy.<Group>.<Tier>`、`EnvBatch.Baked.*`、`EnvBatch.Exclude` 等合批语义。
- 合批组可以是不连续 Actor；普通 StaticMeshActor、带 StaticMeshComponent 的蓝图、处理好的 Level Instance 边界对象都可以按 Actor Tag 进入后续扫描。
- 新增/保留命令行自动标记路径，供后续正式打包链读取。

### 2.5 贴图命名规则与检查

- 新增 `UYogMaterialTextureNamingConvention` 资产类型。
- 新增默认命名规则资产：
  - `/Game/Generated/Performance/DA_MaterialTextureNamingRules_Default`
- 新增 `贴图命名规则与检查` 编辑器窗口。
- 支持写入默认规则、打开规则资产、检查 Content Browser 选中 `Texture2D` 的后缀和 sRGB 设置。
- 命名规则覆盖 BaseColor、Normal、ORM、Height、Mask、LightInfo、Emissive 等通道。
- 美术继续提交单体 unique 贴图，不要求手工维护 atlas、UDIM 或 SVT tile。

### 2.6 材质和 VT 合批接口

- 保留并整理材质性能分级接口文档，明确 Source 材质和 Baked/VT Atlas 材质的职责边界。
- 当前生产主线为 UDIM-style SVT / VT Atlas 虚拟大贴图合批管线，不走 Texture2DArray 主线。
- 地面和墙面 Source 阶段允许多层材质、高度混合、mesh paint、顶点色和静态贴花混合；正式低档结果在 bake 后进入 VT Atlas 或 baked texture。
- 动态地面效果放在 bake 后的 RVT/overlay 位置处理；墙面默认不做全量 runtime RVT，优先做静态 bake + 局部动态 overlay。
- `M_Env_Baked_VTAtlas`、PropertyTexture、MappingData、TexCoord7.x 行索引等合批合同已进入代码和文档。

### 2.7 自动化脚本与报告

- 增加或更新构建验证、材质批处理 dry-run、材质模板 setup、工具可用性、环境合批工具检查、artifact contract 检查、运行时 profiling fallback 等自动化脚本。
- 自动化测试已同步新的口径：确认旧的 `SModelPerformanceConfigWidget`、`DevKitModelPerformanceConfig`、`UYogModelPerformanceConfig` 不再作为功能入口存在。

### 2.8 清理内容

- 删除旧的 `aabb.7z`。
- 删除旧的 `Content/Generated/MaterialBatch/Medium/FloorBrick03_Probe/*` 生成资产。
- 删除临时生成的 `DA_ModelPerformance_Corridor_01b.uasset`，因为模型分级配置资产路线已取消。
- 新增 `Content/Generated/MaterialBatch/Mid/*` 样例生成资产和默认贴图命名规则资产。

## 6. 注意事项

- 当前模型合规工具的“排除”是资产分类和状态，不代表删除资产，也不代表正式合批时会强制修改资产。
- 右侧状态设置用于定义单个模型在合规检查中的状态；左侧目录筛选会基于该分类/状态刷新，但应尽量保留当前展开目录和选择。
- 合批 tag 只是后续自动化扫描依据，不直接执行合并和替换。
- 正式合批必须在打包/TA 链执行，并先审核 dry-run 报告。
- VT 和非 VT 混用需要继续关注普通 streaming pool 与 VT physical pool 的重复驻留风险；本次主线仍以 SVT/VT Atlas 为准。
- 本次包含生成 `.uasset` 资源，后续如果对视觉内容有要求，仍需要在 Unreal Editor 中打开检查。
