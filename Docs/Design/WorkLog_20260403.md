# 工作日志 2026-04-03

## 概要

本次开发完成了三个主要系统：**符文动态 GE 系统**、**RuneStatEffect 预制 GE 类**、**BuffFlow 可视化逻辑系统**。

---

## 一、符文动态 GE 系统（DA 驱动属性修改）

### 改动目的
解决"每种数值组合都要创建一个 GE 资产"的爆炸问题。现在 DA 里直接填属性+数值，运行时动态构建 GE。

### 新增

| 文件 | 说明 |
|------|------|
| `RuneDataAsset.h` 中 `FRuneAttributeModifier` | 属性修改器结构体（属性+操作类型+数值） |

### 修改

| 文件 | 改动 |
|------|------|
| `RuneDataAsset.h` | `FRuneInstance` 添加 `TArray<FRuneAttributeModifier> AttributeModifiers`，`BehaviorEffect`（替代旧 `ActivationEffect`），`BuffFlowAsset` |
| `YogAbilitySystemComponent.h/.cpp` | 新增 `ApplyRuneModifiers()` / `RemoveRuneModifiers()` —— 从数组动态构建 GE 并 Apply |
| `YogAbilitySystemComponent.h/.cpp` | 新增 `FDealtDamageDelegate DealtDamage` 委托 + `ReceiveDamage()` 中广播 |
| `BackpackGridComponent.h` | `FPlacedRune` 添加 `BehaviorEffectHandle` |
| `BackpackGridComponent.cpp` | `ActivateRune()` / `DeactivateRune()` 支持数值GE + 行为GE + BuffFlow 三通道 |
| `DamageExecution.cpp` | 恢复 `ReceiveDamage()` 调用（之前被注释掉） |

## 二、RuneStatEffect 预制 GE 类

### 改动目的
为常用数值符文提供 C++ 预制 GE，在编辑器里直接用不需要配置。

### 新增

| 文件 | 说明 |
|------|------|
| `RuneStatEffect.h/.cpp` | 10 个预制 GE 类（Attack、AttackPower、MaxHealth、AttackSpeed、MoveSpeed、CritRate、CritDamage、DmgTaken、Dodge、AttackRange） |

> 注：这些预制类在引入动态 GE 系统后变为可选。如果 DA 的 AttributeModifiers 已经覆盖了需求，这些类可以不使用。

## 三、BuffFlow 可视化逻辑系统

### 改动目的
让非技术人员（策划）能通过 FlowGraph 拖拽节点组合 Buff 效果，无需 Blueprint。

### 新增

| 文件 | 说明 |
|------|------|
| `BuffFlowTypes.h` | `EBFTargetSelector` 枚举 |
| `BuffFlowComponent.h/.cpp` | Flow 生命周期管理组件（挂在角色上） |
| `BFNode_Base.h/.cpp` | 节点基类，提供 GetASC/GetBuffOwner 快捷方法 |
| `BFNode_AddBuff.h/.cpp` | 添加Buff 节点 |
| `BFNode_RemoveBuff.h/.cpp` | 移除Buff 节点 |
| `BFNode_FinishBuff.h/.cpp` | 结束当前Buff 节点 |
| `BFNode_AddTag.h/.cpp` | 添加标签 节点 |
| `BFNode_RemoveTag.h/.cpp` | 移除标签 节点 |
| `BFNode_HasTag.h/.cpp` | 检查标签 分支节点 |

| `BFNode_OnDamageReceived.h/.cpp` | 当受到伤害时 触发器节点 |
| `BFNode_GetAttribute.h/.cpp` | 获取属性值 节点 |
