## 一、设计原则

**打断次数控制在 2 次以内，剩余认知由玩家自行发现。**

| 原则 | 说明 |
|------|------|
| 单次单概念 | 每次弹窗只传递一个核心信息 |
| 被动优于主动 | 能用世界 UI 传递的信息不弹窗 |
| 循环一致性 | 引导流程与正式游戏循环保持一致（三选一在战斗后） |
| 发现优于告知 | 热度升阶、激活区扩展让玩家自己感受，不提前解释 |

## 二、引导流程（完整版）

```
玩家出生
  │
  ▼ [被动] 武器 Actor 世界浮窗
  │   玩家靠近武器时显示，离开隐藏
  │   显示：武器名 / 战斗风格 / 激活区形状预览 / 天然符文
  │   不强制停止，玩家可自由走动
  │
  ▼ [打断 ①] 按 E 拾取武器 → 强制打开背包
  │   背包自动打开
  │   武器天然符文自动放入中央格子
  │   基础符文预放在激活区内
  │   ──────────────────────────
  │   弹窗（只说一件事）：
  │   "激活区内的符文才会在战斗中生效。
  │    你可以移动符文调整位置。"
  │   [知道了 →]
  │   ──────────────────────────
  │   玩家自由调整符文 / 直接关闭背包
  │
  ▼ 进入战斗
  │   热度系统正常运作
  │   第一次升阶 → 角色身上粒子光效（无弹窗，玩家自己观察）
  │   战斗中可打开背包查看（只读模式）
  │     - 激活区边框随热度颜色变化
  │     - 尝试移动符文 → 抖动 + 红色闪光 + 无法移动
  │
  ▼ 战斗结束 → 掉落拾取物 → 玩家按 E 拾取
  │
  ▼ 三选一界面弹出（正式循环顺序）
  │   玩家选择符文
  │
  ▼ [打断 ②] 三选一后自动打开背包
  │   符文进入待放置区（高亮显示）
  │   ──────────────────────────
  │   弹窗（只说一件事）：
  │   "把新符文移动到你想要的位置。
  │    进入下一关后无法再调整。"
  │   [知道了 →]
  │   ──────────────────────────
  │   玩家自由配置符文 / 关闭背包
  │
  ▼ 进入下一关，引导完成
```

## 三、各节点详细规格

### 节点 A — 武器世界浮窗（被动）

**触发**：玩家进入 WeaponSpawner Overlap 范围  
**消失**：玩家离开范围 / 拾取武器后  

**UI 内容**：

```
┌──────────────────────────┐
│  ⚔  大剑                  │
│  战斗风格：格挡 · 韧性    │
│                           │
│  激活区：[ ■ ■ ■ □ □ ]   │
│                           │
│  天然符文：               │
│    铁壁（被动）           │
│    破甲斩（主动）         │
│                           │
│      [ 按 E 拾取 ]        │
└──────────────────────────┘
```

**实现**：`UWidgetComponent`（Screen Space），附加到 WeaponSpawner。  
从 `WeaponDefinitionDA` 读取武器名 / 风格 / 激活区预览 / 天然符文列表。

---

### 节点 B — 打断 ① 背包引导弹窗

**触发**：`TryPickupWeapon()` 成功，且 `TutorialState == NeedWeaponTutorial`  
**流程**：

1. 武器拾取动效
2. 延迟 0.4s
3. 打开背包（`OpenBackpack()`）
4. 天然符文自动入中央格
5. 基础符文预置激活区内格
6. 显示引导弹窗

**弹窗规格**：

| 属性 | 内容 |
|------|------|
| 标题 | 配置你的符文 |
| 正文 | "激活区内的符文才会在战斗中生效。你可以拖动符文调整位置。" |
| 高亮箭头 | 指向激活区格子 |
| 按钮 | "知道了" |

**完成后**：`TutorialState = WeaponTutorialDone`，玩家可自由操作背包。

---

### 节点 C — 战斗阶段背包只读模式

**触发**：`BackpackGridComponent.bIsInCombat == true` 时打开背包  

**视觉规格**：

| 元素 | 表现 |
|------|------|
| 激活区边框 | 随 Phase 变色：0=蓝，1=青，2=橙，3=红橙，+ 脉动光晕 |
| 激活区内 | 浮动粒子光效，强度随 Phase 递增 |
| 未激活符文 | 半透明灰暗（透明度 50%）|
| Tab 键 | 可切换预览其他 Phase 的激活区范围（边框虚线显示） |

**交互锁定**：

| 操作 | 行为 |
|------|------|
| 点击/拖拽符文 | 符文 X 轴抖动（3次，8px，0.08s/次）|
| 锁定视觉 | 符文边框红色闪烁（亮 0.1s → 灭 0.1s，共 2 次）|
| 提示文字 | 屏幕左下角淡入淡出："战斗中无法移动符文"（1s 消失）|

---

### 节点 D — 三选一界面（战斗后）

**触发**：玩家拾取掉落物（正常结算流程）  
**内容**：与正式游戏循环完全一致，无额外教学内容  
引导阶段符文池使用**高感知符文**（击杀爆炸 / 燃烧印记 / 急速涌动）

---

### 节点 E — 打断 ② 战斗后背包引导弹窗

**触发**：三选一完成，且 `TutorialState == NeedPostCombatTutorial`  
**流程**：

1. 三选一 UI 关闭
2. 延迟 0.2s
3. 打开背包，符文进入待放置区（高亮脉动）
4. 显示引导弹窗

**弹窗规格**：

| 属性 | 内容 |
|------|------|
| 标题 | 放置你的新符文 |
| 正文 | "把新符文移动到你想要的位置。进入下一关后，战斗中将无法调整。" |
| 高亮箭头 | 指向待放置区 |
| 按钮 | "知道了" |

**完成后**：`TutorialState = Completed`，后续不再触发引导。

## 四、引导符文池（节点 D 使用）

| 符文名 | 效果 | 选择原因 |
|--------|------|----------|
| 击杀爆炸 | 击杀时触发小范围爆炸 | 制造顿悟时刻，和热度升阶结合效果明显 |
| 燃烧印记 | 攻击附加燃烧粒子 | 每次攻击可见，感知直接 |
| 急速涌动 | 激活时攻速 +40% | 手感立即变化，玩家能感受到差异 |

## 五、新增系统需求（程序）

### 5.1 ETutorialState 枚举

```cpp
UENUM()
enum class ETutorialState : uint8
{
    None,
    NeedWeaponTutorial,      // 等待拾取武器
    WeaponTutorialDone,      // 武器引导完成，等待战斗后
    NeedPostCombatTutorial,  // 等待战斗后三选一完成
    Completed                // 引导全部完成
};
```

### 5.2 BackpackGridComponent 新增

```cpp
// 战斗阶段标志（GameMode 写入）
bool bIsInCombat = false;

// 热度预览阶段（Tab 键切换）
int32 PreviewPhase = 0;

// 返回指定热度的激活格列表（用于预览渲染）
TArray<FIntPoint> GetActivationZoneForPhase(int32 Phase) const;
```

### 5.3 WBP_TutorialPopup（通用弹窗）

> **实现状态**：C++ 完整（`UTutorialPopupWidget`，父类 `UCommonActivatableWidget`）

**WBP 必须放置的命名控件（BindWidget，名字一字不差）：**

| 控件名 | 类型 | 说明 |
|--------|------|------|
| `TitleText` | TextBlock | 当前页标题 |
| `BodyText` | TextBlock | 当前页正文（AutoWrapText 打勾） |
| `BtnConfirm` | Button | 翻页 / 关闭按钮 |
| `BtnConfirmLabel` | TextBlock（Button 内） | 非末页显示"下一页"，末页显示"知道了" |

**翻页逻辑（全在 C++ 里，WBP 不需要写节点）：**

- `ShowPopup(TArray<FTutorialPage>)` — 传入多页数据，自动激活 Widget 并暂停游戏
- `OnNextPressed()` — BtnConfirm 点击回调（`NativeConstruct` 里自动绑定）：
  - 当前页 < 末页 → 翻页，刷新 TitleText / BodyText / BtnConfirmLabel
  - 当前页 = 末页 → `DeactivateWidget()`，游戏恢复

**FTutorialPage 结构（在 `GameDialogWidget.h` 定义）：**

```cpp
USTRUCT(BlueprintType)
struct FTutorialPage
{
    FText Title;
    FText Body;
};
```
