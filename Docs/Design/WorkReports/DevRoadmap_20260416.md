# 开发路线图 2026-04-16

> 基础：4.15 内测版本完成 + 玩家调研反馈  
> 目标：感知强化 → 内容扩充 → 深度构建  
> 适用人群：策划 + 程序  
> 最后更新：2026-04-16

---

## 一、内测核心结论

**玩家能感知到：** 战斗基础手感可用、背包拼图概念有新意  
**玩家感知不到：** 热度系统的作用、符文有没有激活、两个系统有没有联动  
**玩家想要更多：** 武器差异、关卡多样性、更多符文选择

> **一句话方向：先让已有内容被看见，再往里填新内容。**

---

## 二、阶段规划

```
Phase 1（感知强化）  → 让现有三大系统被玩家看见和理解
Phase 2（内容扩充）  → 新武器 + 更多关卡 + 更多符文
Phase 3（深度构建）  → 流派路线 + 符文协同 + Meta 进度
```

---

## Phase 1 — 感知强化

> **目标：下一次测试时，玩家不需要解释就能感知热度-背包联动**

### 1.1 热度系统可视化【最高优先】

| 任务 | 说明 | 类型 |
|------|------|------|
| 热度条动态效果 | 升阶时热度条发光 + 抖动，不是静态数字变化 | UI |
| 升降阶角色特效 | 升阶：角色身上短暂粒子光效（0.5s）；降阶：粒子消散 | VFX |
| Phase 数字显示 | HUD 上明确显示当前 Phase（0/1/2/3）+ 进度条 | UI |
| 激活区边框脉动 | 热度上升时激活区边框同步脉动扩张，颜色与热度条同源 | UI |

> 原则：玩家在**激烈战斗中不看背包**，所以热度状态必须通过**角色本体 VFX** 传递。

---

### 1.2 符文激活感知【最高优先】

| 任务 | 说明 | 类型 |
|------|------|------|
| 激活/未激活视觉对比 | 激活格：高亮 + 光环；未激活格：半透明灰暗 | UI |
| 符文进入激活区瞬间 | 播放激活音效 + 符文亮起动效 | UI + Audio |
| 符文失活瞬间 | 播放失活音效（低沉）+ 符文暗淡收缩 | UI + Audio |
| 符文效果可见化 | 测试期间优先使用 AoE 爆炸/击飞/回血类符文，禁用被动数值类 | 配置 |

---

### 1.3 战斗表现补全

| 任务 | 说明 | 优先级 |
|------|------|--------|
| 玩家受击动画 | Hit React 蒙太奇（轻击/重击两种），当前受伤无反应 | P0 |
| 命中停顿（Hit Stop） | 命中时 0.05-0.08s 时间缩放，增强打击感 | P0 |
| 敌人受击反馈 | 命中时敌人材质闪白 + 身体抖动 | P1 |
| 伤害数字 | 飘字：普通伤害白色，暴击黄色，受伤红色 | P1 |

---

### 1.4 新手第一房间设计【工程化顿悟】

> 不依赖文字说明，用关卡设计让玩家"自己发现"系统。

**配置规则（不改代码，只改数据）：**

1. 第一房间预放 **1 个符文**，刻意放在 Phase 1 激活区边界外
2. 该符文效果：**击杀触发小范围爆炸**（视觉直接）
3. 房间敌人数量：**刚好让玩家在战斗中自然触发 1 次升阶**
4. 第一关通关后 **强制进入背包调整**（不可跳过，全屏显示）

**目标体验链：**
```
普通攻击 → 热度上升 → Phase 1 → 符文亮起 → 下一刀触发爆炸
    → 玩家："这是什么？！"
被打一下 → Phase 0 → 符文熄灭 → 爆炸没了
    → 玩家："我要维持高热度"
```

## Phase 2 — 内容扩充

> **目标：让每次游玩感觉不一样；给玩家"更多东西可选"**

### 2.1 第二武器：匕首

> 与大剑形成明确对比，差异化不只是数值

| 维度 | 大剑（现有） | 匕首（新增） |
|------|------------|------------|
| 风格 | 重甲、格挡、韧性强 | 轻盈、连刺、闪避驱动 |
| 连击结构 | 3 段重击 → 终结踏击 | 5 段轻刺 → 爆发突进 |
| 热度驱动 | 完美格挡升热度 | 完美闪避升热度 |
| 激活区形状 | 中央圆形扩展 | 斜角扇形（差异化激活区） |
| 专属符文方向 | 格挡反弹、破甲、地震 | 毒、出血、速度残影 |
| 目标玩家 | 核心/种子玩家 | 核心玩家 + 泛用户（手速型） |

**实现步骤：**
1. 新建 `BP_GA_DaggerCombo`（参考 MeleeAttack 连击架构）
2. 新建匕首蒙太奇（5 段 + 终结）
3. 配置匕首专属 `EffectContainerMap`（毒/出血/速度）
4. 结算界面加武器选择入口（或初始选择）

---

### 2.2 更多关卡

> 目标：3 种不同主题的战斗房间，增加每局新鲜感

| 关卡类型 | 特点 | 说明 |
|----------|------|------|
| 开阔平原（现有） | 无遮挡，正面对决 | 当前测试关卡 |
| 狭窄走廊 | 敌人拥挤，AoE 价值高 | 有利于爆炸类符文流派 |
| 环境危机房间 | 地板陷阱/定时危险 | 增加走位决策维度 |

**每种房间额外配置：**
- 独立 `DA_Room` 资产（专属敌人池 + 触发条件）
- 关卡 Buff 配置（如"狭窄走廊"给敌人攻击速度 +20%）

---

### 2.3 更多符文（目标：20 种）

> 当前目标 ≥8 种（调研要求）；Phase 2 目标 20 种，覆盖 3 大流派

**扩充方向：**

| 流派 | 符文方向 | 数量目标 |
|------|----------|----------|
| 爆发流（硬核） | 暴击倍率、击杀爆炸、热度暴增 | 5 种 |
| 持续流（轮椅） | 持续回血、护盾充能、永久激活 | 5 种 |
| 控场流（策略） | 减速光环、击退强化、地形锁定 | 5 种 |
| 武器专属 | 大剑 3 种 + 匕首 3 种 | 6 种 |

**符文设计原则（配合感知强化）：**
- 每个符文激活时必须有**肉眼可见的视觉变化**
- 禁止纯数值被动（如"攻击力+10%"没有任何视觉）
- 优先设计触发型效果（击杀时/完美格挡时/热度满时）

## Phase 3 — 深度构建

> **目标：让种子玩家产生"上头"感，愿意多玩几局研究 Build**

### 3.1 流派路线明确化

| 路线 | 核心机制 | 配套设计 |
|------|----------|----------|
| 硬核流 | 外圈高价值符文，只在 Phase 3 激活 | 高风险高收益，需维持满热度 |
| 轮椅流 | 内圈永久激活符文，稳定输出 | 低门槛，容错高，伤害上限低 |
| 混合流 | 内外圈合理分布，Phase 2 为甜蜜点 | 平衡型，适合核心玩家 |

### 3.2 符文协同提示

- 两个符文放在相邻格子时，显示"协同"图标
- 选符文时提示已有符文的协同方向
- 构建"啊哈时刻"：第一次触发协同 = 双符文同时发光 + 爆发效果

### 3.3 Meta 进度（局外成长）

- 解锁新符文进入奖励池（每局通关解锁 1 个）
- 武器解锁（通关 N 次后解锁匕首）
- 起始背包格子扩展（局外升级购买）
