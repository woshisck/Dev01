## 二、已确认设计决策

> 本节记录 2026-04-10 设计问答中确认的所有决策，后续开发直接参照此表。

### 2.1 死亡机制

| 项目 | 设计 |
|---|---|
| 死亡结果 | 跑局结束，所有局内进度清零（金币 / 符文 / HP）|
| 返回位置 | 局外大厅 |
| 参考 | 类《黑帝斯》——局内全重置，局外永久进度保留 |
| 技术影响 | GameInstance 需在局结束时清空局内临时数据，保留局外资源计数 |

### 2.2 跨关状态继承

| 数据 | 继承方式 |
|---|---|
| 当前楼层编号 | `GI->PendingNextFloor`（已实现）|
| 玩家符文背包 | ⚠️ 未实现，需在 ConfirmArrangementAndTransition 前写入 GI |
| 当前 HP | ⚠️ 未实现，需写入 GI，新关卡加载后恢复 |
| 金币 | ⚠️ 未实现，需写入 GI |
| 局外成长资源 | 写入 SaveGame（永久持久化）|

### 2.3 HP 回复机制

| 来源 | 说明 | 实现方式 |
|---|---|---|
| 关卡结算小量回血 | 每关结算固定回复少量 HP | 结算时 GameMode 对玩家施加回血 GE |
| 休息房 | 主要大量回血来源 | 房间类型 = Rest，进入时触发回血 |
| 商店 | 购买回血道具 | 金币消耗换 HP |
| 战斗符文 | 吸血 / 击杀回血 | 符文 Flow Graph 配置 |

### 2.4 局外成长资源

| 项目 | 设计 |
|---|---|
| 资源种类 | 多种：通用资源 × 1 + 专项资源 × N（种数待定，名称待定）|
| 专项资源分工 | 各自对应：角色强化 / 新武器 / 武器强化 / 天赋符文解锁 |
| 稀有度 | 由关卡决定（普通关 / 精英关 / Boss 关掉落不同稀有度）|
| 掉落方式 | 关卡结算时发放，在 DA_Room 中配置，不由敌人掉落 |
| 持久化 | 死亡后保留，写入 SaveGame |

### 2.5 分支预览信息

| 信息 | 显示规则 |
|---|---|
| 房间类型 | 直接标出（战斗 / 精英 / 商店 / 休息 / 事件 / Boss）|
| 敌人种类 | **不预告**，进去才知道 |
| 奖励 | 预告，符文显示**稀有度**（不显示具体效果名称）|
| 难度词条 | 可见 |

### 2.6 完美闪避

| 项目 | 设计 |
|---|---|
| 判定方式 | 闪避期间**无敌帧**内被攻击命中 = 完美闪避 |
| 基础效果 | 授予 `Action.Heat.CanPhaseUp`（已有接入点）|
| 扩展效果 | 全部来自符文（热度提升 / 升阶 / 时停等）|
| 技术实现 | 无敌帧期间检测是否有攻击尝试命中玩家碰撞体 |

### 2.7 Phase 3 满阶

| 层次 | 设计 |
|---|---|
| 视觉 | 角色光环特效 + UI 满阶状态标识 |
| 战斗 | 部分符文在满阶时解锁额外效果（符文配置层控制）|
| 动作 | 后期扩展：满阶专属动作集，改变战斗节奏（P3）|
| 机制 | 满阶是使用特定技能和终结技的前提条件 |

### 2.8 商店系统

| 商品类型 | 说明 |
|---|---|
| 符文购买 | 直接获得符文（随机 or 固定）|
| 符文净化 | 缩小符文占据的背包格形状，效果不变 |
| 被动赐福 | 类《杀戮尖塔》遗物，装备后全局被动生效 |
| HP 回复 | 购买回血（具体量待定）|
| 名称 / 价格体系 | 待定 |

### 2.9 武器系统

| 项目 | 设计 |
|---|---|
| 角色数量 | 1 个玩家角色 |
| 同时持有 | 1 把武器 |
| 影响范围 | 普攻动作 + 技能全部随武器改变 |
| 局内换武器 | **不可以** |
| 选择时机 | 进局前在局外大厅选择 |
| 局外解锁 | 新武器 + 武器强化（元进度解锁）|

## 五、关键决策记录

| 决策 | 原因 |
|---|---|
| 死亡 = 跑局全重置（参考《黑帝斯》）| 局内清零促使玩家规划路线，局外保留激励重玩 |
| 局外资源从关卡配置掉落，不从敌人掉落 | 降低配置成本，策划只需在 DA_Room 填写，不需要逐敌配置 |
| 分支不预告敌人种类 | 保持进入房间的探索感；固定敌人 + 可见词条已给玩家足够信息 |
| 符文奖励只显示稀有度（不显示具体效果）| 减少信息过载；玩家感受到"可能很好"的期待感 |
| 完美闪避效果全来自符文 | 避免基础机制奖励过重，保持符文构建的价值 |
| Phase 3 终结技作为机制门槛 | 推动玩家保持热度、维持进攻节奏，而非被动等待技能冷却 |
| 符文净化（缩小形状）作为商店服务 | 与背包格系统深度咬合，为高级玩家提供空间优化策略 |
| CampaignData 线性 FloorTable | 当前先跑通流程；P2 扩展为多路径结构时 FloorTable 可直接扩展 |
