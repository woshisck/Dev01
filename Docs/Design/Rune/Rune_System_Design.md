# 符文系统设计文档

> 版本：v0.1 | 日期：2026-04-23 | 阶段：框架设计

---

## 一、核心设计原则

### 原则一：行为改变优先于数值提升

好的符文改变的是「玩家做什么」，而不是「玩家数字变大」。

> ❌ 攻击力 +20%（数值符文）  
> ✅ 移动速度越快，击退力越强（行为符文——你开始主动保持高速）

**判断标准：** 玩家装备这个符文后，操作习惯有没有变化？没变化 = 数值符文；有变化 = 机制符文。

---

### 原则二：因果链可感知

每个符文效果都必须能被玩家用一句话描述。组合的意义是把因果链延长，而不是把数字叠加。

| 层次 | 示例 |
|---|---|
| 单符文 | 「我打它，它会流血」 |
| 双符文 | 「流血的敌人被打飞时，流血伤害会爆发」 |
| 三符文 | 「流血的敌人被打飞撞墙，撞击时爆炸并把流血传染给周围」 |

---

### 原则三：顺应战斗逻辑，而非改造它

游戏是类魂战斗——攻击帧、硬直、韧性、回避时机是核心骨架。符文负责填肉，不能动骨头。

**设计红线（以下方向不做）：**
- 要求玩家改变核心操作节奏才能触发的符文
- 要求玩家主动受伤来激活效果
- 需要特定输入序列才能激活
- 「必须站桩才能发挥」的被动

---

## 二、符文配置维度

符文效果由三个维度正交组合而成：

```
符文效果 = 触发源 × 作用对象 × 效果方式
```

---

### 2.1 触发源（什么激活符文）

**攻击动作类**
- 轻攻 / 重攻
- 连击链中第 N 击（第一击、终结击、特定节点）
- 命中 / 打空 / 暴击

**时机精准度类**
- 完美回避（i 帧内闪过攻击）
- 敌人出招前的窗口期内命中
- 连续命中不断链（节奏维持）

**位置与空间类**
- 贴身 / 中距 / 远距
- 从敌人背后攻击
- 多个敌人同时在攻击范围内
- 靠近墙壁 / 场景边缘

**玩家自身状态类**
- 正在移动 / 静止不动
- HP 高于 / 低于某阈值
- 冲刺后立即攻击

**敌人状态类**
- 敌人正处于被击硬直
- 敌人身上有特定状态（流血 / 燃烧等）
- 敌人正在被击退位移中

**击杀类**
- 击杀触发
- 击杀时的连击数 / 剩余 HP

---

### 2.2 攻击逻辑修改（符文直接改写攻击动作本身）

这类符文流派辨识度最高，是建立「风格差异」的核心手段。

**连招结构**

| 方向 | 效果 |
|---|---|
| 缩短连招 | 三击压缩为两击，每击更重，节奏更快 |
| 追加连招 | 在连招末尾解锁额外一击，该击带特殊属性 |
| 改变终结击位置 | 第二击提前成为终结击，每次终结后自动重置 |
| 解锁连招分支 | 特定节点后，重攻替换轻攻触发不同后续动作 |

**攻击属性**

| 方向 | 效果 |
|---|---|
| 范围改变 | 点状伤害 → 弧形 / 360° 范围 |
| 方向改变 | 正面攻击 → 侧面扩展 / 自动朝向最近敌人 |
| 距离改变 | 攻击自带小冲刺贴近 / 弹射型延伸距离 |

**攻击衍生**

| 方向 | 效果 |
|---|---|
| 冲刺融合 | 冲刺中攻击触发专属动作（非普通攻击衔接） |
| 追加动作 | 特定攻击命中后，短窗口内可追加一次动作 |
| 后摇取消 | 允许用冲刺取消攻击后摇（牺牲该次伤害） |

---

### 2.3 伤害方式（伤害的结算逻辑）

与触发源独立，可以自由组合。

**条件触发型**  
伤害挂在某个条件上，条件成立时才结算。条件本身就是设计重点——它决定玩家怎么操控这个状态。

| 状态名 | 触发条件 |
|---|---|
| 流血 | 目标移动时持续扣血 |
| 燃烧 | 目标攻击或被攻击时扣血 |
| 颤栗 | 目标靠近其他敌人时扣血 |

---

**积累叠加型**  
随某种积累增长，有明确的量变到质变节点。

| 状态名 | 逻辑 |
|---|---|
| 撕裂 | 连续命中时每次额外伤害递增，断链清零 |
| 淤伤 | 受击次数积累至阈值时爆发一次大伤害 |
| 脆化 | 每次命中降低目标防御，叠满后进入易碎状态 |

---

**比例伤害型**  
伤害数值基于某个动态数值的百分比。

| 状态名 | 基准 | 设计意图 |
|---|---|---|
| 中毒 | 目标最大 HP % | 血厚的敌人更疼，针对精英/Boss |
| 腐蚀 | 目标当前 HP % | 满血时最高效，鼓励开场施加 |
| 猎杀 | 目标缺失 HP % | 快死时补刀更狠，收割感强 |

---

**延迟结算型**  
伤害不立即生效，有「蓄积 → 释放」过程，存在策略空间。

| 状态名 | 逻辑 |
|---|---|
| 诅咒 | 施加后倒计时，到期集中结算所有积累伤害 |
| 引信 | 施加后等待外部触发（如被击退才爆炸） |
| 余震 | 击杀后延迟一秒，对周围溅射伤害 |

---

**传播扩散型**  
状态或伤害本身具有蔓延能力，能产生连锁视觉效果。

| 状态名 | 传播方式 |
|---|---|
| 感染 | 目标死亡时状态传播给周围敌人 |
| 碰撞传染 | 被击退目标撞到同伴时转移状态 |
| 链式 | 伤害在一定范围内弹射到下一目标 |

## 三、流派方向

### 3.1 四大流派

**物理引擎流**
- **核心幻想：** 把战场变成台球桌，敌人是球，玩家是球杆
- **行为转变：** 不再只盯血量，开始关注敌人位置、墙的距离、敌人密度
- **核心循环：** 攻击 → 击退 → 碰撞（墙 / 同伴）→ 链式反应 → 再击退
- **符文方向：** 击退力、碰撞伤害、弹射、连锁扩散

---

**动能流**
- **核心幻想：** 移动是武器，停下来就是浪费
- **行为转变：** 从站桩输出变为不断移动，冲刺成为主要攻击手段
- **核心循环：** 持续移动 → 积累动能 → 带速攻击 → 动能触发爆发 → 继续移动
- **符文方向：** 速度转伤害、冲刺攻击化、动能积累、急停冲击

---

**武道流**
- **核心幻想：** 每个动作都有时机，精准是力量，ACT 高手的主场
- **行为转变：** 从连打变为等待读怪，精准出手替代数量压制
- **核心循环：** 读取敌人动作 → 精准时机出击 / 回避 → 积累层数 → 满层爆发 → 重置
- **符文方向：** 时机窗口强化、节奏积累、完美回避反制、积累爆发

---

**炼金状态流**
- **核心幻想：** 先埋地雷，再踩地雷
- **行为转变：** 两阶段思维——先「种」状态，再「收」引爆
- **核心循环：** 施加状态 → 状态积累 / 扩散 → 触发引爆条件 → 连锁效果
- **符文方向：** 状态施加、延迟引爆、状态传播、状态升级转化

---

### 3.2 流派约束

每个流派必须有代价，否则玩家全堆所有方向，选择张力消失。

| 流派 | 代价与约束 |
|---|---|
| 物理引擎 | 开阔地带效果大幅削弱；需主动把敌人打向边缘 / 人群中 |
| 动能 | 静止时输出极低；高速移动中更容易被攻击打断 |
| 武道 | 被打断 / 打空时积累归零；冒险出击与保守等待存在取舍 |
| 炼金状态 | 种地阶段输出低；必须活到引爆阶段；引信用错时机状态浪费 |

---

## 四、转化符文体系

转化符文连接两个流派，创造单一流派无法实现的玩法。

**设计标准：1 + 1 必须大于 2，且创造全新策略路径。**

| 转化符文 | 连接流派 | 核心逻辑 |
|---|---|---|
| 横冲直撞 | 动能 × 物理引擎 | 玩家移速越高，击退力与碰撞伤害越高 |
| 流血撕裂 | 炼金状态 × 物理引擎 | 对流血敌人造成击退时，碰撞引爆流血状态 |
| 冲刺播种 | 动能 × 炼金状态 | 冲刺穿过敌人时自动附加状态 |
| 节奏引爆 | 武道 × 炼金状态 | 节奏积累满槽时，下次攻击引爆范围内所有状态 |
| 精准击飞 | 武道 × 物理引擎 | 精准时机出击时，击退力大幅提升 |

---

## 五、维度总览

```
符文效果 = 触发源 × 作用对象 × 效果方式

触发源
  ├─ 攻击动作（轻攻 / 重攻 / 连击节点 / 终结击）
  ├─ 时机精准度（完美回避 / 出招前窗口 / 节奏维持）
  ├─ 位置与空间（距离 / 背击 / 多敌 / 靠墙）
  ├─ 玩家状态（移动中 / HP 阈值 / 冲刺后）
  ├─ 敌人状态（硬直 / 有状态 / 位移中）
  ├─ 击杀（击杀触发 / 击杀时连击数）
  └─ 攻击逻辑修改（连招结构 / 攻击属性 / 衍生动作）

效果方式
  ├─ 条件触发型（满足条件时扣血）
  ├─ 积累叠加型（量变到质变）
  ├─ 比例伤害型（基于动态数值 %）
  ├─ 延迟结算型（蓄积后释放）
  └─ 传播扩散型（状态蔓延 / 链式弹射）
```
