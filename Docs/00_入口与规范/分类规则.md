# Docs 分类规则

> 更新：2026-06-06  
> 适用范围：`Docs/` 下所有手写文档。  
> 核心原则：按工作流分类，让文件夹名直接说明“这是需求、设计、配置、开发实现、完成记录还是归档”。

## 一级分类

| 目录 | 使用场景 | 判断问题 |
| --- | --- | --- |
| `00_入口与规范/` | 文档入口、写作规范、迁移记录 | 文档应该怎么写、怎么放 |
| `01_策划需求与版本方案/` | 版本目标、策划需求、需求拆解、开发方案、协作接口 | 这是要做什么，需求来自哪里 |
| `02_玩法设计与调研/` | 玩法规则、设计草案、调研、世界观、剧情源 | 为什么这么设计，设计依据是什么 |
| `03_策划配置与制作手册/` | DA、WBP、GA、Montage、Niagara、关卡摆放、编辑器工具 | 策划/TA 在编辑器里怎么配置 |
| `04_开发实现与系统文档/` | 系统架构、接口、源码入口、GameplayTag、编码规范 | 程序和 AI 按什么实现 |
| `05_完成记录与任务看板/` | TASKS、FeatureLog、验收报告、策划配置清单 | 做到哪了，哪些完成了，哪些待配置 |
| `06_AI协作与VibeCoding/` | AI 任务交接、代码规范、边界、验证要求 | AI 写代码前必须读什么 |
| `90_自动生成报告/` | Commandlet、脚本、工具自动生成的报告 | 工具运行结果是什么 |
| `98_废弃/` | 与当前方向冲突的文档 | 什么不能再作为依据 |
| `99_归档/` | 历史过程记录、旧需求、旧方案、旧 TODO | 历史背景是什么 |

## 放置决策

| 我要写的内容 | 默认位置 |
| --- | --- |
| 策划提出的新需求、版本目标、需求拆解 | `01_策划需求与版本方案/` |
| 程序/AI 要执行的开发方案、协作接口、联调验收 | `01_策划需求与版本方案/核心方案/` |
| 玩法规则、卡牌/符文设计表、系统设计草案 | `02_玩法设计与调研/核心玩法设计/` 或 `02_玩法设计与调研/设计文档/` |
| 调研和系统盘点 | `02_玩法设计与调研/调研总览/` |
| 策划配置说明 | `03_策划配置与制作手册/核心配置说明/<功能种类>/` |
| 编辑器工具使用说明 | `03_策划配置与制作手册/编辑器/` |
| 系统架构、接口、数据流 | `04_开发实现与系统文档/系统/<模块>/` |
| 编码规范、DataAsset/GAS/Widget/Material 约定 | `04_开发实现与系统文档/编码规范/` |
| GameplayTag 或状态冲突规则 | `04_开发实现与系统文档/标签/` |
| 当前任务、完成功能、验收管理 | `05_完成记录与任务看板/` |
| 已完成但需要策划继续配置的事项 | `05_完成记录与任务看板/策划配置清单.md` |
| 临时审查、过程计划、旧 TODO、旧方案 | `99_归档/` |
| 明确与当前方向冲突的内容 | `98_废弃/` |

## 任务状态字段

新增任务、需求或功能记录时，至少写清四个字段：

| 字段 | 说明 |
| --- | --- |
| 需求来源 | 策划、玩家反馈、商业计划书、版本目标、技术债 |
| 程序状态 | 未开始 / 开发中 / C++ 完成 / 已编译 / 已接入 |
| 配置状态 | 无需配置 / 待 DA / 待 WBP / 待 GA / 待 Montage / 待关卡摆放 |
| 验收方式 | 编译 / 自动化 / PIE / 包体验证 / 策划体验确认 |

## 核心方案内部规则

`01_策划需求与版本方案/核心方案/` 按功能分类：

| 目录 | 内容 |
| --- | --- |
| `00_总览与规范/` | 核心方案总览、当前上下文 |
| `01_开发方案/<功能>/` | 各功能的执行方案和后续方案 |
| `02_联调验收/` | 联调、验收、跨系统测试方案 |
| `03_协作接口/` | 模块间接口契约 |
| `04_系统需求/<功能>/` | 更细的需求拆分、运行规则、数据结构 |
| `05_开发路线/` | 按优先级组织的执行路线和配置清单 |

## 配置说明最低标准

配置说明可以短，但必须回答五个问题：

1. 这个配置影响什么。
2. 在编辑器哪里配置。
3. 字段怎么填。
4. 推荐示例是什么。
5. 怎么在 PIE 或关卡中测试。

如果涉及 WBP、DA、DataTable、材质 Custom Node、Niagara，应按对象拆文档，不把多种配置混在一篇里。

## 命名规则

| 类型 | 推荐命名 |
| --- | --- |
| 使用/配置指南 | `功能名_Guide.md` 或 `功能名配置说明.md` |
| 技术文档 | `功能名_Technical.md` |
| 设计说明 | `功能名_Design.md` 或中文规则名 |
| 参考手册 | `功能名_Reference.md` |
| 工作记录 | `功能名_WorkReport_YYYYMMDD.md` |

## 归档规则

以下内容放入 `99_归档/`：

- 已被主文档吸收的旧 TODO。
- 临时过程记录、计划评审、代码审查记录。
- 已经不作为当前方向的旧方案，例如旧 2D 背包方案。
- 只用于追溯历史、不应作为最新配置依据的文档。

明确与当前方向冲突的内容放入 `98_废弃/`。归档和废弃都不等于删除，但引用处必须说明状态。
