## 维护规则

1. 新增文档先看 [分类规则](00_入口与规范/分类规则.md)。
2. 移动文档必须同步更新 [路径迁移表](00_入口与规范/路径迁移表.md) 或在变更说明中记录。
3. 配置说明必须面向策划，写清楚位置、字段、示例和测试方式。
4. 长期系统文档应优先放到 `01_长期系统文档/系统/<模块>/`。
5. 不再使用的方案移入 `99_归档/旧方案/`，不要留在核心入口里。
