## 维护规则

1. 新需求先进入 `01_策划需求与版本方案/`，不要直接塞进系统文档。
2. 设计推演、调研、世界观和玩法草案进入 `02_玩法设计与调研/`。
3. 需要策划/TA 在编辑器里操作的内容进入 `03_策划配置与制作手册/`。
4. 已经稳定的技术实现、接口和代码规范进入 `04_开发实现与系统文档/`。
5. 任务状态、完成功能、验收记录进入 `05_完成记录与任务看板/`。
6. `99_归档/` 仅用于历史追溯；`98_废弃/` 不得作为当前实现依据。
