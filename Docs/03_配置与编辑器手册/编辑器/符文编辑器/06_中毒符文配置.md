# 中毒符文配置指南

---

## 一、定位

中毒符文命中目标后附加 **可叠层的中毒 GE**，通过 SetByCaller 控制叠层数。  
连招越深，附加的层数越多；最大层数在 GE 层级设置上限。

---

## 二、右侧基本信息

| 字段 | 值 |
|---|---|
| 类型 | `Debuff` |
| 触发方式 | `OnAttackHit` 或 `Passive + OnDamageDealt` |
| 流程资产 | 关联对应 FA |

---

## 三、数值表配置

| Key | 显示名 | 值 | 分类 | 连招模式 | 每段奖励 | 上限 | 取整 |
|---|---|---|---|---|---|---|---|
| `Poison.Stack.01` | 中毒层数 | 3 | Stack | 加算 | 1 | 4 | Floor |
| `Poison.Duration.01` | 中毒持续时间 | 6 | Duration | 无 | — | — | — |

**连招加成计算示例（Poison.Stack.01）：**

| 连招段 | 层数 |
|---|---|
| 第1段 | 3 层 |
| 第2段 | 4 层 |
| 第3段 | 5 层 |
| 封顶 | 7 层 |

> 取整模式设为 Floor，防止浮点误差导致层数出现小数。

---

## 四、流程图配置

```
Entry → 触发伤害事件(OnDamageDealt)
              Out → 施加状态(GE_Poison)
                       Target           = LastDamageTarget
                       SetByCallerTag   = Data.Poison.Stack
                       SetByCallerValue ← [读取数值 Pure] TuningKey = "Poison.Stack.01"
                  → 播放特效(中毒命中 VFX)
```

若需要通过持续时间 Key 控制 GE 时长：

```
施加状态(GE_Poison)
  SetByCallerTag2   = Data.Poison.Duration
  SetByCallerValue2 ← [读取数值 Pure] TuningKey = "Poison.Duration.01"
```

---

## 五、GE_Poison 配置要点

| GE 字段 | 建议值 |
|---|---|
| Duration Policy | `Has Duration` |
| Duration Magnitude | `SetByCaller (Data.Poison.Duration)` 或固定值 6 秒 |
| Period | 1.0（每秒跳伤） |
| Modifier Attribute | `BaseAttributeSet.Health` |
| Modifier Op | Additive |
| Magnitude | `SetByCaller (Data.Poison.Stack)` × 每层伤害系数（在 GE 内计算） |
| Stack Policy | `Aggregate by Source` |
| Stack Limit Count | 7（最大层数上限，与数值表封顶一致） |
| Asset Tags | `Buff.Status.Poison` |

---

## 六、多 Index 使用场景

若同一符文有两种中毒形态（如普通命中 vs 连携命中）：

| Key | 用途 |
|---|---|
| `Poison.Stack.01` | 普通命中时附加的基础层数 |
| `Poison.Stack.02` | 连携激活时额外附加的层数（在 LinkFlow FA 中引用） |

FA 内通过条件节点分支，分别引用 `.01` 或 `.02`。

---

## 七、注意事项

- `Stack Limit Count` 建议在 GE 层级设硬上限，防止设计失误导致叠无限层
- 连招加成的**封顶（MaxBonus）必须填写**，否则连招足够深时层数无上限
- `Poison.Stack` 的取整模式必须设 `Floor`，层数是整数
- 若持续时间不需要随连招变化，`Poison.Duration` 的连招模式设为`无`即可
- 蚀月连携（Poison→Moon）的附加中毒使用独立的 `GE_Moonlight_Poison`，不复用此 GE
