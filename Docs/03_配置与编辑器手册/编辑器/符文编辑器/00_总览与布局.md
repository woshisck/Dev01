# 符文编辑器 总览与布局

> 打开路径：`工具栏 → Tools > DevKit Data > Rune Editor`  
> 最后更新：2026-05-09

---

## 一、窗口布局

```
┌─────────────────────────────────────────────────────────────────────┐
│  顶部工具条                                                           │
│  [运行符文] [打开符文] [打开流程] [刷新]                               │
├──────────────┬────────────────────────────────────────┬─────────────┤
│              │  中间面板（Tab 切换）                    │  右侧详情   │
│  左侧        │  ┌─ 数值表 ─┬─ 流程图 ─┬─ 连携配方 ─┬─ 模块配置 ─┐  │  ┌─ 基本信息 ┐│
│  资源管理     │  └──────────┴─────────┴────────────┴────────────┘  │  └─ 战斗卡  ┘│
│              │  底部面板（Tab 切换）                    │             │
│  ▸ 分类筛选  │  节点库 / 校验 / 运行日志 / 选中节点      │             │
└──────────────┴────────────────────────────────────────┴─────────────┘
```

## 二、各区域说明

### 2.1 左侧资源管理

- **分类筛选**：按类型（基础 / 敌人 / 关卡 / 终结技 / 战斗卡）筛选符文列表
- **搜索框**：按名称实时过滤
- **符文列表**：点击选中一个符文，所有面板同步显示其数据
- **操作按钮**：新建 / 复制 / 粘贴 / 重命名 / 删除 / 在内容浏览器定位

### 2.2 中间面板 — Tab 切换

| Tab | 功能 |
|---|---|
| **数值表** | 配置所有调参 Key，包括连招加成 |
| **流程图** | 可视化编辑 FlowAsset（FA）节点图 |
| **连携配方** | 配置当前符文与相邻战斗卡的连携关系 |
| **模块配置** | 模块化参数（投射物/光环模块），通过 GetProjectileModule / GetAuraModule 节点读取 |

### 2.3 底部面板 — Tab 切换

| Tab | 功能 |
|---|---|
| **节点库** | 按分类（技能/效果/生成/条件/表现/生命周期）搜索节点并一键添加到流程图 |
| **校验** | 检测 Flow 连接错误、节点配置异常 |
| **运行日志** | 点击"运行符文"后显示执行输出，用于 Debug |
| **选中节点** | 显示流程图中当前选中节点的属性 |

### 2.4 右侧详情面板 — Tab 切换

| Tab | 功能 |
|---|---|
| **基本信息** | 名称 / Tag / 摘要 / 类型 / 稀有度 / 触发方式 / 关联 Flow 资产 |
| **战斗卡** | 启用战斗卡、填写 Card ID Tag / 效果 Tags / 卡类型 / 连招缩放参数 |

## 三、创建符文步骤

1. 在左侧面板左上角填写 **名称**、**Rune Tag**（格式 `Rune.ID.xxx`）、**文件夹**
2. 选择 **类型**（Buff / Debuff / None）、**稀有度**、**触发方式**
3. 点击 **新建符文** → 编辑器在目标文件夹自动创建 `DA_Rune_xxx`
4. 在右侧 **基本信息 → 流程资产** 关联一个已有 `UYogRuneFlowAsset`（或从模板复制）

---

## 四、通用工作流

```
新建符文
  │
  ├─→ 右侧基本信息：填写元数据 + 关联 FA
  │
  ├─→ 中间数值表：添加所有调参 Key（可用"插入模板"快速填入预设）
  │
  ├─→ 中间流程图：连节点，描述效果逻辑
  │        └─ 底部节点库：搜索节点，点击"添加"插入
  │
  ├─→ [可选] 右侧战斗卡：填写卡信息，启用连招缩放
  │
  ├─→ [可选] 中间连携配方：填写邻居 Tag + 倍率 + LinkFlow
  │        └─ 点击 → 按钮在流程图面板中直接打开对应连携 FA
  │
  └─→ 底部校验 → 运行调试 → 保存资产
```

---

## 五、注意事项

- **保存**：数值表和基本信息修改后需点击对应的"保存"按钮才会写入 `.uasset`，流程图修改会即时标脏
- **重命名**：左侧列表中选中后点击"重命名"，不要在内容浏览器手动重命名（会丢失引用）
- **Flow 资产**：每个符文通常需要独立的 `YogRuneFlowAsset`；连携 LinkFlow 可以多个符文共用
- **Debug**：运行日志面板中节点执行时会打印来源、数值、GUID 等关键字段，开发阶段保持 LogTemp 可见
