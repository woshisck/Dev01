# 刷怪难度分配置说明

面向：策划 / 关卡设计  
适用范围：512 普通房间、精英房间、小房间 / 大房间敌人上限

## 1. DA_Room 难度档位

在 `DA_Room` 的 `LowDifficulty`、`MediumDifficulty`、`HighDifficulty` 中配置：

| 字段 | 用法 | 推荐 |
|---|---|---|
| `MaxWaveCount` | 本档位最多波次数 | 普通 2-3；精英 3-4 |
| `MaxEnemiesPerWave` | 单波同屏上限，0 为不限制 | 普通 6-8；精英 4-6 |
| `MaxEnemiesPerLevel` | 整关计划刷怪总上限，0 为不限制 | 小关卡 10；大关卡 20 |
| `BuffCount` | 从房间 Buff 池抽取几个房间 Buff | 普通 0-1；精英 1 |

`MaxEnemiesPerWave` 控制同屏压力，`MaxEnemiesPerLevel` 控制整关总量。两者可以同时配置。

## 2. Enemy Pool

在 `DA_Room.EnemyPool` 中配置：

| 字段 | 用法 | 推荐 |
|---|---|---|
| `EnemyData` | 候选敌人 DA | 老鼠、死亡守卫 |
| `MaxCountPerLevel` | 该敌人在整关最多出现几只，-1 为不限制 | 老鼠 8-12；死亡守卫 1-3 |

程序会优先选择 `DifficultyScore` 高的敌人；预算不足或类型上限已满时才回退到低分敌人。

## 3. 难度分来源

整关总预算来自 `DA_Campaign.FloorTable.TotalDifficultyScore`。程序按总分选择 `DA_Room` 的 Low / Medium / High 档位，再按波次拆分预算。

单只敌人的实际消耗：

```text
EnemyData.DifficultyScore
+ 当前房间已选 Room Buff 的 DifficultyScore
+ 敌人 EnemyBuffPool 中本次通过 ApplyChance 的 DifficultyScore
```

`EnemyBuffPool.ApplyChance` 未触发时，不会扣对应 Buff 分。

## 4. 配置示例

小普通房间：

| 配置 | 值 |
|---|---:|
| `TotalDifficultyScore` | 24-32 |
| `MaxWaveCount` | 2 |
| `MaxEnemiesPerWave` | 6 |
| `MaxEnemiesPerLevel` | 10 |
| 老鼠 `MaxCountPerLevel` | 10 |
| 死亡守卫 `MaxCountPerLevel` | 1 |

大精英房间：

| 配置 | 值 |
|---|---:|
| `TotalDifficultyScore` | 50-70 |
| `MaxWaveCount` | 3 |
| `MaxEnemiesPerWave` | 8 |
| `MaxEnemiesPerLevel` | 20 |
| 老鼠 `MaxCountPerLevel` | 12 |
| 死亡守卫 `MaxCountPerLevel` | 3 |

如果房间节奏过慢，优先降低 `MaxEnemiesPerLevel`；如果同屏太乱，优先降低 `MaxEnemiesPerWave`。
