# 配置文档编写规范

## 目的

后续任何功能开发，只要涉及策划配置、数值配置、DA 配置、WBP 配置、DataTable 配置、奖励池配置或引擎内字段配置，都必须同步提供配置说明文档。

配置文档的目标是让策划知道：

- 这个功能为什么要配。
- 在哪里配置。
- 每个关键字段怎么填。
- 推荐默认值是什么。
- 如何验证配置是否生效。
- 哪些字段当前不要使用或容易误用。

## 放置规则

配置文档统一放在：

`Docs/03_配置与编辑器手册/核心方案配置说明/`

先按功能种类建立文件夹，再按配置对象拆分文档。

示例结构：

```text
配置说明/
  符文/
    符文DA配置说明.md
    符文WBP配置说明.md
    符文奖励池配置说明.md
  武器/
    武器DA配置说明.md
    武器连招配置说明.md
  战斗/
    HitStop配置说明.md
    打击反馈配置说明.md
  UI/
    HUD配置说明.md
```

## 拆分规则

同一功能如果涉及多个配置对象，需要拆成多份文档。

| 配置对象 | 文档建议 |
| --- | --- |
| DataAsset | 单独写 `XXXDA配置说明.md` |
| Widget Blueprint | 单独写 `XXXWBP配置说明.md` |
| DataTable | 单独写 `XXX表格配置说明.md` |
| 奖励池 / 生成池 | 单独写 `XXX奖励池配置说明.md` 或 `XXX生成池配置说明.md` |
| 多个配置对象协作 | 可额外写一份 `XXX配置总览.md` |

示例：

- 符文功能如果同时涉及 `RuneDataAsset` 和 WBP，就放在 `配置说明/符文/` 下，拆成 `符文DA配置说明.md` 和 `符文WBP配置说明.md`。
- 打击感功能如果涉及 HitStop 数值和特效反馈，就放在 `配置说明/战斗/` 下，拆成 `HitStop配置说明.md` 和 `打击反馈配置说明.md`。

## 文档内容要求

每份配置文档尽量短，优先使用以下结构：

```text
# 功能名配置说明

## 作用
说明这个配置解决什么问题。

## 配置位置
写清楚在哪个 DA、WBP、DataTable 或引擎面板中配置。

## 字段说明
用表格说明字段怎么填。

## 推荐配置
给出当前版本建议默认值。

## 配置示例
给一个可直接照着配的例子。

## 验收方式
列出 3-6 条检查步骤。

## 注意事项
说明当前不要配置、容易误用、后续会变更的点。
```

## 写作要求

- 面向策划配置，不写无关代码实现细节。
- 字段名必须和引擎中显示或代码中的字段名一致。
- 推荐值必须明确，不写“看情况”。
- 如果功能还没完全开发完，要标明“当前可配置”和“后续待开发”。
- 如果某个字段当前只是兼容旧逻辑，要明确标记为旧字段或暂不推荐。
- 文档应尽量短，能指导配置即可。

## 开发交付要求

每次功能开发完成后，交付内容必须包含：

1. 代码或资源改动说明。
2. 配置文档位置。
3. 策划需要配置的最小清单。
4. 验收方式。

如果需求开始时没有指定配置文档位置，需要先确认文档分类和放置目录；未指定时，默认放到对应功能种类的 `配置说明/<功能种类>/` 下。
