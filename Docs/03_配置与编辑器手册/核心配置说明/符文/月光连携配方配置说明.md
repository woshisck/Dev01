# 月光连携配方配置说明

## 作用

配置月光 Link 卡的正向/反向配方。背包翻转 UI 未完成前，使用两张测试卡验证方向：

- `DA_Rune512_Moonlight_Forward`
- `DA_Rune512_Moonlight_Reversed`

## 基础字段

打开月光 DA：
`RuneInfo -> CombatCard`

| 字段 | 值 |
| --- | --- |
| `CardType` | `Link` |
| `CardIdTag` | `Card.ID.Moonlight` |
| `CardEffectTags` | `Card.Effect.Moonlight` |
| `BaseFlow` | `FA_Rune512_Moonlight_Base` |
| `DefaultLinkOrientation` | Forward 测试卡填 `Forward`；Reversed 测试卡填 `Reversed` |

## 正向配方

`Direction=Forward`：月光读取上一张卡。

| RequiredNeighborEffectTags | LinkFlow |
| --- | --- |
| `Card.Effect.Burn` | `FA_Rune512_Moonlight_Forward_Burn` |
| `Card.Effect.Poison` | `FA_Rune512_Moonlight_Forward_Poison` |
| `Card.Effect.Shield` | `FA_Rune512_Moonlight_Forward_Shield` |
| `Card.Effect.Pierce` | `FA_Rune512_Moonlight_Forward_Pierce` |
| `Card.Effect.Attack` | `FA_Rune512_Moonlight_Forward_Attack` |
| `Card.Effect.Defense.ReduceDamage` | `FA_Rune512_Moonlight_Forward_ReduceDamage` |

## 反向配方

`Direction=Reversed`：月光先进入 pending，强化下一张卡。

| RequiredNeighborEffectTags | LinkFlow |
| --- | --- |
| `Card.Effect.Burn` | `FA_Rune512_Moonlight_Reversed_Burn` |
| `Card.Effect.Poison` | `FA_Rune512_Moonlight_Reversed_Poison` |
| `Card.Effect.SplashSplit` | `FA_Rune512_Moonlight_Reversed_Split` |
| `Card.Effect.Shield` | `FA_Rune512_Moonlight_Reversed_Shield` |
| `Card.Effect.Pierce` | `FA_Rune512_Moonlight_Reversed_Pierce` |
| `Card.Effect.Attack` | `FA_Rune512_Moonlight_Reversed_Attack` |
| `Card.Effect.Defense.ReduceDamage` | `FA_Rune512_Moonlight_Reversed_ReduceDamage` |

## 配置注意

- `RequiredNeighborEffectTags` 读的是邻接卡的 `CardEffectTags`，不要填旧字段 `CardTags`。
- `LinkRecipes` 不要配 Link 读 Link；运行时已阻止 Link 卡互相触发。
- 两张月光测试卡可以共用同一组配方，区别只在 `DefaultLinkOrientation`。
- 复杂效果先在 `LinkFlow` 内手动检查节点，模板副本只是起点，不代表最终玩法数值。

## 验收

1. 普通卡后接 `Moonlight_Forward`，能触发对应 Forward 配方。
2. `Moonlight_Reversed` 后接普通卡，能触发对应 Reversed 配方。
3. 洗牌后月光配方仍按当前顺序和方向结算。
