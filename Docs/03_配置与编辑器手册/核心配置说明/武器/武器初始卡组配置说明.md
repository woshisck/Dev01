# 武器初始卡组配置说明

## 作用
武器决定玩家开局进入战斗时携带哪些战斗卡牌，以及打空卡组后的洗牌时间。

## 配置位置
打开目标 `WeaponDefinition`，找到 Combat Deck 相关字段：

- `InitialCombatDeck`
- `ShuffleCooldownDuration`
- `MaxActiveSequenceSize`

如果 `InitialCombatDeck` 为空，系统会从旧字段 `InitialRunes` 中筛选已开启 Combat Card 的符文作为兼容回退。

## 字段说明
| 字段 | 怎么填 | 影响 |
| --- | --- | --- |
| `InitialCombatDeck` | 按顺序放入 `RuneDataAsset` | 玩家初始卡组顺序，战斗中固定按这个顺序消耗 |
| `ShuffleCooldownDuration` | 推荐 `1.0` | 卡组打空后多久回填 |
| `MaxActiveSequenceSize` | `0` 表示全部进入激活序列 | 大于 0 时，只取前 N 张进入一轮卡组 |

## 推荐配置
V1 推荐：

- `ShuffleCooldownDuration = 1.0`
- `MaxActiveSequenceSize = 0`
- `InitialCombatDeck` 放 4 到 8 张已开启 Combat Card 的符文

## 注意事项
- `InitialCombatDeck` 的顺序就是战斗消耗顺序，不会随机洗牌。
- `InitialCombatDeck` 中没有开启 `bIsCombatCard` 的符文会被忽略。
- 不要依赖热度控制卡牌是否生效，512 V1 中卡牌不以热度作为门槛。
- 地图里通过 `WeaponSpawner` 拾取武器时，系统会自动读取该武器 DA 的 `InitialCombatDeck`。
- 靠近武器时，武器浮窗也会显示同一份 `InitialCombatDeck` 初始卡牌；该浮窗由 HUD 加入视口，位置来自武器世界点的屏幕投影，并会自动夹在屏幕内。
- 武器浮窗显示或淡出期间会隐藏关卡/Portal 预览浮窗，避免两个信息浮窗重叠。
- 武器浮窗中的初始卡牌只显示图标、名称和简短摘要；摘要优先读取卡牌 DA 的 `CombatCard -> HUDSummaryText`，建议控制在 1-2 行。
- 如果 HUD 没显示卡牌，先确认拾取的是配置过 `InitialCombatDeck` 的同一个 `WeaponDefinition`，不是另一个展示/测试武器 DA。

## 排查方式

| 现象 | 优先检查 |
| --- | --- |
| 拾取武器后 HUD 没有卡牌 | `WeaponSpawner -> WeaponDefinition` 是否指向你配置过的武器 DA |
| HUD 有卡槽但为空 | `InitialCombatDeck` 里的符文是否勾选 `RuneInfo -> CombatCard -> Is Combat Card` |
| 武器浮窗显示“未配置初始卡牌” | `InitialCombatDeck` 为空且 `InitialRunes` 中也没有开启 Combat Card 的符文 |
| 武器浮窗位置不在武器旁边 | 确认 `WeaponSpawner` 正在调用 `ShowWeaponFloatInfoAtLocation`，并检查 HUD 的 `WeaponFloatViewportSize` / `WeaponFloatScreenOffset` |
| 拾取后卡牌顺序不对 | `InitialCombatDeck` 数组顺序是否正确；该字段非空时不会读取 `InitialRunes` |
| 攻击后没有触发效果 | 卡牌 DA 的 `Trigger Timing` 是否和攻击触发时机一致，普通攻击卡建议先用 `On Commit` |

## 测试方式
1. 给玩家当前武器配置 `InitialCombatDeck`。
2. 进入战斗房间，确认 HUD 显示卡牌顺序与数组顺序一致。
3. 连续命中消耗到最后一张，确认进入洗牌。
4. 等待 `ShuffleCooldownDuration` 后，确认卡组按固定顺序回填。
