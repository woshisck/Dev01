# 旧热度/背包/符文临时断开说明

## 1. 当前状态

本次为了优先测试 Combat Card / 符文卡牌逻辑，旧系统入口已临时断开：

| 系统 | 当前状态 |
| --- | --- |
| 热度 Heat | Heat 属性 GE 修改被阻断，热度不再升阶 |
| 武器热度表现 | 武器不再绑定 `OnHeatPhaseChanged`，不再根据热度切换 Overlay |
| 背包激活区 | 武器不再注入 `BackpackConfig`，背包不再根据热度刷新激活 |
| 武器旧符文 | 武器 `InitialRunes` 不再自动放入背包 |
| 背包旧符文 | `PermanentRunes`、`DebugTestRunes`、`HiddenPassiveRunes` 不再自动激活 |
| 旧命中符文 | `ReceiveOnHitRune` 不再启动旧符文 FA |
| Combat Card | 保持启用，武器仍会读取 `InitialCombatDeck` 和 `WeaponComboConfig` |

## 2. 当前测试应该配置什么

当前只需要配置武器 DA 上的：

| 字段 | 用途 |
| --- | --- |
| `InitialCombatDeck` | 武器获得后加载的战斗卡牌列表 |
| `WeaponComboConfig` | 武器连招配置 |

暂时不要依赖以下旧字段验收效果：

| 字段 | 原因 |
| --- | --- |
| `BackpackConfig` | 已临时不注入 |
| `InitialRunes` | 已临时不自动放入背包 |
| 旧 Rune DA 的 `FlowAsset` | 背包旧激活入口已临时关闭 |

## 3. 之后恢复旧系统的方法

代码中搜索以下临时开关，将值从 `true` 改成 `false` 后重新编译：

| 文件 | 开关 |
| --- | --- |
| `WeaponDefinition.cpp` | `bDisableLegacyHeatBackpackRuneForCardTest` |
| `WeaponSpawner.cpp` | `bDisableLegacyHeatBackpackRuneForCardTestSpawner` |
| `BackpackGridComponent.cpp` | `bDisableLegacyBackpackRuneRuntimeForCardTest` |
| `YogCharacterBase.cpp` | `bDisableLegacyOnHitRuneRuntimeForCardTest` |
| `BaseAttributeSet.cpp` | `bDisableLegacyHeatRuntimeForCardTest` |

恢复前需要确认旧热度、旧背包符文、Combat Card 三套系统的触发边界是否重新设计完成。
