# 火绳枪系统 Blueprint 配置清单

> 版本：v1.0 | 日期：2026-04-19
> 前置条件：C++ 已编译通过（DevKitEditor Win64 Development）

---

## 概览

所有游戏逻辑已在 C++ 中实现。Blueprint 只需在 **Class Defaults** 中填写资产引用和数值参数，**不写任何蓝图逻辑节点**。

创建顺序建议：
1. 先创建 `GE_MusketBullet_Damage`
2. 创建 `BP_AimArcActor` + `M_AimArc` 材质
3. 创建 `BP_MusketBullet`
4. 创建 6 个 `BP_GA_Musket_*`
5. 创建 `DA_Weapon_Musket`
6. 创建 `WBP_AmmoCounter`

## 2. M_AimArc（Decal 材质）

**创建：** Content Browser → 右键 → Material → 命名 `M_AimArc`

### Material 属性

| 属性 | 值 |
|---|---|
| Material Domain | Deferred Decal |
| Blend Mode | Translucent |
| Decal Blend Mode | Translucent |

### 节点连接（在材质编辑器中手动搭建）

```
步骤 1：建立 UV 坐标
  [TexCoord(0)] → 减 0.5 → UVCentered（Vector2 型，范围 [-0.5, 0.5]）

步骤 2：计算角度 Mask
  Atan2(UVCentered.X, UVCentered.Y) → Angle（弧度）
  [ScalarParameter: "HalfAngle"（默认 45.0，单位度）] → 乘 (π/180 = 0.01745) → HalfRad
  Abs(Angle) < HalfRad → InAngle（Bool）

步骤 3：计算半径 Mask
  Length(UVCentered) < 0.5 → InRadius（Bool，UV 归一化，0.5 = Decal 边界）

步骤 4：合并 Mask
  InAngle AND InRadius → Mask
  Mask × SmoothStep(0, 0.05, Mask) → SoftMask（边缘柔化）

步骤 5：输出
  SoftMask → Opacity（连接到 Opacity 输出引脚）
  [VectorParameter: "Color"（默认 RGBA(1,0.55,0,1)）] × SoftMask → Emissive Color
```

**参数名必须与 C++ 代码中常量一致（区分大小写）：**
- `HalfAngle`（标量 Scalar Parameter）
- `ArcRadius`（标量，可选，供径向渐变使用）
- `Color`（向量 Vector Parameter）

## 4. BP_MusketBullet

**创建：** Content Browser → 右键 → Blueprint Class → 父类搜索 `MusketBullet` → 命名 `BP_MusketBullet`

### Class Defaults 填写

| 字段 | 值 |
|---|---|
| `Speed` | `2800`（cm/s） |
| `Lifetime` | `0.8`（秒，超时自动销毁） |

### Blueprint 事件（唯一需要写蓝图逻辑的地方）

在事件图谱中实现两个 Implementable Event：

**`BP_OnHitEnemy`**（命中敌人时由 C++ 调用）
- 参数：`HitActor`（AActor*）、`HitLocation`（FVector）
- 建议逻辑：
  1. 在 HitLocation 生成命中 Niagara 特效（`Spawn System At Location`）
  2. 播放命中音效（`Play Sound at Location`）
  3. （可选）对 HitActor 施加击退 GE

**`BP_OnMiss`**（子弹超时消失时由 C++ 调用）
- 无参数
- 建议逻辑：生成消散 Niagara 特效

## 12. WBP_AmmoCounter（弹药 HUD 组件）

**创建：** Content Browser → 右键 → User Interface → Widget Blueprint → 命名 `WBP_AmmoCounter`

### 布局建议

```
[Canvas Panel]
  └── [Horizontal Box]（水平排列子弹图标）
       ├── [Image] × MaxAmmo 个（每颗一个图标）
       └── [Text] （可选，显示 "当前/最大"）
```

### 数据绑定

在 Graph 中绑定属性集：
1. 获取 `OwningPlayerController → GetPlayerState → GetAbilitySystemComponent`
2. 监听 `PlayerAttributeSet.CurrentAmmo` 属性变化（`BindToAttributeValueChange`）
3. 根据 `CurrentAmmo` 显示/隐藏图标（亮 = 有弹药，暗/灰 = 无弹药）

属性名称（精确）：
- `PlayerAttributeSet.CurrentAmmo`（当前弹药，范围 0–6）
- `PlayerAttributeSet.MaxAmmo`（最大弹药，默认 6）

## 13. Mesh 骨骼 Socket 配置

### 枪口 Socket（必须）

在武器 Skeletal Mesh 编辑器中：
1. 打开武器 Mesh（火绳枪 Skeletal Mesh）
2. 右键某个骨骼（枪管末端骨骼） → Add Socket
3. 命名：`Muzzle`
4. 调整位置使其对准枪口位置（Z 轴朝前）
5. 保存

> **位置优先级（C++ 自动查找）：**
> 1. 角色 Mesh 上的 `Muzzle` Socket
> 2. 装备武器 Mesh 上的 `Muzzle` Socket
> 3. 近似位置（角色位置 + 前方 80cm + 上方 60cm）

## 14. AnimMontage 配置要点

每个蒙太奇需要设置的字段：

| 蒙太奇 | Slot | 时长参考 | 备注 |
|---|---|---|---|
| 速射蒙太奇 | DefaultSlot 或 上半身Slot | 0.4–0.6s | 若用 FireEventTag，需在动画内添加 AnimNotify |
| 重攻击开枪蒙太奇 | 同上 | 0.5–0.8s | 松手即发射，蒙太奇仅视觉表现 |
| 单发上弹蒙太奇 | 同上 | 0.7–1.0s | C++ 递归调用，直到弹满 |
| 全换弹蒙太奇 | 同上 | 1.5–2.0s | |
| 冲刺攻击蒙太奇 | 同上 | 0.5–0.8s | |

### 速射 FireEventTag（可选配置）

若 `FireEventTag` 不为空（如 `GameplayEvent.Musket.FirePoint`）：
1. 打开速射蒙太奇
2. 在时间轴上找到开枪时机帧
3. 添加 `AnimNotify_GameplayEvent`
4. 设置 Tag = `GameplayEvent.Musket.FirePoint`（与 `FireEventTag` 字段一致）

若 `FireEventTag` 留空，激活 GA 时立即开枪，不等蒙太奇。

## 15. 测试验证清单

创建完所有资产后，进入 Play 模式逐项验证：

- [ ] **弹药显示**：HUD 显示 6/6 弹药图标
- [ ] **速射**：按攻击键，开枪，弹药减 1，音效播放
- [ ] **重攻击蓄力**：长按，地面显示扇形弧，弧随蓄力时间收窄变细长
- [ ] **重攻击满蓄**：满蓄后弧变黄色，提示音播放
- [ ] **重攻击松手**：松手后开枪，弧消失，伤害为满蓄倍率
- [ ] **单发上弹**：弹药不满时按换弹键，动画循环播放，每次加 1 发
- [ ] **取消上弹**：上弹中按攻击键，上弹停止，立即开枪
- [ ] **全换弹**：弹药不满时按全换弹键，动画播放一次后弹药满
- [ ] **弹药耗尽**：弹药为 0 时速射和重攻击无法激活
- [ ] **冲刺扇射**：冲刺中按攻击键，所有子弹扇形射出，弹药清空
- [ ] **冲刺换弹**：冲刺中按换弹键，冲刺继续，短暂延迟后弹药满

## 16. 常见问题

**Q: AimArcActor 扇形不显示？**
A: 检查 `M_AimArc` 材质 Domain 是否为 `Deferred Decal`，检查 `BP_AimArcActor.ArcMaterial` 是否已填写。

**Q: 子弹生成但不移动？**
A: 检查 `BP_MusketBullet.Speed` 是否填写（默认 2800），检查 `ProjectileMovement` 组件是否存在（C++ 父类已添加，不需要手动加）。

**Q: 换弹无法被攻击取消？**
A: 检查攻击 GA（LightAtk/HeavyAtk）的 Class Defaults → Gameplay Ability → Tags → Cancel Abilities With Tag 是否包含 `Ability.Musket.Reload`。（构造函数已设置，只要父类正确，无需手动配置。）

**Q: 冲刺攻击无法激活？**
A: 检查角色 ASC 上是否有 `Buff.Status.DashInvincible` Tag（冲刺 GA 的 ActivationOwnedTags 需含此 Tag）。

**Q: 重攻击蓄力弧不更新（弧一直不动）？**
A: C++ 已修复（使用 `UAbilityTask_MusketCharge` bTickingTask）。如仍无法更新，检查 `AimArcClass` 是否填写，`BP_AimArcActor.ArcMaterial` 是否正确。
