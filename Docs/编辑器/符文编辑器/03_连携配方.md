# 连携配方（Combo Recipe）使用说明

> 中间面板 → 切换到 **连携配方** Tab  
> 右侧详情 → 切换到 **战斗卡** Tab（需先启用战斗卡）

---

## 一、前置：启用战斗卡

连携配方只对**战斗卡类型**符文有效。使用前必须在右侧 **战斗卡** Tab 开启：

| 字段 | 说明 | 示例 |
|---|---|---|
| **启用战斗卡** | 勾选后此符文可参与连携系统 | ✅ |
| **Card ID Tag** | 精确匹配用，格式 `Card.ID.xxx` | `Card.ID.Moonlight` |
| **效果 Tags** | 效果匹配用，格式 `Card.Effect.xxx`，推荐用这个 | `Card.Effect.Moonlight` |
| **卡类型** | Attack / Defense / Utility / Finisher | `Attack` |
| **启用连招缩放** | 勾选后此卡伤害随 ComboIndex 整体缩放 | 按需 |
| **每段缩放系数** | 每段连招加的倍率增量 | `0.05` |
| **最大缩放倍率** | 连招缩放上限 | `0.30` |

---

## 二、连携方向说明

| 方向 | 含义 | 示例 |
|---|---|---|
| **正向（Forward）** | **邻居卡在前，本卡在后**（邻居→本卡） | Fire→Moon：月光的正向连携 |
| **反向（Backward）** | **本卡在前，邻居卡在后**（本卡→邻居） | Moon→Fire：月光的反向连携 |

> 月光配方里，`NeighborTag = RuneTag.Fire` 的一行同时管 **正向**（火在前）和 **反向**（月光在前）。

---

## 三、配方字段说明

每条配方配置以下字段：

| 字段 | 说明 |
|---|---|
| **邻居 Tag** | 相邻卡的 Effect Tag 或 ID Tag |
| **匹配方式** | `按效果Tag匹配`（推荐，一个邻居类型匹配所有同效果的卡）或 `精确ID匹配` |
| **正向 Flow** | 正向连携激活时额外运行的 FA（LinkFlow），留空则只有倍率效果 |
| **正向倍率** | 正向连携时对本次伤害乘以该系数 |
| **正向说明** | 显示在 HUD 上的连携名称（如"月焰"） |
| **反向 Flow** | 反向连携激活时额外运行的 FA，留空则只有倍率效果 |
| **反向倍率** | 反向连携时对本次伤害乘以该系数 |
| **反向说明** | 显示在 HUD 上的连携名称（如"月照"） |
| **→ 按钮** | 点击后在中间面板流程图区域打开对应的 LinkFlow FA，便于查看和调试 |

---

## 四、添加/删除配方行

- 点击 **添加连携行** 新增一条配方
- 每行末尾有红色删除按钮
- 配方全部填写完成后点击 **保存连携配方**

---

## 五、LinkFlow（连携 FA）说明

LinkFlow 是连携激活时额外运行的 FlowAsset，与主符文 FA 并行执行：

- LinkFlow 内部可以访问 `CombatDeckActionContext`（伤害/GUID/投射物类等）
- LinkFlow 内的节点通常需要关掉 `bRequireRangedWeaponTag`（连携不检查武器）
- LinkFlow 运行完毕后自动结束，不影响主 FA 生命周期

**点击 → 按钮**可以直接在流程图面板内加载该 LinkFlow，无需手动在 Content Browser 里查找。

---

## 六、注意事项

- 同一张卡可以有多条连携配方（多个不同的邻居类型）
- 正向和反向倍率是**叠加在连招倍率之上**的额外乘算，不是替代
- 若 LinkFlow 为空但倍率不为 1.0，仍然生效（纯倍率连携）
- 反向连携的 LinkFlow 留空，则反向连携只有倍率、无额外效果（月照/月毒的设计）
- 连携系统需要 `State.Combo.ChainActive` Tag 激活（冲刺会保留此 Tag，续连有效）
