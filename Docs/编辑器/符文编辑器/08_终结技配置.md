# 终结技符文配置指南

---

## 一、定位

终结技符文是**战斗卡中的收尾技**，通常在连招末端打出，具有高倍率 + 大范围 + 无法在连招中段重复触发的特性。  
典型形态：大范围 AOE 爆发伤害，或单目标极高单次伤害。

---

## 二、右侧基本信息

| 字段 | 值 |
|---|---|
| 类型 | `Buff` |
| 触发方式 | `CombatCard`（战斗卡主动使用） |
| 流程资产 | `FA_Rune_Finisher`（按实际命名） |

---

## 三、右侧战斗卡配置

| 字段 | 值 |
|---|---|
| 启用战斗卡 | ✅ |
| Card ID Tag | `Card.ID.Finisher` |
| 效果 Tags | `Card.Effect.Finisher` |
| 卡类型 | `Finisher` |
| 启用连招缩放 | ❌（终结技通常固定高倍率，不随连招段数额外缩放） |

> 卡类型选 `Finisher` 而非 `Attack`，系统会据此在 HUD 上显示专用图标，并可触发"终结技击杀"判定。

---

## 四、数值表配置

| Key | 显示名 | 值 | 分类 | 连招模式 | 每段奖励 | 上限 | 取整 |
|---|---|---|---|---|---|---|---|
| `Finisher.Damage.01` | 终结技伤害 | 80 | Damage | 无 | — | — | — |
| `Finisher.AOERadius.01` | 爆炸半径 | 300 | AOE | 无 | — | — | — |

> 终结技的核心数值一般**不随连招缩放**（`连招模式 = 无`），依靠本身的高基础值体现终结感。  
> 若设计希望连招深度影响终结伤害，可将 `Finisher.Damage.01` 改为乘算模式（`每段奖励 = 0.10`）。

---

## 五、流程图配置

### 方案 A：AOE 爆炸（推荐终结技标准做法）

```
Entry → 范围伤害(SpawnAOEDamage)
          Effect              = GE_FinisherDamage
          Radius            ← [读取数值 Pure] TuningKey = "Finisher.AOERadius.01"
          DamageValue       ← [读取数值 Pure] TuningKey = "Finisher.Damage.01"
          Target Policy      = EnemiesOnly
          Origin            = BuffOwner Location
          bShareAttackInstanceGuid = ❌
       → 播放特效(终结技 AOE VFX, Target=BuffOwner)
       → 结束符文
```

> `bShareAttackInstanceGuid = ❌`：终结技 AOE 命中多个目标时，每个目标独立计算伤害实例，避免共享 GUID 导致伤害只算一次。

### 方案 B：单目标高爆发

```
Entry → 属性修改(Instant)
          修改属性   = BaseAttributeSet.Health
          运算类型   = Add
          修改数值 ← [读取数值 Pure] TuningKey = "Finisher.Damage.01"
          施加目标   = LastDamageTarget（需先有命中上下文）
       → 播放特效(终结技命中 VFX)
       → 结束符文
```

## 六、GE_FinisherDamage 配置要点（AOE方案）

| GE 字段 | 建议值 |
|---|---|
| Duration Policy | `Instant` |
| Modifier Attribute | `BaseAttributeSet.Health` |
| Modifier Op | `Additive` |
| Magnitude | SetByCaller（Tag = `Data.Damage.Finisher`） |
| Asset Tags | `Buff.Type.Finisher` |

---

## 七、终结技上下文字段说明

终结技节点内可使用 `bIsComboFinisher` 上下文判断——此字段由战斗卡系统在打出 `Finisher` 类型卡时自动置为 `true`：

```
如果 bIsComboFinisher == true：
    → AOE 伤害（更大范围）
否则：
    → 单目标伤害（作为普通卡使用时）
```

> 通常直接在连携层区分（连携配方 `卡类型 = Finisher`），FA 内不需要额外判断。

---

## 八、连携配方配置（可选）

终结技通常作为连携**触发目标**（其他卡在前，终结技在后），也可配置反向倍率（终结技在前，额外卡在后）：

| # | 邻居 Tag | 匹配方式 | 正向 Flow | 正向倍率 | 正向说明 | 反向 Flow | 反向倍率 | 反向说明 |
|---|---|---|---|---|---|---|---|---|
| 0 | `Card.Effect.Moonlight` | 效果Tag | 空 | **2.0** | 月殒 | 空 | **1.2** | 旧月 |

> 终结技的连携倍率通常较高（正向 × 2.0），凸显"续连终结"的爆发感。

---

## 九、注意事项

- **`bShareAttackInstanceGuid = ❌`**：AOE 类终结技必须关闭，否则多目标命中时伤害实例 GUID 相同导致只计算一次伤害
- **卡类型必须选 `Finisher`**：系统依据此字段做 HUD 特效区分及击杀统计，不要错填为 `Attack`
- **连招缩放默认关闭**：终结技的高基础值本身即为连招奖励；如需开启，确保 `MaxBonus` 上限合理，避免多段连招后伤害过高
- **范围半径来自数值表**：`Finisher.AOERadius.01` 在 FA 内通过 Pure 节点读取，策划可以在编辑器直接调整，不需要改 FA
- **单张终结技可配置多套 AOE 半径**：如 `.01` 为普通版，`.02` 为连携触发版，通过 FA 内条件节点分支引用不同 Key
