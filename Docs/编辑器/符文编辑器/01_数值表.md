# 数值表（Value Table）使用说明

> 中间面板 → 切换到 **数值表** Tab

---

## 一、用途

数值表是符文所有可调参数的集中存储处。Flow 节点通过 **Key 字符串**引用这里的值，运行时由统一解析函数计算最终值（含连招加成）。

**好处：**
- 节点不写死数值，策划改数字只改数值表，不需要重新配节点
- 同一个 Key 可以被多个节点引用（如同一符文里多处读取同一伤害值）
- 每行可独立配置连招加成，无需在节点里手写连招逻辑

---

## 二、工具栏

```
[新增数值]  [插入模板 ▼]  [导出 CSV]  [导入 CSV]
```

| 按钮 | 功能 |
|---|---|
| **新增数值** | 插入一行空白数值，Key 默认为 `NewValue` |
| **插入模板 ▼** | 下拉选择符文类型，批量插入该类型的预设 Key + 默认值 + 连招配置 |
| **导出 CSV** | 将当前数值表复制为 CSV 到剪贴板（供外部对比/批量编辑） |
| **导入 CSV** | 从剪贴板读取 CSV 覆盖当前数值表（格式与导出一致） |

### 插入模板可选类型

| 类型 | 插入的 Key |
|---|---|
| 攻击类 | `Attack.Damage.01` ~ `.03` |
| 燃烧类 | `Burn.Damage.01~03`、`Burn.Duration.01~02` |
| 中毒类 | `Poison.Stack.01~02`、`Poison.Duration.01~02` |
| 月光类 | `Moonlight.ProjectileCount.01~02`、`Moonlight.ProjectileSpeed.01` |
| 终结技类 | `Finisher.Damage.01~02`、`Finisher.AOERadius.01` |

## 三、分类筛选栏

```
[全部] [伤害] [飞行物] [层数] [持续时间] [连招奖励]
```

点击分类按钮仅显示该分类的行，方便集中调整。`连招奖励` 只显示开启了连招加成的行。

## 四、列说明

| 列 | 说明 |
|---|---|
| **Key** | 节点引用用的字符串，全符文唯一；右侧 ▾ 按钮可从预定义列表中选择 |
| **显示名** | 策划可读名称，显示在编辑器中，不影响运行时 |
| **分类** | 用于筛选栏归组，建议用：`Damage` / `Duration` / `Stack` / `Projectile` / `Radius` |
| **数值方式** | 下拉选择：**具体值**（填数字）/ **公式**（填表达式）/ **MMC**（C++ 类计算）/ **上下文** |
| **具体值** | 当数值方式=具体值时生效，直接填数字，背景高亮绿色 |
| **公式** | 当数值方式=公式时生效，背景高亮蓝色 |
| **上下文** | 当数值方式=上下文时生效，背景高亮棕色 |
| **最小值/最大值** | 运行时解析值的钳制范围（0表示不钳制） |
| **ValueTag** | 可选 Gameplay Tag，供 MMC 或特殊读取使用 |
| **单位** | 显示用，如 `cm/s`、`秒`，不影响数值 |
| **说明** | 策划注释，不影响运行时 |
| **连招模式** | 无 / 加算 / 乘算，决定连招如何影响此 Key 的值 |
| **每段奖励** | 每增加一个连招段数的增量 |
| **奖励上限** | 连招加成的最大值（0=不限，建议始终填写） |
| **取整** | — / Floor / Round / Ceil，整数类数值必须配置 |
| **操作** | 红色删除按钮 |

## 五、Key 命名规范

```
符文类型.维度.序号
```

- 序号从 `.01` 开始，同一维度有多个独立值时递增（`.01`、`.02`、`.03`）
- 不同序号的 Key 完全独立，节点各自引用，互不影响

| 正确示例 | 说明 |
|---|---|
| `Burn.Damage.01` | 燃烧伤害第1阶段 |
| `Burn.Damage.02` | 燃烧伤害第2阶段（如暴击分支） |
| `Moonlight.ProjectileCount.01` | 月光弹数（标准） |
| `Finisher.Damage.01` | 终结技伤害 |

## 六、连招加成计算

```
ComboStacks = max(0, ComboIndex - 1)
```

| 连招模式 | 公式 |
|---|---|
| 加算 | `Final = Base + min(MaxBonus, Stacks × BonusPerStack)` |
| 乘算 | `Final = Base × (1 + min(MaxBonus, Stacks × BonusPerStack))` |

**取整**在加成计算后执行，整数类（弹数、层数）必须配置 Floor。

---

## 七、Key 与节点的对应方式

Flow 节点不直接读 Key，需要通过**数据引脚连线**传值：

```
[读取数值（Pure）]
  TuningKey = "Burn.Damage.01"    ──→  [属性修改]  修改数值 ○
  DefaultValue = 0.0
```

步骤：
1. 节点库 → 条件分类 → 找到 **「读取数值（Pure）」** → 添加
2. 在节点属性里填 `TuningKey`
3. 将 Value 输出引脚连到下游节点的数值输入引脚（圆形小点）

> Pure 节点无执行引脚，只在被查询时从当前 DA 数值表读值并自动带连招加成。  
> 如果不需要连招缩放，也可以在节点上直接填固定数字，不接 Pure 节点。

---

## 八、数据存储位置

数值表保存在符文 DataAsset 内：

```
URuneDataAsset → RuneInfo.RuneConfig.TuningScalars  (TArray<FRuneTuningScalar>)
```

双击 Content Browser 里任意 `DA_Rune_xxx`，在 Details 面板展开 `TuningScalars` 数组即可直接查看和编辑。

---

## 九、注意事项

- Key 区分大小写，节点里引用时必须精确匹配
- 同一符文内 Key 不能重复，否则只读取第一个匹配行
- `最小值 = 最大值 = 0` 表示不钳制；若要钳制请填非零值
- 数值表修改不会自动保存，点击编辑器工具栏的保存按钮或 `Ctrl+S` 保存 `.uasset`
