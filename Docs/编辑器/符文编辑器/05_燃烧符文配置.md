# 燃烧符文配置指南

---

## 一、定位

燃烧符文在命中目标时附加 **持续灼伤（DoT）GE**，并可选在地面留下火焰路径区域。  
核心参数：每周期伤害量 + 持续时间 + 触发间隔。

---

## 二、右侧基本信息

| 字段 | 值 |
|---|---|
| 类型 | `Debuff` |
| 触发方式 | `OnAttackHit`（命中附加）或 `Passive + OnDamageDealt`（常驻监听命中） |
| 流程资产 | 关联对应 FA |

---

## 三、数值表配置

| Key | 显示名 | 值 | 分类 | 连招模式 | 每段奖励 | 上限 | 取整 |
|---|---|---|---|---|---|---|---|
| `Burn.Damage.01` | 燃烧伤害/周期 | 20 | Damage | 乘算 | 0.15 | 0.60 | — |
| `Burn.Duration.01` | 燃烧持续时间 | 3 | Duration | 无 | — | — | — |

**连招加成计算示例（Burn.Damage.01）：**

| 连招段 | 伤害/周期 |
|---|---|
| 第1段 | 20 |
| 第2段 | 23 |
| 第3段 | 26 |
| 封顶 | 32 |

---

## 四、流程图配置

### 方案 A：命中附加燃烧 GE（推荐标准做法）

```
Entry → 触发伤害事件(OnDamageDealt)
              Out → 施加状态(GE_Burn)
                       Target    = LastDamageTarget
                       SetByCallerTag  = Data.Damage.Burn
                       SetByCallerValue ← [读取数值 Pure] TuningKey = "Burn.Damage.01"
                  → 播放特效(燃烧命中 VFX)
```

> GE_Burn 内部配置 Period（如 1.0s），Duration 由 GE 自身决定，或通过 SetByCaller 传入。

### 方案 B：地面火焰路径

```
Entry → 地面路径效果(SpawnRuneGroundPathEffect)
          Effect            = GE_Burn
          TargetPolicy      = EnemiesOnly
          Duration        ← [读取数值 Pure] TuningKey = "Burn.Duration.01"
          SetByCallerTag1   = Data.Damage.Burn
          SetByCallerValue1 ← [读取数值 Pure] TuningKey = "Burn.Damage.01"
          Length/Width/Height ← [读取光环模块 GetAuraModule] 输出引脚
```

---

## 五、GE_Burn 配置要点

| GE 字段 | 建议值 |
|---|---|
| Duration Policy | `Has Duration`（由 SetByCaller 或固定值控制时长） |
| Period | 1.0（每秒跳伤） |
| Modifier Attribute | `BaseAttributeSet.Health` |
| Modifier Op | Additive |
| Magnitude | SetByCaller（Tag = `Data.Damage.Burn`） |
| Stack Policy | `Aggregate by Source`（多次命中叠层） |
| Stack Limit | 3（防止无限叠加） |
| Asset Tags | `Buff.Status.Burn`（用于视觉判断） |

---

## 六、注意事项

- 燃烧 GE 的 Duration 建议用固定值（直接在 GE 里填），如果要让策划通过数值表控制则用 SetByCaller 传入
- 地面路径效果的 `Length/Width/Height` 推荐使用 **GetAuraModule** 节点，这样可以在模块配置 Tab 统一调整尺寸
- 如果燃烧符文同时有"伤害"和"地面路径"两种行为，建议分为两个不同的 FA，通过触发类型区分
- 连招模式配在数值表，FA 内不写 ComboIndex 判断，保持节点简洁
