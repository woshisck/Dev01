# 动作平衡编辑器（Action Balance）

菜单路径：`DevKit Data → Action Balance`

---

## 一、定位

动作平衡编辑器用于批量查看和调整**近战与枪械动作**的帧数参数，包括命中窗口、连招窗口和早期退出帧。适合：
- 调整攻击的命中判定时机
- 对比不同武器动作的节奏参数
- 快速创建缺失的攻击数据资产

---

## 二、支持的动作类型

| 类型标识 | 说明 |
|---|---|
| MeleeAttackData | 近战攻击数据（基础参数） |
| MeleeHitWindow | 近战命中窗口（伤害判定开始/结束帧） |
| MeleeComboWindow | 近战连招窗口（可续招的输入接受帧） |
| MeleeEarlyExit | 近战早期退出（连招衔接的提前结束帧） |
| MusketLight | 枪械轻攻击参数 |
| MusketHeavy | 枪械重攻击参数 |
| MusketSprint | 枪械冲刺攻击参数 |

---

## 三、关联资产

| 资产类型 | 说明 |
|---|---|
| `UMontageConfigDA` | 动画 Montage 配置，管理连招序列 |
| `UMontageAttackDataAsset` | 每段攻击的帧数区间数据 |
| `UGA_MusketBase` | 枪械 GA 基类，枪械参数入口 |

---

## 四、主要功能

### 4.1 列表与统计

顶部状态栏显示：
- 蓝图资产总数
- Montage 配置总数
- 攻击数据总数

列表按武器/GA 分组展示，点击展开查看每段攻击的帧数参数。

### 4.2 直接编辑

点击命中窗口的数字单元格可直接输入新的帧数值（起始帧/结束帧），回车提交。

### 4.3 创建新攻击数据

工具栏 → **创建攻击数据**：为选中的命中窗口生成对应的 `UMontageAttackDataAsset`，自动命名和路径。

### 4.4 枪械调优数据

工具栏 → **创建枪械调优**：为枪械 GA 生成配套的调优数据资产，用于分离帧数配置和数值配置。

### 4.5 验证与导出

- **验证**：检查每段攻击是否有有效的命中窗口数据
- **导出 CSV**：包含所有动作的帧数参数，可用于版本对比

---

## 五、典型使用场景

### 调整近战攻击第2段的命中帧

```
1. 打开 DevKit Data → Action Balance
2. 在列表中找到目标武器/角色的 MeleeComboGraph
3. 展开，找到第2段 (Combo_02)
4. 点击 HitWindow.Start 单元格 → 修改为目标帧数
5. 点击 HitWindow.End → 同步修改
6. Ctrl+S 保存
7. PIE 测试，结合战斗日志验证命中时机
```

### 检查连招衔接是否流畅

```
1. 找到目标连招序列
2. 查看 ComboWindow（连招输入窗口）的开始帧
3. 对比 EarlyExit（早期退出）的帧数
4. ComboWindow.Start ≤ 动画实际衔接帧 ≤ EarlyExit.Frame
   → 不满足此条件会导致连招卡顿或无法续招
```

---

## 六、注意事项

- 帧数参数以**动画帧**为单位，不是秒（需结合动画的 FrameRate 换算）
- 修改不支持 Undo，建议修改前记录原值
- 枪械参数（MusketLight/Heavy/Sprint）需要打开对应的 `UGA_MusketBase` 蓝图才能最终生效
- 此编辑器只管帧数窗口，**伤害数值**在符文编辑器的数值表或 GE 中配置
