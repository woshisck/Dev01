# 游戏开局教程配置清单

## 文档目的

本文是《Codex教程开发执行指南》的第 1 个执行产物，用于把“游戏开局教程方案”落成可配置、可拆任务、可验收的开发清单。

本轮只做审计和开发计划，不修改 C++、不创建资产、不改 DataAsset。

| `UCombatDeckComponent` | 已支持攻击消耗、洗牌、奖励入组、连携、终结技事件广播 | HUD 高亮和教程提示可监听这些事件 | 需要教程 HUD 订阅并触发短提示/高亮，而不是新增核心卡组逻辑 |

## 教程房间配置清单

| 阶段 | 房间目标 | 需要配置 | 可复用系统 | 需要新增实现 |
| --- | --- | --- | --- | --- |
| T0 默认局外选择 | 新玩家无需选择复杂系统即可开始 | 默认基础卡组模板；默认终结技；默认武器或武器房入口 | 存档 `TutorialState`；卡组初始化；临时终结技配置 | 新存档进入教程 Run 的入口判断；教程默认模板选择策略 |
| T1 武器房 | 玩家拾取武器后看到 1D 卡组 | 低压空房或无敌人房；武器拾取点；拾取后提示 `tutorial_weapon_pickup` 或新事件 | 武器拾取流程；`TutorialManager` 弹窗；卡组条 | 拾取武器后强制显示/高亮卡组条；进入下一房的条件 |
| T2 基础卡房 | 玩家理解攻击会按顺序消耗卡牌 | 1-2 个低威胁敌人或木桩；基础攻击卡；当前卡高亮提示 | `ResolveAttackCard`；卡组条当前卡显示 | 教程提示事件：第一次攻击消耗卡；HUD 高亮当前卡 |
| T3 洗牌房 | 玩家打空卡组并看到洗牌回填 | 高血量木桩或慢速敌人；较少初始卡；避免高伤秒杀 | `OnShuffleStarted`；卡组回填逻辑 | 教程提示事件：卡组打空、洗牌中、回填完成；需要保证敌人配置能稳定触发洗牌 |
| T4 奖励房 | 玩家清场后获得普通卡并加入卡组 | 房间 `LootPool` 限定为 1-3 张普通卡；清场后奖励拾取物 | `RewardPickup`；`LootSelectionWidget`；`SelectLoot`；`AddCardFromRuneReward`；`OnRewardAddedToDeck` | 教程固定奖励池策略；奖励入组后的卡组条高亮 |
| T5 连携房 | 玩家获得第一张连携卡并触发一次连携 | 固定连携卡；固定摆位或自动摆位；低压敌人用于触发 | `OnLinkTriggered`、`OnForwardLinkTriggered`、`OnBackwardLinkTriggered`；已有 `tutorial_card_link` | 连携卡教学奖励和自动摆位规则；连携原因提示文案 |
| T6 终结技房 | 玩家完成一次终结技爆发 | 默认终结技可用；敌人血量足够；战斗时机能稳定触发 | 临时终结技卡；`OnFinisherTriggered` | 教程中终结技解锁条件需要强控，避免默认“完成 3 战后解锁”阻断教学 |
| T7 路线选择 | 玩家第一次理解“本局变强”和“投资下一局” | 两个传送门；A 战斗奖励房；B 局外资源房；传送门预览文案 | `PortalDestinations`；传送门 UI；房间选择 | 路线奖励标识；局外资源奖励记录；教程完成或转入第二局入口 |

## T0-T7 详细配置建议

### T0 默认局外选择

- 默认基础模板：只包含攻击、防御或位移相关的基础卡，不出现连携说明。
- 默认终结技：只给 1 张，不开放多终结技选择。
- 推荐实现策略：新存档检测 `TutorialState != Completed` 时进入教程 Run，使用固定 Campaign/Room 序列。
- 验收点：玩家点击开始后不需要理解局外成长，也不会看到复杂卡池。

### T1 武器房

- 房间类型：安全房或无战斗房。
- 房间配置：武器拾取点、出口或下一房传送门。
- 提示重点：武器决定攻击方式，屏幕上的卡组会影响攻击效果。
- 验收点：拾取武器后卡组条可见，并出现短提示。

### T2 基础卡房

- 房间类型：低压战斗房。
- 敌人配置：低血量、低攻击频率，允许玩家观察卡牌消耗。
- 奖励配置：不掉落奖励。
- 提示重点：每次攻击会消耗当前卡，下一张卡进入待使用位置。
- 验收点：玩家至少完成 3 次攻击消耗。

### T3 洗牌房

- 房间类型：教学战斗房。
- 敌人配置：高血量木桩或慢速敌人，避免玩家在洗牌前清场。
- 奖励配置：不掉落奖励，或只在下一阶段掉落。
- 提示重点：卡组打空后进入洗牌，洗牌结束后卡组回填。
- 验收点：玩家至少触发一次 `OnShuffleStarted`。

### T4 奖励房

- 房间类型：普通战斗房。
- 敌人配置：低压敌人，清场后进入整理阶段。
- 奖励配置：固定普通卡，例如伤害卡、防御卡、范围卡三选一；第一版也可以只给 1 张。
- 提示重点：清场奖励会加入本局卡组，并从下一房开始参与循环。
- 验收点：选择奖励后触发 `OnRewardAddedToDeck`，卡组条展示新卡。

### T5 连携房

- 房间类型：连携教学房。
- 敌人配置：低压但血量足够，保证连携触发后能看出伤害变化。
- 奖励配置：固定连携卡，不从完整连携池随机。
- 提示重点：某些卡会影响相邻卡，卡牌顺序会改变效果。
- 验收点：玩家至少触发一次连携事件，并看到原因提示。

### T6 终结技房

- 房间类型：爆发验证房。
- 敌人配置：血量和数量要能让玩家积累并释放终结技。
- 奖励配置：不引入新系统，避免分散注意力。
- 提示重点：终结技是本局构筑目标，满足条件后产生强反馈。
- 验收点：玩家至少触发一次 `OnFinisherTriggered`。

### T7 路线选择

- 房间类型：二选一传送门房。
- 关卡 A：战斗奖励房，预览“获得卡牌/升级/强化本局战力”。
- 关卡 B：局外资源房，预览“获得材料/研究碎片/图纸，推动下局成长”。
- 提示重点：本局变强和局外投资是两条不同收益线。
- 验收点：玩家能在传送门预览中看懂奖励倾向。

## 当前缺口列表

| 缺口 | 影响 | 建议优先级 |
| --- | --- | --- |
| 教程 Run 入口 | 新存档无法稳定进入强控第一局 | P0-1 |
| 教程房间强控顺序 | 现有房间选择偏随机或依赖 Campaign 配置，不能保证 T1-T7 顺序 | P0-2 |
| 固定奖励池 | T4/T5 不能保证奖励是普通卡/连携卡 | P0-2 |
| HUD 高亮和短提示 | 玩家可能看不到卡牌消耗、洗牌、奖励入组原因 | P0-3 |
| 连携教学强触发 | 只给连携卡不等于玩家必然触发 | P0-4 |
| 终结技强触发 | 临时终结技可能受完成战斗数锁定影响 | P0-5 |
| 路线奖励标识 | 玩家无法区分战斗奖励房和局外资源房 | P0-6 |
| 教程完成标记 | 完成第一局后可能重复进入教程 | P0-7 |
| 局外资源记录 | 资源房奖励无法带回局外成长 | P1 |

## 第一阶段最小实现任务

### P0-1 新存档进入主城并触发首局教程

- 目标：新存档第一次开始游戏时进入主城，由教程/故事事件系统按全局阶段触发首局教学。
- 修改范围：`TutorialManager`、存档教程状态、主城入口、`CampaignDataAsset.FFloorConfig` 的全局流程接口。
- 不做：教程专用关卡入口、局外成长 UI、武器制作、道具池。
- 验收：新存档进入主城后触发首次武器/卡组提示；教程完成后再次进入主城不重复触发。

当前方向：

- 主城包含武器选择、武器升级、局外成长和出发入口，不再把“武器房”做成独立关卡房。
- 关卡编辑器只编辑正式关卡数据和全局流程数据，不和教程流程强绑定。
- 教程系统通过 `GlobalStageTag` / `StoryEventTags` 监听全局阶段，例如主城、第一战斗关、奖励选择、资源选择。

需要配置：

- 在 `CampaignDataAsset.FloorTable` 中为关键楼层填写 `GlobalStageTag`。
- 在 `StoryEventTags` 中填写教程/故事可监听标签，例如 `Tutorial.CardConsume.Enabled`、`Tutorial.RewardToDeck.Enabled`。

验收步骤：

- 新存档进入主城后，教程事件系统可根据全局阶段触发提示。
- `TutorialState == Completed` 后，同一全局阶段不重复触发首局教程。
- 编译 `DevKitEditor Win64 Development` 通过。

### P0-2 教程房间顺序和固定奖励池

- 目标：让 T1-T7 按固定顺序推进，并让 T4/T5 使用教程奖励池。
- 修改范围：Campaign/RoomData 配置入口、`YogGameMode` 房间选择和奖励生成。
- 不做：完整随机地图、完整奖励池平衡。
- 验收：每次新教程 Run 都按 T1-T7 顺序出现。

### P0-3 教程 HUD 高亮和短提示

- 目标：卡组消耗、洗牌、奖励入组有可见反馈。
- 修改范围：教程事件、HUD/卡组条监听。
- 不做：大规模 UI 重做。
- 验收：第一次攻击、第一次洗牌、第一次奖励入组都出现短提示。

### P0-4 连携教学房

- 目标：玩家在 T5 必然获得并触发一次连携。
- 修改范围：固定连携奖励、卡组摆位、连携事件提示。
- 不做：完整正逆位规则说明。
- 验收：玩家看到连携原因和伤害/效果变化。

### P0-5 终结技教学房

- 目标：玩家在 T6 必然触发一次默认终结技。
- 修改范围：终结技临时解锁条件、敌人配置、终结技事件提示。
- 不做：多终结技切换。
- 验收：触发 `OnFinisherTriggered`，并出现强反馈提示。

### P0-6 战斗奖励房 / 局外资源房二选一

- 目标：T7 让玩家第一次做“本局变强 vs 投资下一局”的选择。
- 修改范围：传送门预览、RoomData 目标池、资源房奖励记录。
- 不做：完整据点设施。
- 验收：两个门显示不同奖励倾向，选择后进入对应房间。

### P0-7 教程完成标记和第二局入口

- 目标：教程结束后写入完成状态，并开放第二局默认入口。
- 修改范围：`TutorialState`、结算/返回局外流程、第二局起始配置。
- 不做：完整天赋树和武器制作。
- 验收：教程完成后回到局外，第二次开始不再走强控教程。

## 推荐开发顺序

```text
P0-1 教程 Run 入口
-> P0-2 房间顺序 + 固定奖励池
-> P0-3 卡组 HUD 反馈
-> P0-4 连携教学
-> P0-5 终结技教学
-> P0-6 路线二选一
-> P0-7 完成标记和第二局入口
```
