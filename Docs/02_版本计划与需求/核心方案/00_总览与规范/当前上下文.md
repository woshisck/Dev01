# 当前上下文

## 来源

该计划来自 429 反馈迭代后的方向确认，并在 核心方案项目计划中拆成多个 Codex 小组执行。

核心反馈包括：

- 热度系统能理解设计意图，但存在感不强，并且有强迫攻击感。
- 旧 2D 背包太复杂，玩家很少停下来整理。
- 玩家更希望战斗是快速、爽快、连续验证构筑的体验。
- 关卡结算和奖励拾取流程偏生硬。

## 已确认方向

游戏战斗节奏采用“快速动作战斗”为主，参考黑帝斯的爽快战斗节奏，同时保留少量魂系的出招承诺和读招压力。

系统方向为：

- 攻击卡组成为前台主系统。
- 旧 2D 背包退为后台存储。
- 普通卡不再依赖轻重动作匹配，攻击出牌即正常触发。
- 少数连携卡通过卡牌前后顺序、连招是否继续、动作条件产生明显机制变化。
- 卡牌打空后进入洗牌或装弹空窗。
- 洗牌期间玩家仍可攻击，但不触发卡牌。
- 热度用于强化连携、表现和倍率，不再控制符文激活区；终结技后续阶段再处理。
- 关卡奖励选择后自动加入后台背包和卡组，不自动打开旧背包。

## Codex 小组

| 小组 | 说明 |
| --- | --- |
| Codex-背包符文卡组组 | 当前我们所在小组，负责 CombatDeckComponent、卡牌数据、洗牌、后台背包、奖励入卡组 |
| Codex-战斗系统组 | 负责 Light / Heavy、连招终结、DashSave、攻击事件派发 |
| Codex-关卡奖励组 | 负责房间节奏、清场奖励、小怪掉落、传送门 |
| Codex-UI表现组 | 负责 1D 卡组 HUD、洗牌表现、连携原因提示 |
| Codex-热度 BuffFlow 组 | 负责热度新定位、Flow 执行、倍率和表现强度 |
| Codex-联调验收组 | 负责跨系统联调、验收场景、问题归属 |

## 最小战斗循环

1. 玩家进入房间，看到当前 1D 卡组。
2. 玩家用 Light / Heavy 攻击敌人。
3. 每次攻击尝试消耗当前卡牌。
4. 普通卡攻击出牌即触发基础效果。
5. 连击数或连招阶段可以提升普通卡倍率。
6. 少数连携卡根据前一张/后一张卡和连招是否继续，触发正向连携、反向赋能或断链释放。
7. 卡组打空后进入洗牌，UI 暂时为空。
8. 洗牌期间玩家继续攻击，但不触发卡牌。
9. 冷却结束后卡组按固定顺序回填。
10. 清场后选择奖励，奖励进入后台背包与卡组。
11. 下一房间立即验证新卡组。
