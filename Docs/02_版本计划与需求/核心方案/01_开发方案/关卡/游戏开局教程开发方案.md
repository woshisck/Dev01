# 游戏开局教程开发方案

## 定位

本方案负责整理当前确认的开局教学方向。新的开局教学不再等同于“第一局强控教程 Run”，而是拆成三个阶段：

```text
第一局前：记忆碎片基础教程，教玩家动作身体感。
第一局中：正式接触世界能力，开始讲符文、奖励、构筑。
第一局后：进入主城和局外成长，建立大循环。
```

当前文档只详细定义第一阶段，也就是“记忆碎片基础教程”。第一局正式能力引导、第一局后主城成长、故事引擎 MVP 需要另起文档继续展开。

## 核心体验

玩家在第一局前不是直接进入正式 Roguelite 流程，而是进入教宗残缺记忆中的战斗片段。这个阶段的目的不是解释世界观，也不是完整讲系统，而是让玩家恢复最基础的战斗直觉。

玩家此时扮演的是教宗的记忆碎片。教学中的移动、攻击、闪避、连招都可以被理解为身体记忆的重新校准。普通失败不会进入正式死亡循环，而是回到最近的记忆锚点。教程结尾会安排一次剧情杀，把玩家从记忆碎片推向正式游戏。

## 设计边界

### 本阶段负责

| 模块 | 目标 |
| --- | --- |
| 移动与观察 | 让玩家理解移动、视角、空间方向和出口。 |
| 交互 | 让玩家理解拾取、调查、触发记忆物件。 |
| 固定教宗武器 | 用一把固定武器教学基础动作语言。 |
| 轻攻击 | 教快速起手、稳定输出和基础击杀。 |
| 重攻击 | 教重攻击是连招分支的一部分，而不是单纯破防按钮。 |
| 连招分支 | 让玩家理解轻重攻击混合后动作会变化。 |
| 闪避 | 教玩家处理危险距离和敌人攻击。 |
| 敌人前摇/红光 | 教玩家观察攻击预兆，而不是无脑换血。 |
| 综合遭遇 | 让玩家整合轻重攻击、连招、闪避、读招。 |
| 普通失败回退 | 失败回到记忆锚点，不触发正式死亡循环。 |
| 剧情杀 | 由骑士团成员终止记忆片段，进入正式游戏。 |

### 本阶段不负责

- 不讲完整符文构筑。
- 不讲热度。
- 不讲连携系统。
- 不讲三选一奖励。
- 不讲路线选择。
- 不讲局外成长。
- 不讲商店、祭坛、主动技能。
- 不让黑夜少女在最开始承担教程播报。
- 不展开骑士团 Boss 身份、关系和完整背景设定。

## 卡牌消耗的呈现边界

记忆碎片教程中可以出现“攻击会消耗或推进符文卡牌能力”的现象，但只作为 HUD、动作手感和战斗反馈的一部分存在，不做完整系统解释。

玩家在这个阶段应该得到模糊直觉：

```text
攻击时，某些卡牌或符文状态会变化。
这些变化似乎和武器动作有关。
但完整规则还没有被说明。
```

第一局正式游玩时，再通过玩法和引导详细说明：

```text
攻击会消耗当前卡牌。
卡牌来自武器和符文。
获得符文后会进入构筑。
背包可以调整顺序。
后续可以通过热度和连携扩展效果。
```

因此，记忆碎片阶段是“先看见现象”，第一局阶段才是“理解规则”。

## 武器方案

记忆碎片教程只使用一把固定的教宗武器。其他武器不在本阶段开放。

这把武器承担三个职责：

| 职责 | 说明 |
| --- | --- |
| 教学职责 | 轻击、重击、轻重组合、闪避反击都要清晰。 |
| 身份职责 | 让玩家意识到自己不是普通囚徒，而是在重演教宗记忆。 |
| 断裂职责 | 剧情杀后武器可以失去、破碎、封印或降格，解释正式游戏中能力不完整。 |

正式第一局之后，再逐步开放其他武器、符文能力、构筑奖励和局外成长。

## 空间结构

教程空间采用“小型训练庭院式”结构。它不是线性走廊，也不是完整小关卡，而是一个中央区域连接多个训练点的记忆空间。

建议结构：

```text
中央记忆庭院
  -> 入口锚点：苏醒、移动、观察
  -> 武器台：交互、拾取教宗武器
  -> 轻击残影区：轻攻击
  -> 重击/分支残影区：轻重连招变化
  -> 红光残影区：闪避和读招
  -> 综合战斗区：组合敌人
  -> 封仪门/审判台：剧情杀
```

空间上允许玩家感到自己在探索记忆，但具体开放节奏可以在制作阶段根据关卡实现成本决定。

## 教学内容顺序

### 1. 苏醒与移动

玩家在中央记忆庭院醒来。空间破损、封闭，远处能看到武器台、残影门或审判台。

教学目标：

- 移动。
- 观察空间。
- 看见可交互物和出口方向。

### 2. 交互与教宗武器

玩家靠近武器台，拾取或唤醒教宗武器。

教学目标：

- 理解交互提示。
- 获得固定教程武器。
- 看见 HUD 中基础攻击/卡牌状态开始出现。

### 3. 轻攻击

出现低压残影敌人，敌人几乎不主动进攻或攻击频率很低。

教学目标：

- 轻攻击能快速起手。
- 轻攻击可以稳定击杀低阶敌人。
- 击杀反馈明确。

### 4. 重攻击与连招分支

出现更耐打或更容易被连段压制的残影敌人。

教学目标：

- 重攻击不是单纯破防按钮，而是连招分支的一部分。
- 轻攻击后接重攻击，动作会变化。
- 重攻击有更高承诺，不能无脑使用。

可引导玩家体验：

```text
轻击
重击
轻轻轻
轻轻重
轻重
重轻
```

本阶段不要求玩家记住完整 Combo 表，只要形成“轻重攻击可以混合，动作会变化”的认知。

### 5. 闪避与红光

出现攻击前摇明显的敌人。敌人在攻击前使用红光、蓄势动作或音效提示。

教学目标：

- 敌人攻击前有可读信息。
- 红光或明显前摇意味着危险。
- 玩家需要闪避后反击。

### 6. 综合遭遇

出现一组简单敌人，例如一个快敌人和一个慢敌人。

教学目标：

- 玩家整合轻攻击、重攻击、连招分支、闪避和读招。
- 玩家理解不要只追求打完完整连段，需要根据敌人动作中断或调整节奏。

### 7. 剧情杀

综合战斗完成后，玩家走向封仪门、审判台或出口。骑士团成员进入记忆并终止片段。

设计原则：

- 剧情杀来源是骑士团成员。
- 具体 Boss 身份、关系和后续定位暂不展开。
- 剧情杀不是普通数值失败，而是记忆片段被骑士团封仪、处刑或规则外力量终止。
- 玩家可以短暂抵抗，但最终必须被推出记忆。

剧情杀的作用：

```text
确认玩家已经完成基础战斗学习。
展示骑士团压迫感。
让玩家意识到教宗记忆并不完整。
把玩家送入正式第一局或现实醒来段。
```

## 失败与检查点

记忆碎片教程允许失败，但失败只回到最近的记忆锚点，不触发正式死亡循环。

```text
普通失败：画面破碎、记忆回卷、回到当前训练点前的锚点。
剧情杀：教程结束事件，进入正式游戏，不视为玩家操作失败。
```

记忆锚点可以理解为：

- 教宗灵魂碎片中的稳定片段。
- 教程段检查点。
- 玩家失败后重新校准身体记忆的位置。

## 引导文案原则

本阶段使用“记忆回响”或“战斗记录”式提示，不使用黑夜少女作为完整教程播报员。

文案要像残缺记忆，不要像系统说明书。

推荐语气：

```text
快击，是试探。
重击，不是结束，是转折。
记忆里的身体仍记得下一段。
红光之前，身体已经知道要躲。
不要追逐完整的连段，活着完成它。
```

避免语气：

```text
按下鼠标左键进行轻攻击。
普通攻击会按顺序消耗战斗卡牌。
轻攻击两次后接重攻击可以触发某某连招。
```

系统提示可以存在，但要短，只负责告诉玩家当前可操作项。

## 与后续阶段的关系

### 第一局正式游玩

第一局不是重复基础战斗教学，而是在玩家已经掌握动作基础后，开始引入世界能力。

第一局可以负责：

- 解释攻击与符文卡牌消耗的关系。
- 解释符文奖励。
- 解释背包和构筑。
- 逐步露出热度或连携，但不一定一次讲全。
- 让玩家理解战斗中的收获会改变后续战斗。

### 第一局后

第一局后再进入主城和局外成长。

第一局后可以负责：

- 解释战斗带回资源。
- 解释主城。
- 解释升级或功能解锁。
- 引出黑夜少女和《遗圣目录》的分工。
- 建立下一局出发目标。

### 故事引擎 MVP

故事引擎第一版不需要做完整剧情系统。它应优先服务这些体验节点：

```text
记忆碎片教程开始
记忆锚点回退
剧情杀
正式第一局开始
首次获得世界能力
第一局结束
首次回到主城
首次局外成长
```

后续需要单独设计 StoryFlag、StoryEvent、StoryCondition、StoryAction 和 QuestTask。

## 当前确认结论

- 开局教程拆到第一局之前，不再把完整第一局作为强控教程。
- 教程段是教宗记忆碎片中的基础战斗校准。
- 教程空间采用小型训练庭院式。
- 教程武器固定为教宗武器。
- 教程要教轻攻击、重攻击、连招分支、闪避、红光和综合战斗。
- 攻击消耗或推进符文卡牌能力的现象可以出现，但不完整解释。
- 普通失败回到记忆锚点。
- 教程结尾由骑士团成员剧情杀。
- 黑夜少女、符文构筑、连携、局外成长放到教程之后逐步展开。
