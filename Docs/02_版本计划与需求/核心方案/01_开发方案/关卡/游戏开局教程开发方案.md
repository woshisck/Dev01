# 游戏开局教程开发方案

## 组定位

本方案负责把第一局做成可玩的强控教程 Run。重点是教程体验，不是完整随机局。

## 负责范围

| 模块 | 说明 |
| --- | --- |
| 教程入口 | 新存档默认进入教程 Run |
| 教程房间 | 武器、基础卡、洗牌、奖励、连携、终结技、路线选择 |
| 教程奖励 | 固定普通卡、固定连携卡、固定资源奖励 |
| 教程提示 | 短弹窗、HUD 高亮、卡组条反馈 |
| 教程完成 | 标记教程完成，开放第二局自由度 |

## 不负责范围

- 不做完整局外设施。
- 不做复杂道具池。
- 不做完整随机地图。
- 不重写 CombatDeckComponent 核心逻辑。

## 开发步骤

1. 增加教程 Run 标记：新存档进入教程第一局。
2. 配置教程房间顺序：武器房 -> 基础卡房 -> 洗牌房 -> 奖励房 -> 连携房 -> 终结技房 -> 路线选择。
3. 强控第一局奖励池：只掉已教学过的普通卡和连携卡。
4. 配置敌人血量和波次：保证玩家能看到消耗、洗牌、连携和终结技。
5. 接入短提示：每个教学点只出现 1-2 句。
6. 接入 HUD 高亮：当前卡、洗牌条、新卡入组、连携原因、终结技条件。
7. 教程结束后写入完成状态，进入局外结算。

## 验收标准

- 新存档能稳定进入教程 Run。
- 第一局没有复杂商店、祭坛、完整房间 Buff。
- 玩家必然触发一次奖励入组、一次连携、一次终结技。
- 第一局结束后能进入局外结算，不重复触发教程。
