# Codex-联调验收组开发方案

## 组定位

本组负责跨系统联调、验收场景、问题回收和版本收口。联调验收组不拥有主要系统实现，但负责确认多个 Codex 小组的工作能组合成完整闭环。

## 负责范围

| 模块 | 说明 |
| --- | --- |
| 接口检查 | 确认战斗、卡组、UI、关卡使用同一套接口 |
| 场景验收 | 跑通 4 张卡、洗牌、奖励、终结、DashSave |
| 问题记录 | 标记问题属于哪个 Codex 小组 |
| 回归检查 | 修复后重复跑同一验收场景 |
| 版本收口 | 确认 512 最小闭环可演示 |

## 验收主线

```text
进入房间 -> 看到 1D 卡组 -> 攻击消耗卡牌 -> 卡组打空 -> 洗牌空窗 -> 回填 -> 清场 -> 选奖励 -> 加入卡组 -> 下一房验证
```

## 核心联调场景

| 场景 | 预期 |
| --- | --- |
| Light 卡 + Light 攻击 | 触发 BaseFlow + MatchedFlow |
| Light 卡 + Heavy 攻击 | 卡牌消耗，只触发 BaseFlow |
| 4 张卡连续攻击 4 次 | 第 4 张后进入洗牌 |
| 洗牌期间攻击 | 攻击不被阻止，不触发卡牌 |
| 洗牌结束 | UI 按固定顺序回填 |
| Finisher 卡 + 普通攻击 | 不触发终结技 |
| Finisher 卡 + Combo Finisher | 触发终结技效果 |
| DashSave 后攻击 | 保留连招和卡牌上下文 |
| 清场奖励选择 | 奖励进入后台背包和 DeckList |
| 跳过奖励进门 | 流程不阻塞，但放弃成长 |

## 问题归属规则

| 问题 | 归属 |
| --- | --- |
| 卡牌顺序错误、洗牌错误 | Codex-背包符文卡组组 |
| Light / Heavy 传参错误 | Codex-战斗系统组 |
| UI 不刷新、提示不清楚 | Codex-UI表现组 |
| 奖励不出现、进门阻塞 | Codex-关卡奖励组 |
| 热度倍率异常、Flow 未执行 | Codex-热度 BuffFlow 组 |
| 多系统都有问题 | Codex-联调验收组先复现，再拆归属 |

## 开发步骤

1. 建立一张固定 4 卡测试卡组。
2. 建立一个短普通房，能稳定清场。
3. 建立一个精英房，能稳定经历洗牌。
4. 按核心联调场景逐条验证。
5. 每个失败场景记录复现步骤、预期、实际、归属小组。
6. 修复后重新跑对应场景。
7. 最后跑完整最小战斗循环。

## 验收标准

- 最小战斗循环可以从进房跑到下一房。
- 卡组打空、洗牌、回填逻辑稳定。
- 战斗期间没有因为卡组状态阻止攻击。
- 奖励选择后不打开旧背包。
- 问题归属清楚，可以分给对应 Codex 小组。
