# 冲刺与 DashSave

## 目标

冲刺要继续保留快速动作游戏的流畅感，但不能随时打断攻击前摇。否则玩家不需要承担出招选择，卡牌和连招节奏也会失去存在感。

## 冲刺衔接规则

| 攻击阶段 | 是否允许冲刺 | 说明 |
| --- | --- | --- |
| 攻击前摇 | 默认不允许 | 保留攻击选择的承诺感 |
| 攻击命中帧附近 | 按技能配置 | 特殊武器或技能可开放 |
| 攻击后摇 | 允许 | 让战斗保持快速 |
| 桥接窗口 | 允许 | 用于轻重攻击之间的节奏衔接 |
| DashSave 窗口 | 允许 | 冲刺后保留一次当前连招与卡牌上下文 |

## DashSave 需要保留的内容

DashSave 不只保留战斗连招，也要保留卡牌连携上下文。

| 内容 | 是否保留 | 说明 |
| --- | --- | --- |
| 当前连招段 | 是 | 冲刺后允许继续衔接下一段 |
| Combo Finisher 判断 | 是 | 不能因为冲刺导致终结击丢失 |
| 上一张卡牌上下文 | 是 | Link 卡需要读取上一张或传递给下一张 |
| 当前卡组 Index | 是 | DashSave 不额外跳卡、不回退卡 |
| 洗牌冷却 | 是 | DashSave 不暂停洗牌计时 |

## 与卡组接口的关系

DashSave 后的攻击需要调用：

```cpp
ResolveAttackCard(ActionType, bIsComboFinisher, true);
```

普通攻击调用：

```cpp
ResolveAttackCard(ActionType, bIsComboFinisher, false);
```

## 验收标准

- 冲刺不能随时取消攻击前摇。
- 冲刺可以在后摇、桥接窗口、DashSave 窗口衔接。
- DashSave 后继续攻击不会重置卡组序列。
- DashSave 期间如果卡组已经打空，洗牌冷却继续推进。
