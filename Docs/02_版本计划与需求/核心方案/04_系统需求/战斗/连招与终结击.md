# 连招与终结击

## 目标

核心方案不追求完整复杂分支连招树。第一版只需要让玩家能通过轻攻击、重攻击的顺序，和卡牌序列产生明确关系。

核心体验是：玩家知道自己下一张卡需要什么动作，并尝试用轻重攻击顺序打出更高收益。

## 第一版范围

| 内容 | 是否进入第一版 | 说明 |
| --- | --- | --- |
| 轻攻击序列 | 是 | 例如 Light-Light-Light |
| 重攻击序列 | 是 | 例如 Heavy-Heavy |
| 轻重混合序列 | 是 | 例如 Light-Light-Heavy |
| Combo Finisher 标记 | 是 | 每条基础连招的最后一击标记为终结击 |
| 鬼泣式复杂分支树 | 否 | 暂不制作大量派生路径 |
| 长时间浮空追击 | 否 | 暂不作为 512 核心目标 |

## 终结击规则

每条基础连招的最后一击应提供 `bIsComboFinisher = true`。

示例：

| 连招 | 终结击 |
| --- | --- |
| Light-Light-Light | 第三下 Light |
| Light-Light-Heavy | 第三下 Heavy |
| Heavy-Heavy | 第二下 Heavy |

## 与卡牌的关系

卡牌系统会根据 `bIsComboFinisher` 判断 Finisher 卡是否触发额外效果。

| 场景 | 卡组表现 |
| --- | --- |
| 普通攻击消耗 Finisher 卡 | 只触发 BaseFlow |
| 终结击消耗 Finisher 卡，动作也匹配 | 触发 BaseFlow + MatchedFlow + Finisher 效果 |
| 终结击消耗普通 Attack 卡 | 正常按 Attack 卡规则处理 |

## 验收标准

- 每条基础轻重连招都有明确最后一击。
- 最后一击能稳定传出 `bIsComboFinisher = true`。
- 非最后一击不会误标记为终结击。
- Finisher 卡不会在普通攻击上触发终结技效果。
