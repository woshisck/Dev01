# 攻击事件接口

## 目标

战斗系统只负责告诉卡组系统“玩家刚刚打出了一次什么攻击”。卡组系统负责判断是否有卡、是否消耗、是否匹配、是否触发效果、是否进入洗牌。

这样可以避免战斗逻辑直接读取卡组内部状态，也避免卡组系统反过来控制攻击动作。

## 战斗侧调用

建议冻结接口：

```cpp
FCombatCardResolveResult ResolveAttackCard(
    ECardRequiredAction ActionType,
    bool bIsComboFinisher,
    bool bFromDashSave
);
```

## 参数说明

| 参数 | 来源 | 说明 |
| --- | --- | --- |
| `ActionType` | 战斗输入或攻击段配置 | 本次攻击类型，第一版只支持 Light / Heavy / Any |
| `bIsComboFinisher` | 连招系统 | 本次攻击是否为轻重连招的最后一击 |
| `bFromDashSave` | DashSave 系统 | 本次攻击是否来自 DashSave 后的保留衔接 |

## 调用时机

第一版优先选择以下任一稳定时机，不在多个时机重复调用：

| 时机 | 推荐程度 | 说明 |
| --- | --- | --- |
| 攻击命中时 | 推荐 | 卡牌效果与命中反馈同步，玩家理解成本最低 |
| 出招确认时 | 可选 | 适合需要未命中也消耗卡牌的设计 |

当前建议先采用“命中时消耗卡牌”。如果后续希望玩家空挥也承担节奏成本，再改为“出招确认时消耗”。

## 返回结果

战斗侧只需要读取返回结果来播放表现，不需要知道卡组内部列表。

| 字段 | 说明 |
| --- | --- |
| `bHadCard` | 本次攻击是否有可用卡牌 |
| `bConsumedCard` | 是否消耗了一张卡牌 |
| `bActionMatched` | 攻击动作是否符合卡牌 RequiredAction |
| `bTriggeredBaseFlow` | 是否触发基础效果 |
| `bTriggeredMatchedFlow` | 是否触发匹配奖励 |
| `bTriggeredLink` | 是否触发连携 |
| `bTriggeredFinisher` | 是否触发终结技效果 |
| `bStartedShuffle` | 本次消耗后是否进入洗牌 |
| `ResolvedCardId` | 被消耗卡牌的运行时标识 |

## 验收标准

- Light 攻击能把 `ActionType` 传为 Light。
- Heavy 攻击能把 `ActionType` 传为 Heavy。
- 连招最后一击能把 `bIsComboFinisher` 传为 true。
- DashSave 衔接攻击能把 `bFromDashSave` 传为 true。
- 洗牌期间调用接口时，返回 `bHadCard = false`，攻击本身不被阻止。
