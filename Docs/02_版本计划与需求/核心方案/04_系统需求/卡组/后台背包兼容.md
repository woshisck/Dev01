# 后台背包兼容

## 目标

旧 `BackpackGridComponent` 不删除，但不再作为玩家每局战斗后的主要操作界面。它退到后台，负责存储、存档、奖励接收、金币、升级等基础能力。

前台战斗心智由 `CombatDeckComponent` 和 1D 卡组 UI 承担。

## 保留职责

| 职责 | 说明 |
| --- | --- |
| 持有符文或卡牌 | 玩家获得的所有符文和卡牌仍可进入后台背包 |
| 存档 | 保存玩家拥有物、等级、来源、卡组顺序 |
| 奖励接收 | 关卡奖励选择后先进入后台背包 |
| 金币和资源 | 保留现有资源持有能力 |
| 升级与强化 | 后续升级可继续依赖后台背包数据 |

## 移除前台负担

| 旧行为 | 核心方案处理 |
| --- | --- |
| 战斗后自动打开 2D 背包 | 不再默认打开 |
| 要求玩家旋转、移动、裁剪符文 | 第一版不做 |
| 背包格子决定战斗触发 | 改由 1D 卡组顺序决定 |
| 热度扩展激活区 | 热度改为强化连携和终结表现 |

## 与 CombatDeckComponent 的关系

后台背包负责“玩家拥有什么”，卡组组件负责“本次战斗按什么顺序触发”。

数据流：

```text
关卡奖励 -> 后台背包 -> 当前卡组 DeckList -> ActiveSequence -> 战斗触发
```

## 验收标准

- 选择奖励后，旧背包能记录该卡牌或符文。
- 选择奖励后，不自动打开旧背包 UI。
- CombatDeckComponent 能从后台背包读取当前卡组顺序。
- 旧 2D 背包相关存档不被删除。
