# 联调验收清单

## 目标

该清单用于 核心方案第一次联调。只要以下场景通过，就说明战斗系统、卡组系统、关卡奖励系统的核心闭环已经成立。

## 战斗与卡组

| 场景 | 操作 | 预期 |
| --- | --- | --- |
| Light 匹配 Light 卡 | 使用 Light 攻击消耗 RequiredAction 为 Light 的卡 | 触发 BaseFlow + MatchedFlow |
| Heavy 不匹配 Light 卡 | 使用 Heavy 攻击消耗 RequiredAction 为 Light 的卡 | 卡牌消耗，只触发 BaseFlow |
| Any 卡 | 使用 Light 或 Heavy 消耗 RequiredAction 为 Any 的卡 | 都算匹配 |
| 无卡攻击 | 洗牌期间继续攻击 | 攻击照常播放，不触发卡牌 |
| 终结技卡 | 在 Combo Finisher 上消耗 Finisher 卡 | 触发终结技效果 |

## 洗牌装弹

| 场景 | 操作 | 预期 |
| --- | --- | --- |
| 4 张卡打空 | 连续攻击 4 次 | 第 4 张消耗后进入洗牌 |
| UI 清空 | 进入洗牌 | 1D UI 清空或显示装填进度 |
| 冷却期间攻击 | 洗牌 1 秒内继续攻击 | 不触发卡牌，不阻止冷却 |
| 冷却结束 | 等待洗牌完成 | UI 按固定顺序回填 |
| 固定顺序 | 多轮洗牌 | 每轮顺序一致 |

## DashSave

| 场景 | 操作 | 预期 |
| --- | --- | --- |
| DashSave 保留连招 | 在允许窗口冲刺后继续攻击 | 连招段不被错误重置 |
| DashSave 保留卡牌上下文 | Link 卡前后发生 DashSave | Link 上下文按规则保留 |
| 洗牌中 DashSave | 洗牌期间冲刺并攻击 | 攻击照常，不触发卡牌 |

## 关卡奖励

| 场景 | 操作 | 预期 |
| --- | --- | --- |
| 清场奖励 | 清空房间 | 出现主奖励 |
| 选择奖励 | 三选一选择卡牌 | 奖励进入后台背包和当前卡组 |
| 不打开旧背包 | 选择奖励后 | 不自动打开 2D 背包 UI |
| 跳过奖励 | 清场后直接进门 | 允许进入下一关，但放弃成长 |

## 通过标准

- 以上核心场景均能在编辑器内复现。
- 不出现攻击被卡组系统阻止的问题。
- 不出现洗牌期间卡牌仍触发的问题。
- 不出现奖励选择后强制打开旧背包的问题。
