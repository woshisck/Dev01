# 策划需求与版本方案

本目录存放来自策划、版本目标和玩家反馈的当前需求，以及已经拆成可开发、可协作、可验收的版本方案。

## 常用入口

- [核心方案](核心方案/README.md)：当前版本主方案。
- [开发方案](核心方案/01_开发方案/)：程序/AI 可执行的拆分方案。
- [联调验收](核心方案/02_联调验收/)：跨系统验收标准。
- [协作接口](核心方案/03_协作接口/)：模块之间的接口契约。
- [系统需求](核心方案/04_系统需求/)：功能运行规则、数据结构、需求细分。
- [开发路线](核心方案/05_开发路线/)：按优先级推进的路线和配置清单。

## 使用规则

这里回答“当前版本要做什么、需求从哪里来、如何拆给程序和 AI”。已经被当前方案替代的旧版本需求，应进入 `99_归档/旧需求/`；明确与当前方向冲突的内容，应进入 `98_废弃/`。
