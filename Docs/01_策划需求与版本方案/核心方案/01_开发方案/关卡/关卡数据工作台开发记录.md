# 关卡数据工作台开发记录

## 当前定位

关卡编辑器不是路线编辑器，也不替换现有运行时数据。它是一个面向策划的关卡数据工作台，集中编辑当前已有的 `RoomDataAsset` 和 `CampaignDataAsset`。

## 当前已实现

- 新增 `Level Data Workbench` 编辑器窗口。
- 菜单入口：`Tools -> DevKit Tools -> Level Data Workbench`。
- 左侧列表：扫描并展示所有 `RoomDataAsset`。
- 中间细则：选中房间后展示该 `RoomDataAsset` 的 Details 面板，可直接编辑敌人池、奖励池、Buff、难度档、商店、事件、传送门配置等现有字段。
- 右侧全局流程：扫描并展示所有 `CampaignDataAsset`，选中后展示其 Details 面板，可编辑 `FloorTable`、`RoomPool`、`DefaultStartingRoom`、`LayerTag` 等全局流程字段。
- `FFloorConfig` 增加全局事件接口字段：
  - `GlobalStageTag`
  - `StoryEventTags`

## 数据边界

继续使用现有 DA：

```text
CampaignDataAsset
RoomDataAsset
RoomDataAsset.PortalDestinations
RoomDataAsset.EnemyPool
RoomDataAsset.LootPool
RoomDataAsset.BuffPool
```

当前没有新增运行时路线图资产，也没有改变 `YogGameMode` 的关卡选择主逻辑。

## 教程 / 故事接口方向

教程和故事事件系统不绑定具体房间细则，只监听全局流程接口。

建议使用：

```text
GlobalStageTag:
- Level.Stage.Hub
- Level.Stage.FirstCombat
- Level.Stage.RewardChoice
- Level.Stage.ResourceChoice

StoryEventTags:
- Story.Enter.MainHub
- Tutorial.WeaponSelect.Enabled
- Tutorial.CardConsume.Enabled
- Tutorial.RewardToDeck.Enabled
- Tutorial.LinkCard.Enabled
```

后续独立制作 `Story / Tutorial Event Editor`，监听这些接口后触发：

- 教程弹窗
- HUD 高亮
- 镜头引导
- 状态推进
- 故事事件
