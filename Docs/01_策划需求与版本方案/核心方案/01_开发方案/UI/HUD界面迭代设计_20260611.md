# HUD 界面迭代设计（2026-06-11）

## 目标

本次 HUD 调整服务于玩法完善后的战斗阅读性：

- 右下资源区区分“常驻资源”和“临时拾取反馈”。
- 玩家血条上方显示当前玩家 Buff，参考 Dota 技能栏上方的持续效果行。
- 在缺少正式 UI 图片时先使用占位符，后续用生成图像替换。
- 保持暗黑哥特 stylized-realism 的 UI 气质，避免遮挡战斗主体。

## HUD 布局

### 右下资源区

`PlayerCommonInfoHud` 继续挂在 `WBP_HUDRoot.BottomRightPlayerInfoRegion`。

显示规则：

- 金币行永久显示。
- 局外提升素材默认隐藏。
- 当素材数量增加时，素材行显示 8 秒，然后自动隐藏。
- 如果 8 秒内再次获得素材，显示计时刷新。
- 消耗或读档导致素材数变化时更新数值，但不强制长时间显示；负变化可短暂显示 3 秒用于确认。

布局建议：

- 底部为金币常驻条。
- 金币上方为临时资源提示堆叠区。
- 资源提示使用小型黑铁/玻璃框，最多显示 2-3 行，避免右下角过密。

### 玩家血条和 Buff 行

在 `BottomLeftPlayerInfoRegion` 中，玩家血条上方新增 `PlayerBuffBar`。

显示规则：

- 当前玩家可见 Buff 以 32x32 方框横向排列。
- 初版没有正式图标时使用占位方框。
- 每个格子预留冷却/剩余时间遮罩能力。
- Buff 行随血条左边缘对齐，位于血条上方 6-10 px。
- 最大显示 10 个，超出时合并为 `+N` 占位格。

数据范围：

- 读取玩家 ASC 的 `Buff.Status.*` 状态。
- 过滤技术性或已有独立表现的 Tag，例如 `Buff.Status.Dead`、`Buff.Status.HitReact`、`Buff.Status.HitStop.*`、`Buff.Status.Heat.Phase.*`。
- 优先展示对玩家决策有意义的状态，例如 `Poisoned`、`Bleeding`、`Burning`、`Shielded`、`SuperArmor`、`OilBlade`、`InSmoke`、`FinisherCharge`、`FinisherQTEOpen`、`DuoAssaultReady`、`NextHitCrit`。
- 第一版仅显示存在/不存在，不读取剩余时间；倒计时遮罩在图像资源和 Active Gameplay Effect 剩余时间读取稳定后再接入。

### 切换提示

参考图中的黄色箭头表达“切换关系”。切换反馈需要表现前后关系交替，而不是静态箭头。

处理原则：

- 在装备、主动技能、卡组方向等真实可切换模块中，用“滚筒式前后交替”动效表达。
- 前景 UI 表示当前生效项，亮度和饱和度保持正常。
- 背景 UI 表示备用项或下一项，整体变暗，透明度降低。
- 触发切换时，背景项沿短弧线或纵向滚筒轨迹移动到前景，前景项退到背景。
- 切换动效时长控制在 0.25-0.35 秒，动效结束后只保留前景和暗化背景，不保留大面积箭头。
- 常规战斗画面不保留手绘式大箭头，避免遮挡场景。

动效规格：

- 前景项：RenderScale 1.0，RenderOpacity 1.0，色彩正常。
- 背景项：RenderScale 0.78-0.86，RenderOpacity 0.45-0.60，叠加暗色遮罩。
- 切换过程中：新前景项从后层向前滚入，Scale 从 0.84 到 1.0，Opacity 从 0.55 到 1.0。
- 旧前景项退到后层，Scale 从 1.0 到 0.84，Opacity 从 1.0 到 0.5。
- 若模块有两个以上可切换项，只展示当前项和下一项；其余项不常驻显示。

## 占位符策略

在正式图片缺失前：

- Buff 图标使用 32x32 半透明暗色方框，带冷青描边。
- 临时资源框使用简化边框和图标位。
- 倒计时遮罩先不做复杂材质，用简单透明度或 ProgressBar 占位。
- 所有占位控件保留稳定命名，方便后续替换素材而不改 C++ 绑定。

## 图像放置约定

请将生成后的源 PNG 放入以下目录。后续实现会按这些固定文件名读取或导入。

| 用途 | 源文件路径 |
| --- | --- |
| Buff 图标占位框 | `SourceArt/UI/HUD/Buff/T_HUD_BuffIconFrame_32.png` |
| Buff 剩余时间遮罩 | `SourceArt/UI/HUD/Buff/T_HUD_BuffDurationMask_32.png` |
| 资源拾取临时提示框 | `SourceArt/UI/HUD/Resource/T_HUD_ResourceToastFrame.png` |
| 金币常驻计数框 | `SourceArt/UI/HUD/Resource/T_HUD_GoldCounterFrame.png` |
| 切换滚筒前景框 | `SourceArt/UI/HUD/SwitchRoller/T_HUD_SwitchRoller_FrontFrame.png` |
| 切换滚筒背景框 | `SourceArt/UI/HUD/SwitchRoller/T_HUD_SwitchRoller_BackFrame.png` |
| 切换滚筒运动拖影 | `SourceArt/UI/HUD/SwitchRoller/T_HUD_SwitchRoller_MotionStreak.png` |

命名规则：

- 文件名以 `T_HUD_` 开头。
- 源文件使用透明背景 PNG。
- 框体图片不要带文字和数字。
- 32x32 小图只用于 Buff；资源和切换框可以使用更高分辨率，导入后由 UMG 控制显示尺寸。

## 验收标准

- 金币在右下角永久可见。
- 局外素材默认不可见；获得素材后显示 8 秒，再自动隐藏。
- 玩家血条上方出现 Buff 占位方框行。
- 切换模块播放前后交替的滚筒动效，前景项变为当前项，后景项暗化。
- Buff 行不会遮挡角色脚下、场景中心或底部战斗主体。
- `MainUISetupCommandlet` 重新生成 `WBP_HUDRoot` 后，新增控件命名稳定。
- 自动化测试覆盖 HUD 根布局中的玩家血条、资源区和 Buff 行基础挂载。
