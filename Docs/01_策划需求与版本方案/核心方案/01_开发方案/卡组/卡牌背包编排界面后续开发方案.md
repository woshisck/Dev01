# 卡牌背包编排界面后续开发方案

## 背景

当前打开的是 2D 符文背包，主要服务符文占格、旋转、放置和出售。它不能满足战斗卡牌编排需求：

- 调整战斗卡牌顺序
- 查看卡牌之间的连携关系
- 对单张连携卡切换 `Forward / Reversed`

## 目标

新增一个“卡牌编排界面”，与符文 2D 背包分开：

| 功能 | 说明 |
| --- | --- |
| 卡牌列表 | 显示当前武器/背包生成的 Combat Deck |
| 顺序调整 | 支持拖拽或按钮上移/下移 |
| 连携反转 | 对 `CardType=Link` 的卡牌切换 `LinkOrientation` |
| 配方预览 | 显示该卡可触发的 `LinkRecipes` |
| 保存结果 | 战斗系统读取玩家调整后的顺序和方向 |

## 建议实现

1. 新增 `CombatDeckEditWidget` 作为卡牌编排界面。
2. 新增单卡槽 `CombatDeckEditCardSlotWidget`。
3. `CombatDeckComponent` 增加编辑接口：
   - 获取当前完整卡组顺序
   - 移动卡牌位置
   - 设置某张卡的 `LinkOrientation`
   - 保存并刷新战斗用 ActiveSequence
4. HUD 或背包界面新增入口：从 2D 背包切到卡牌编排页。

## 本阶段不做

- 不把 2D 符文格子系统改造成卡牌排序界面。
- 不做三连携和终结技专用 UI。
- 不做复杂自动推荐连携。

## 验收

1. 玩家能看到当前卡牌顺序。
2. 玩家能调整卡牌顺序。
3. 玩家能把月光从正向切为反向。
4. 战斗中读取调整后的顺序和方向。
