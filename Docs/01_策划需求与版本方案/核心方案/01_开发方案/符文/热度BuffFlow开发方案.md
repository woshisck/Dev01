# Codex-热度 BuffFlow 组开发方案

## 组定位

本组负责热度系统的新定位，以及卡牌效果通过 BuffFlow 执行的衔接。核心方案中，热度不再控制背包激活区，而是强化高质量战斗行为。

## 负责范围

| 模块 | 说明 |
| --- | --- |
| 热度来源 | 命中、连招终结、完美闪避、打断、连携成功 |
| 表现强度参数 | 根据热度等级调整特效、音效、震动或倍率 |
| 数据配置 | 热度倍率和 Flow 参数可配置 |

## 不负责范围

- 不再做背包激活区扩展。
- 不决定卡牌消耗顺序。
- 不实现轻重攻击输入。
- 不生成关卡奖励。
- 不制作 UI 布局。

## 热度新规则

| 行为 | 热度处理 |
| --- | --- |
| 普通命中 | 少量获得或维持热度 |
| 连招终结命中 | 明显获得热度 |
| 连携成功 | 获得热度，并可被当前热度强化 |
| 完美闪避 | 获得热度或短时强化下一张卡 |
| 打断敌人 | 获得热度 |
| 长时间未命中 | 热度自然下降 |

## 与卡牌 Flow 的关系

卡牌组只决定触发哪个 Flow，热度 BuffFlow 组决定 Flow 的具体效果和倍率。

## 开发步骤

1. 梳理现有 BuffFlow 架构和热度系统入口。
2. 移除或停用“热度扩展背包激活区”的前台规则。
4. 定义 BaseFlow 和 MatchedFlow 的执行入口。
6. 接入战斗侧高质量行为：连招终结、完美闪避、打断。
7. 给 UI 表现组提供热度强化原因和强度参数。

## 验收标准

- 热度不影响卡牌是否生效。
- 连携成功能提升或消耗热度收益。
- 玩家不会因为热度不足而失去基础卡牌效果。
