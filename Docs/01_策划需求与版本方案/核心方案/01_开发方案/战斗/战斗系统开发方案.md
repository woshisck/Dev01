# Codex-战斗系统组开发方案

## 组定位

本组负责玩家攻击动作、轻重攻击序列、连招终结、DashSave、攻击事件派发。战斗系统组不决定卡牌是否触发，只负责把正确的攻击事件传给卡组系统。

## 负责范围

| 模块 | 说明 |
| --- | --- |
| Light / Heavy 输入 | 区分轻攻击和重攻击 |
| 攻击段配置 | 每段攻击知道自己的动作类型和是否为终结击 |
| Combo Finisher | 每条基础连招最后一击标记为终结击 |
| DashSave | 在允许窗口内冲刺后保留一次连招和卡牌上下文 |
| 攻击事件派发 | 在命中或出招确认时调用 `ResolveAttackCard` |
| 冲刺打断规则 | 冲刺不随时打断攻击前摇，只在后摇、桥接窗口、DashSave 窗口衔接 |

## 不负责范围

- 不维护 DeckList、ActiveSequence、CurrentIndex。
- 不判断卡牌动作是否匹配。
- 不执行卡牌 Flow。
- 不把奖励写入背包。
- 不直接控制卡组 UI。

## 与卡组组的接口

战斗侧在稳定时机调用：

```cpp
ResolveAttackCard(ActionType, bIsComboFinisher, bFromDashSave);
```

建议第一版调用时机优先使用“攻击命中时”。如果后续希望空挥也消耗卡牌，再调整为“出招确认时”。

## 开发步骤

1. 确认现有轻攻击和重攻击 Gameplay Ability 或攻击流程。
2. 给每段攻击补充 ActionType：Light / Heavy。
3. 给每条基础连招最后一击补充 Combo Finisher 标记。
4. 在攻击命中事件里组装卡组接口参数。
5. 调用 `ResolveAttackCard` 并读取返回结果。
6. 根据返回结果播放基础卡牌、匹配、连携、终结、洗牌开始的战斗反馈。
7. 调整冲刺规则，禁止随时打断攻击前摇。
8. 实现 DashSave 窗口内的连招保留。
9. DashSave 后攻击调用接口时传 `bFromDashSave = true`。

## 验收标准

- Light 攻击传入 `ActionType = Light`。
- Heavy 攻击传入 `ActionType = Heavy`。
- 连招最后一击传入 `bIsComboFinisher = true`。
- DashSave 后的攻击传入 `bFromDashSave = true`。
- 洗牌期间攻击不会被卡组系统阻止。
- 非 DashSave 窗口冲刺不能随意取消攻击前摇。
