# 卡牌数据结构

## 目标

第一版卡牌数据要足够表达攻击触发、普通卡全局倍率、少数机制连携，同时不要把旧 2D 背包的复杂空间规则带回来。本阶段不扩展三连携和终结技。

## 建议枚举

```cpp
enum class ECombatCardType
{
    Attack,
    Link,
    Finisher,
    Passive
};

enum class ECardRequiredAction
{
    Light,
    Heavy,
    Any
};

enum class ECardLinkMode
{
    None,
    ReadPrevious,
    PassToNext
};

enum class ECombatCardLinkDirection
{
    None,
    ForwardReadPrevious,
    BackwardEmpowerNext,
    Both
};

enum class ECombatLinkBreakPolicy
{
    ReleaseBaseFlow,
    ReleaseBreakFlow,
    Fizzle
};

enum class EDeckState
{
    Ready,
    EmptyShuffling
};
```

## 卡牌配置字段

| 字段 | 说明 |
| --- | --- |
| `CardType` | Attack / Link / Passive；Finisher 本阶段暂不扩展 |
| `RequiredAction` | 旧字段；普通卡建议填 Any，后续只作为少数连携条件 |
| `BaseFlow` | 卡牌正常释放效果，只要有效出牌就触发 |
| `MatchedFlow` | 旧字段；不再作为普通卡动作匹配奖励，后续拆分为连携 Flow |
| `LinkConfig` | 新字段建议；表达正向连携、反向赋能和断链释放；反向赋能使用单独配置的新 Flow |
| `GlobalComboMultiplierConfig` | 新全局配置建议；普通卡根据连击段数读取倍率，不在每张卡里单独配置 |
| `LinkMode` | 旧字段；兼容已有资产，不足以表达新连携 |
| `FinisherRequirement` | 本阶段暂不扩展，终结技后续单独整理 |
| `DisplayName` | UI 显示名称 |
| `HUDSummaryText` | 紧凑 HUD 摘要，建议 1-2 行 |
| `HUDReasonText` | 连携触发时显示给玩家的原因 |

## 运行时实例

运行时建议使用 `FCombatCardInstance`，不要直接修改 DataAsset。

| 字段 | 说明 |
| --- | --- |
| `InstanceGuid` | 本张卡的运行时唯一标识 |
| `SourceData` | 来源卡牌 DataAsset |
| `Level` | 当前等级 |
| `OwnerSource` | 来源：初始武器、关卡奖励、商店、事件 |
| `RuntimeState` | 当前运行时状态 |

## 第一版限制

- 不做随机洗牌。
- 不做 2D 格子旋转、裁剪、拼接。
- Passive 卡可以先进入数据结构，但第一版可暂不作为主战斗触发重点。
- Link 卡只做少数机制卡，不再要求所有卡牌都拥有泛化连携。
- 不做三连携和终结技。

## 验收标准

- 普通卡能表达“攻击出牌即触发 BaseFlow”。
- 普通卡能根据连击数或连招阶段读取全局倍率。
- Link 卡能声明正向读取上一张。
- Link 卡能声明反向赋能下一张，并配置独立的新效果 Flow。
- Link 卡能声明断链时释放 BaseFlow、BreakFlow 或失效。
