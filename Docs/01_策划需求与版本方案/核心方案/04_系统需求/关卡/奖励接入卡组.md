# 奖励接入卡组

## 目标

关卡奖励要服务新的战斗卡组循环。玩家清场后选择奖励，奖励自动进入后台背包与当前卡组，不再强迫玩家打开旧 2D 背包整理。

## 奖励流程

```text
清场 -> 生成主奖励 -> 三选一 -> 玩家选择 -> 写入后台背包 -> 更新 DeckList -> 刷新 1D 卡组
```

## 第一版规则

| 规则 | 说明 |
| --- | --- |
| 清场后生成主奖励 | 保留当前清场奖励节奏 |
| 三选一 | 仍使用卡牌或符文奖励选择 |
| 自动加入后台背包 | 玩家选择后系统自动记录 |
| 自动进入卡组 | 第一版默认加入当前卡组末尾，减少整理负担 |
| 不打开旧背包 | 选择完成后直接回到关卡节奏 |
| 可跳过奖励 | 玩家可以直接进门，但等于主动放弃成长 |

## 与旧系统的关系

可以继续复用：

- `RewardPickup`
- `LootSelectionWidget`
- 后台背包持有数据
- 当前关卡清场事件

需要调整：

- SelectLoot 后不自动打开旧背包。
- 奖励选择完成后通知 CombatDeckComponent 更新 DeckList。
- UI 提示“已加入卡组”，而不是引导玩家整理背包。

## 验收标准

- 清场后能出现主奖励。
- 玩家选择奖励后，该奖励进入后台背包。
- 玩家选择奖励后，该奖励进入当前卡组或待加入队列。
- 选择奖励后不自动打开旧背包 UI。
- 玩家不拾取奖励直接进门时流程不阻塞。
