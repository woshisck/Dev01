受限条目清单

1. WeaponSkill 当前语义仍与 Dash fallback 混在一起

优先级 P0 需要代码决策

当前输入链路是：按 WeaponSkill 先尝试武器 ComboGraph 的 WeaponSkill 节点，失败后回退到 PlayerState.AbilityCast.Dash。

阻塞点

图驱动的 WeaponSkill 在运行时目前映射为 Dash / Catalyst，而设计目标里“战技”更像 WeaponSkill / Finisher。

需要解决

明确战技是否就是终结/引爆。如果是，需要把 graph-based weapon skill 改为 WeaponSkill / Finisher，只让 fallback dash 保持 Dash / Catalyst。

资产需求

武器 ComboGraph 需要有明确 WeaponSkill root；战技节点需要有效 montage 或 montage config。

2. 重型武器变闪避还缺武器分类与 Dodge 能力

优先级 P0 需要数据结构 需要资产

设计目标是“无武器默认冲刺，携带重型武器变为闪避，或者由武器覆盖玩家移动技能”。当前代码还不能完整表达这个规则。

阻塞点

EWeaponType 目前只有 Melee 和 Ranged，没有 Heavy；项目里也没有独立 Dodge GA 和 PlayerState.AbilityCast.Dodge 路径。

需要解决

在 WeaponDefinition 增加移动覆盖配置，例如 DefaultDash、Dodge、OverrideAbility。

资产需求

Dodge GA、Dodge montage、Dodge gameplay tag、重型武器 DA 的移动覆盖配置。

3. 后摇取消强化需要动画窗口和标签协议

优先级 P1 需要动画配置

当前 SpecialAttack 和普通攻击之间已经有 CanCombo 连接基础，但“后摇阶段取消获得强化”还没有完整协议。

阻塞点

缺少统一的 recovery cancel window tag、cancel performed tag、cancel bonus active tag。

需要解决

定义 montage notify window；在技能、战技、冲刺取消攻击后摇时写入状态；BuffFlow 根据状态给予强化。

资产需求

攻击 montage 的后摇窗口 notify、取消强化 BuffFlow、取消成功的反馈音效/VFX。

4. 卡组 UI 还没有表现四类槽位

优先级 P1 需要 UI

代码已经拆出 Attack 顺序卡组和三个单槽，但 UI 仍需要把这些槽位清晰展示给玩家。

阻塞点

现有 UI 多数仍围绕 full deck / remaining deck 展示，缺少 Skill、WeaponSkill、Dash 单槽的装备与替换交互。

需要解决

用 GetActionSlotCardSnapshot() 支持 UI 展示，并增加替换/清空单槽的交互接口。

资产需求

槽位图标、卡面状态、攻击顺序编号、单槽锁定/空槽视觉。

5. 双武器热度切换是独立大系统

优先级 P2 建议暂缓

双武器、热度条、爆豆打断、弹反失衡、CD 重置、两套卡组不共用，已经超出当前输入和卡组槽位重构的范围。

阻塞点

缺少第二武器槽、每把武器独立 deck 存储、热度资源和切换能力。

需要解决

先作为单独 feature 做数据模型和能力链路，再接入当前 CombatDeck slot 体系。

资产需求

双武器 UI、热度条 UI、切换 montage、弹反/失衡 GE、命中反馈和切换 VFX。

建议下一步

  1. 先在编辑器里验证无武器 NormalAttack 不再触发 dash，WeaponSkill 仍能触发 dash fallback。
  2. 给两手剑测试资产补齐 NormalAttack root 与 WeaponSkill root，并确认每个节点有 montage/montage config。
  3. 决定 graph-based weapon skill 的 deck context 是否改成 WeaponSkill / Finisher。
  4. 配置四张最小验证卡：Attack / Starter、Attack / Catalyst、Skill / Catalyst、WeaponSkill / Finisher。
  5. 确认四槽 UI 方案后，再做单槽替换/清空接口。
  6. 重武器 Dodge 和双武器热度切换作为后续独立任务拆分。
