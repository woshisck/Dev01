# 献祭符文被动配置说明

## 1. 作用

献祭符文是全局被动，不进入战斗卡组，也不出现在背包拖拽卡牌列表里。

当前实现了三张默认献祭符文：

| 符文 | 效果 |
| --- | --- |
| 月光之影 | 冲刺后在原地留下暗影。暗影不移动，会跟随玩家攻击动作姿态，并同步玩家本次打出的攻击型卡牌效果。默认 4 次攻击后消失。 |
| 暗影印记 | 冲刺次数 +1。冲刺路径上的敌人获得暗影印记，之后被玩家攻击会引爆，造成范围伤害。 |
| 巨力挥舞 | 玩家所有攻击都会击退敌人。敌人被击退过程中碰到其他单位时，对被碰到的单位造成碰撞伤害。 |

## 2. 资产位置

| 资产 | 路径 |
| --- | --- |
| 月光之影 DA | `/Game/Docs/BuffDocs/V2-RuneCard/512Generated/Sacrifice/DA_Rune512_Sacrifice_MoonlightShadow` |
| 暗影印记 DA | `/Game/Docs/BuffDocs/V2-RuneCard/512Generated/Sacrifice/DA_Rune512_Sacrifice_ShadowMark` |
| 巨力挥舞 DA | `/Game/Docs/BuffDocs/V2-RuneCard/512Generated/Sacrifice/DA_Rune512_Sacrifice_GiantSwing` |
| Flow 目录 | `/Game/Docs/BuffDocs/V2-RuneCard/512Generated/Sacrifice/Flow` |

## 3. DA 配置规则

献祭符文 DA 需要保持以下规则：

| 字段 | 值 |
| --- | --- |
| `Rune Config -> Trigger Type` | `Passive` |
| `Shape -> Cells` | 空，不占背包格子 |
| `Flow -> Flow Asset` | 对应 `FA_Rune512_Sacrifice_*` |
| `Combat Card -> Is Combat Card` | 不勾选 |
| `Combat Card -> Base Flow / Link Recipes` | 不填写 |

投放位置：放入祭坛/房间奖励/全局被动奖励池，不要放入武器 `InitialCombatDeck`。

## 4. FA 节点配置

献祭符文 Flow 使用同一个节点：

```text
Start
  -> Grant Sacrifice Passive
```

### 月光之影

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `Passive Type` | `Moonlight Shadow` | 启用月光之影 |
| `Shadow Attack Charges` | `4` | 暗影同步攻击次数，归零后消失 |
| `Shadow Lifetime` | `10` | 暗影最长存在时间 |
| `Shadow Damage Multiplier` | `0.45` | 暗影追加伤害 = 玩家本次伤害 * 该倍率 |
| `Shadow Attack Range` | `380` | 暗影追加伤害检测半径 |
| `Shadow Attack Cone Degrees` | `90` | 暗影前方攻击角度 |

当前暗影会同步带有攻击生成性质的卡牌 Flow，例如月光刃、地面路径等。纯数值 Buff 类卡牌不额外复制表现。

### 暗影印记

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `Passive Type` | `Shadow Mark` | 启用暗影印记 |
| `Dash Charge Bonus` | `1` | 最大冲刺次数增加值 |
| `Dash Mark Radius` | `95` | 冲刺路径左右判定宽度 |
| `Mark Duration` | `8` | 印记保留时间 |
| `Mark Explosion Damage` | `35` | 引爆伤害 |
| `Mark Explosion Radius` | `260` | 引爆范围 |
| `Shadow Mark Tag` | `Buff.Status.ShadowMark` | 印记状态 Tag |

### 巨力挥舞

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `Passive Type` | `Giant Swing` | 启用巨力挥舞 |
| `Knockback Distance` | `520` | 攻击触发的击退距离 |
| `Knockback Collision Damage` | `22` | 击退碰撞伤害 |
| `Knockback Collision Radius` | `120` | 碰撞检测半径 |
| `Knockback Collision Duration` | `0.35` | 击退后检测碰撞的持续时间 |
| `Knockback Collision Tick Interval` | `0.05` | 碰撞检测频率 |

## 5. 引擎配置顺序

1. 运行或确认批量生成器已生成上述三个 DA 和 FA。
2. 打开三个 `FA_Rune512_Sacrifice_*`，确认只有 `Start -> Grant Sacrifice Passive`。
3. 打开三个 DA，确认 `Shape.Cells` 为空，`Is Combat Card` 未勾选。
4. 将 DA 放入祭坛或房间奖励池。
5. 不要把它们放进武器卡组，否则会破坏“全局被动”定位。

## 6. 验收测试

1. 获得献祭符文后，背包战斗卡组列表不新增卡牌。
2. 月光之影：冲刺后原地出现暗影；玩家接下来打出月光等攻击卡时，暗影位置也会同步释放对应攻击表现；4 次攻击后消失。
3. 暗影印记：最大冲刺次数 +1；冲刺穿过敌人后敌人带 `Buff.Status.ShadowMark`；攻击该敌人后印记消失并造成范围伤害。
4. 巨力挥舞：玩家攻击命中敌人后触发击退；被击退敌人碰到其他敌人时产生一次碰撞伤害。
5. 切换武器或移动卡组时，这三张献祭符文不参与卡组排序和卡牌消耗。
