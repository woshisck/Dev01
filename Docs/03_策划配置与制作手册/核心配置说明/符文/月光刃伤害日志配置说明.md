# 月光刃伤害日志配置说明

## 1. 作用
说明 `Spawn Slash Wave Projectile` 节点中的伤害和日志字段。月光刃可以直接在 FA 里配置伤害，不需要额外写 GA。

## 2. FA节点字段
| 字段 | 推荐填写 | 说明 |
| --- | --- | --- |
| `Damage Effect` | `GE_SlashWaveDamage` | 推荐使用现有 GE，GE 内用 `Attribute.ActDamage` SetByCaller 接收伤害值 |
| `Damage` | 例如 `10` / `20` | 月光刃命中敌人时造成的伤害 |
| `DamageLogType` | 见下表 | 伤害日志显示的来源类型 |

如果暂时没有 `GE_SlashWaveDamage`，`Damage Effect` 可以留空。代码会创建临时 Instant GE，把 `Damage` 数值作为伤害打到目标身上。

## 3. 月光推荐日志类型
| 月光效果 | `DamageLogType` |
| --- | --- |
| 普通月光 | `Rune_Moonlight_Base` |
| 正向连携月光 | `Rune_Moonlight_Forward` |
| 反向连携月光 | `Rune_Moonlight_Reversed` |

## 4. 是否可以绕过GE/GA
可以绕过 GA。月光刃已经由 FA 节点直接生成投射物并造成伤害。

不建议用普通 `Apply Attribute Modifier` 节点直接扣怪物血量替代月光刃伤害。月光刃需要投射物碰撞、命中数量、同一敌人只命中一次、飞行距离和寿命，这些逻辑都在 `Spawn Slash Wave Projectile` 和 `BP_SlashWaveProjectile` 中。
