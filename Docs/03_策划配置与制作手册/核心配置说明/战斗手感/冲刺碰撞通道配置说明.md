# 冲刺碰撞通道配置说明

## 作用
控制冲刺相关碰撞通道的项目默认响应。当前规则是：`DashTrace` 和 `DashThrough` 默认都不阻挡任何对象，由具体角色、场景物体或蓝图按需要单独开启响应。

## 配置位置
项目配置文件：

```text
Config/DefaultEngine.ini
```

配置段：

```text
[/Script/Engine.CollisionProfile]
```

当前默认值：

| 通道 | 类型 | 默认响应 |
| --- | --- | --- |
| `DashTrace` | Trace Channel | `Ignore` |
| `DashThrough` | Object Channel | `Ignore` |

## 配置思路
- `DashTrace` 用于冲刺检测时，默认不让新物体自动阻挡冲刺检测。
- `DashThrough` 用于允许或限制冲刺穿过对象时，默认不让新物体自动参与阻挡。
- 如果某个墙体、敌人、机关需要影响冲刺，应在该对象的 Collision 设置里单独把对应通道改成 `Block` 或 `Overlap`。

## 测试方式
1. 打开 Project Settings -> Engine -> Collision。
2. 确认 `DashTrace` 的 Default Response 是 `Ignore`。
3. 确认 `DashThrough` 的 Default Response 是 `Ignore`。
4. 进入 PIE，测试玩家冲刺不会被新建默认物体意外拦截。
5. 对需要拦截冲刺的物体单独设置 `DashTrace` 或 `DashThrough` 响应，再确认拦截逻辑生效。
