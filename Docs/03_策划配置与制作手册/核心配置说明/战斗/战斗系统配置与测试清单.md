# 战斗系统配置与测试清单

## 作用

用于汇总当前玩家动作、武器能力和战斗卡牌的制作与验收口径。旧连招树、轻/重攻击和消耗式卡组只作为兼容内容保留，新制作以本文和 `PlayerActionCombatDeck_Current.md` 为准。

## 需要配置的内容

| 类型 | 配置位置 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| 武器 Attack 能力数据 | `WeaponDefinition.AttackAbilityData` | 武器必须 | 配置普通攻击使用的 `UWeaponAttackAbilityMontageData` |
| 武器战技能力数据 | `WeaponDefinition.WeaponSkillAbilityData` | 有战技的武器必须 | 配置战技使用的 `UWeaponSkillAbilityMontageData` |
| 被动能力数据 | `WeaponDefinition.PassiveAbilityData` | 按需求 | 武器常驻效果和被动能力入口 |
| 蒙太奇配置 | `AbilityData.MontageConfigMap` | 必须 | Ability Tag 到 `MontageConfigDA` 的映射；`MontageMap` 只作旧内容回退 |
| 命中窗口 | `MontageConfigDA.Entries.Hit Detection Window` | 攻击类必须 | 决定什么时候开判定 |
| 攻击参数 | `MontageAttackDataAsset` | 攻击类必须 | 配伤害、距离、判定框、HitStop |
| Attack 卡组 | `RuneDataAsset.CombatCard.RequiredActionSlot = Attack` | 构筑必须 | 按卡组顺序结算，是主出牌链 |
| Skill 单槽卡 | `RequiredActionSlot = Skill` | 按需求 | 玩家主动技能触发，不推进 Attack 序列 |
| WeaponSkill 单槽卡 | `RequiredActionSlot = WeaponSkill` | 按需求 | 战技触发，默认承担 Finisher / 引爆职责 |
| Dash 单槽卡 | `RequiredActionSlot = Dash` | 按需求 | 冲刺/闪避触发，不推进 Attack 序列 |
| 流程角色 | `CombatCard.RequiredFlowRole` | 构筑推荐 | Starter / Catalyst / Finisher 表达构筑循环 |

## 当前配置流程

1. 在武器 DA 上配置 `AttackAbilityData` 和需要的 `WeaponSkillAbilityData`。
2. 在对应 AbilityData 的 `MontageConfigMap` 里按 Ability Tag 配置蒙太奇。
3. 在 `MontageConfigDA` 上配置命中窗口，并引用 `MontageAttackDataAsset`。
4. 在卡牌 DA 上配置 `RequiredActionSlot` 和 `RequiredFlowRole`。
5. Attack 卡按卡组顺序摆放；Skill / WeaponSkill / Dash 每个动作只放一个单槽卡。
6. 常驻效果放 Passive Flow；需要顺序影响结果的效果放 Base / Matched / Link Flow。

## 需要测试的内容

| 测试项 | 验收标准 |
| --- | --- |
| Attack 顺序结算 | 连续攻击会按 `ActiveSequence` 与 `CurrentIndex` 依次触发卡牌 |
| 不移除卡牌 | 卡牌结算后仍保留在卡组，只推进顺序或进入洗牌 |
| 单槽不推进序列 | Skill / WeaponSkill / Dash 结算各自槽位卡，不推进 Attack 序列 |
| 流程角色 | Starter 上 Buff，Catalyst 提升 Buff，WeaponSkill / Finisher 引爆 |
| Link 连携 | 前后卡顺序、CardIdTag 或 FlowRole 条件能正确触发 Link |
| 共享冷却 | Skill 与 WeaponSkill 使用 `PlayerState.Cooldown.SkillShared` 互相锁 CD |
| 后摇切武器 | 攻击后摇阶段切武器触发 recovery cancel 奖励，并清理 Skill / WeaponSkill CD |
| 输入语义 | 玩家只有 Attack / Skill / WeaponSkill / Dash，不出现轻/重攻击切换 |
| 旧资产兼容 | 旧 `LightAttack` / `HeavyAttack` / `SpecialAttack` 资产能加载，但不作为新制作入口 |

## 自动化测试建议

- Attack 顺序卡结算与洗牌边界。
- Skill / WeaponSkill / Dash 单槽卡不推进 Attack 序列。
- `OnCardResolved` 广播与旧 `OnCardConsumed` 兼容广播。
- `RequiredActionSlot`、`RequiredFlowRole`、Link 条件匹配。
- Skill / WeaponSkill 共享冷却和后摇切武器 CD 清理。

## 注意事项

- 新文档、新卡牌说明里使用“结算卡牌”，不要写“消耗卡牌”。
- 新玩家动作说明里使用 Attack / Skill / WeaponSkill / Dash，不再写轻/重攻击分支。
- 旧 ComboGraph / WeaponComboConfig 仅用于旧资产兼容；当前主配置走 typed AbilityData。
- 战斗侧只负责输出动作槽、流程角色、卡牌顺序、Link 上下文和伤害事实；具体 Buff / Flow 逻辑由符文系统配置。
