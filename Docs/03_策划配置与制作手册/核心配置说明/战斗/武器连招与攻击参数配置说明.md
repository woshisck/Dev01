# 武器动作与攻击参数配置说明

## 作用

用于配置玩家武器的 Attack、WeaponSkill、命中窗口、攻击参数，以及这些动作给卡牌/符文系统提供的动作槽上下文。

旧 `GameplayAbilityComboGraph`、`WeaponComboConfigDA` 和轻/重攻击连招字段只保留兼容旧资产。当前新武器不要把它们当作主配置入口。

## 配置位置

| 配置对象 | 配置位置 |
| --- | --- |
| Attack 能力数据 | `WeaponDefinition.AttackAbilityData` |
| WeaponSkill 能力数据 | `WeaponDefinition.WeaponSkillAbilityData` |
| 被动能力数据 | `WeaponDefinition.PassiveAbilityData` |
| 蒙太奇配置 | `AbilityData.MontageConfigMap` |
| 旧蒙太奇回退 | `AbilityData.MontageMap` |
| 命中窗口 | `MontageConfigDA.Entries.Hit Detection Window` |
| 攻击伤害/距离/判定框/HitStop | `MontageAttackDataAsset` |
| 卡牌触发动作槽 | `RuneDataAsset.CombatCard.RequiredActionSlot` |

## WeaponDefinition

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `AttackAbilityData` | 普通攻击能力数据 | 新武器必须配置 |
| `WeaponSkillAbilityData` | 武器战技能力数据 | 有战技的武器必须配置 |
| `PassiveAbilityData` | 武器被动或反应数据 | 按需求配置 |
| `SpecialAbilityData` | 旧特殊攻击字段 | 不再用于新制作 |
| `GameplayAbilityComboGraph` / `WeaponSkillComboGraph` | 旧连招图引用 | Transient 兼容字段，不要新配 |

## AbilityData

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `MontageConfigMap` | Ability Tag 到一组 `MontageConfigDA` 候选的映射 | 当前主入口 |
| `MontageMap` | Ability Tag 到 `AnimMontage` 的直接映射 | 只用于旧内容回退 |
| `PassiveMap` | 被动能力数据 | 仅 `PassiveAbilityData` 使用 |

`UWeaponAttackAbilityMontageData` 用于 Attack，`UWeaponSkillAbilityMontageData` 用于 WeaponSkill。多个候选 `MontageConfigDA` 可以通过 RequiredTags / BlockedTags / Priority 选择。

## MontageConfigDA

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `Montage` | 实际播放的动画蒙太奇 | 必填 |
| `ConfigTags` | 蒙太奇配置标签，用于候选选择或调试 | 只填需要被规则读取的标签 |
| `TotalFrames` | 蒙太奇总帧数 | 按动画帧数填写 |
| `Entries` | 命中窗口、事件等配置 | 攻击类至少有一个命中窗口 |

`Entries` 常用类型：

| 类型 | 说明 | 推荐 |
| --- | --- | --- |
| `Hit Detection Window` | 命中判定窗口 | 每个攻击动作至少配置一次 |
| `Gameplay Event at Frame` | 单帧 GameplayEvent | 只在需要额外事件时配置 |
| `Combo Window` | 旧连招窗口 | 旧资产兼容；当前主设计不依赖连招续段 |

## MontageAttackDataAsset

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `EventTag` | 命中事件标签 | 按当前 GA/事件约定填写 |
| `ActDamage` | 攻击伤害 | 按武器、动作和构筑需求配置 |
| `ActRange` | 攻击距离 | 与实际动画位移匹配 |
| `ActResilience` | 韧性相关数值 | 按敌人受击规则配置 |
| `ActDmgReduce` | 减伤相关数值 | 默认 `0` |
| `HitboxTypes` | 判定框数组 | 普通近战至少配置一个 |
| `HitStopMode` | 命中停顿模式 | Attack 常用 `Freeze`，WeaponSkill / Finisher 可用更强反馈 |
| `OnHitEventTags` | 命中后抛出的事件标签 | 只填需要系统监听的标签 |
| `AdditionalRuneEffects` | 额外符文效果入口 | 当前战斗侧不配置具体效果 |

攻击参数优先级：

1. `Hit Detection Window.AttackDataCandidates` 选出的 `AttackData`
2. 旧 `AN_MeleeDamage` 自身字段

## 卡牌上下文

- Attack GA 结算 `ECombatDeckActionSlot::Attack`，会推进 Attack 序列。
- WeaponSkill GA 结算 `ECombatDeckActionSlot::WeaponSkill`，默认作为 Finisher / 引爆逻辑。
- Dash GA 结算 `ECombatDeckActionSlot::Dash`，不推进 Attack 序列。
- 主动 Skill 由 `UPlayerActiveSkillComponent` 结算 `ECombatDeckActionSlot::Skill`。
- `RequiredAction = Light/Heavy` 是旧字段兼容，新卡请用 `RequiredActionSlot` 和 `RequiredFlowRole` 表达条件。

## 验收方式

1. Attack 能播放对应蒙太奇，并在命中窗口内造成伤害。
2. Attack 每次出手会按卡组顺序结算下一张 Attack 卡。
3. WeaponSkill 能触发自己的蒙太奇或蓄力逻辑，并结算 WeaponSkill 单槽卡。
4. Skill / WeaponSkill 共享冷却，后摇切武器能清理两类 CD。
5. 卡牌结算后仍保留在卡组，不因结算被移除。
6. 旧武器若只配置 `MontageMap` 仍可回退播放，但新制作不继续扩展旧连招字段。

## 注意事项

- 新内容不要配置 `SpecialAbilityData`。
- 新内容不要依赖轻/重攻击分支表达构筑差异；用卡牌顺序、动作槽、流程角色和 Link 条件表达。
- 如果某个旧资产仍使用 ComboGraph，请先确认是否必须兼容；可运行迁移工具把内容同步到 AbilityData 后再继续制作。
- 符文 Flow、具体 Buff、Link 效果由符文系统配置文档说明；战斗侧只提供动作槽、AbilityTag、TriggerTiming 和卡牌上下文。
