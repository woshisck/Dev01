# 数据编辑器（DataEditor）

菜单路径：`DevKit → DataEditor`

---

## 一、定位

DataEditor 是**跨 DA 类型的批量查看与修改工具**，适合：
- 需要横向对比多个符文/角色/效果的数值
- 批量设置同类字段（如统一调整所有符文的稀有度）
- 将数据导出为 CSV 给策划在 Excel 中批注

单个符文的精细配置请用 **符文编辑器**，DataEditor 负责批量操作。

---

## 二、支持的资产类型

| 资产类型 | 说明 |
|---|---|
| `URuneDataAsset` | 所有符文 DA |
| `UEffectDataAsset` | 效果资产（DoT、Buff 等） |
| `UCharacterData` | 角色数据 |

---

## 三、主要功能

### 3.1 资产列表

打开后自动扫描项目内所有匹配资产并以列表显示，点击行可选中单个资产。

### 3.2 批量赋值

选中多行后，工具栏提供批量设置字段的操作（所有操作支持 Undo/Redo）：

| 字段 | 操作 |
|---|---|
| 金币成本 | 批量设为指定值 |
| 稀有度 | 批量设定 Rarity 枚举 |
| 触发类型 | 批量设定 TriggerType |
| 效果持续时间 | 批量设为秒数 |
| 最大堆叠数 | 批量设为整数 |

> 批量修改会自动标脏（MarkPackageDirty），操作后记得保存资产（Ctrl+S / File→Save All）。

### 3.3 迁移工具

- **RuneID → RuneIdTag 迁移**：从旧版 `int32` RuneID 迁移到 Gameplay Tag 系统
- 迁移会自动生成映射，操作不可逆，请提前备份

### 3.4 数据验证

工具栏的 **验证** 按钮会比对字段访问器和直接读取结果，报告数据语义不一致的资产。

### 3.5 导出 CSV

- **导出符文数据**：输出所有 `URuneDataAsset` 的关键字段
- **导出效果数据**：输出 `UEffectDataAsset` 的持续时间、周期、堆叠等字段
- 文件保存在项目根目录下，文件名为 `RuneExport_YYYYMMDD.csv`

---

## 四、使用流程

```
1. 打开 DevKit → DataEditor
2. 等待资产扫描完成（底部状态栏显示资产数量）
3. 在列表中框选需要批量修改的资产
4. 点击工具栏对应的批量操作按钮
5. 确认预览弹窗后执行
6. Ctrl+S 保存资产
```

---

## 五、注意事项

- **不要用 DataEditor 修改符文 Flow 数据**，Flow 只能在符文编辑器中编辑
- 批量操作执行后立刻 Ctrl+Z 可整体撤销，但关闭编辑器后无法撤销
- CSV 导出为快照，不会自动更新，修改后需重新导出
