# 战斗日志（Combat Log）

菜单路径：`DevKit → Combat Log`

---

## 一、定位

战斗日志实时记录 PIE 中每一次伤害事件，按类型分类统计，用于：
- 验证符文效果是否正确触发
- 检查伤害数值是否符合预期（倍率/连招加成）
- 对比不同卡组配置下的 DPS

---

## 二、界面区域

```
┌─────────────────────────────────────┐
│  过滤栏（类型 / 攻击者 / 目标 / 时间窗口） │
├─────────────────────────────────────┤
│  统计汇总（伤害合计 / 暴击率 / 各类分项）  │
├─────────────────────────────────────┤
│  事件列表（时间戳 / 来源 / 目标 / 数值）   │
└─────────────────────────────────────┘
```

---

## 三、伤害类型与颜色

| 类型 | 颜色 | 触发条件 |
|---|---|---|
| 普通 | 白色 | 基础近战/远程命中 |
| 暴击 | **黄色** | 触发暴击判定 |
| 符文 | **紫色** | 符文 Flow 产生的额外伤害 |
| 流血 | **红色** | DoT 持续伤害 |
| 卡牌 | **蓝色** | 战斗卡主动打出的伤害 |
| 终结技 | **金色** | Finisher 卡造成的伤害 |
| 连携 | **青色** | 连携倍率触发时附加记录 |
| 洗牌 | 灰色 | 洗牌机制触发的额外效果 |

---

## 四、过滤与筛选

| 过滤项 | 说明 |
|---|---|
| 类型过滤 | 勾选/取消对应类型，仅显示选中类型的条目 |
| 攻击者 | 下拉选择来源角色（空 = 全部） |
| 目标 | 下拉选择受击目标（空 = 全部） |
| 时间窗口 | 滑动条控制，默认 30 秒，最大 60 秒 |

---

## 五、统计汇总面板

| 统计项 | 说明 |
|---|---|
| 总伤害 | 时间窗口内所有伤害累加 |
| 暴击次数 / 暴击率 | 暴击命中次数与总命中的比值 |
| 符文伤害占比 | 符文伤害 / 总伤害 |
| 每秒伤害（DPS） | 总伤害 / 时间窗口秒数 |

---

## 六、调试符文的典型流程

```
1. 打开 Combat Log 窗口，保持浮动（不阻塞游戏视口）
2. PIE 进入游戏
3. 对敌人打出目标符文/卡组
4. 在日志中按 "符文" 类型过滤
5. 核对：
   - 符文是否有对应的伤害条目？
   - 连招第1/2/3段的伤害是否按数值表缩放？
   - 连携条目是否出现（倍率是否正确）？
6. 如有异常，回到符文编辑器修正 Flow 或数值表
```

---

## 七、注意事项

- 战斗日志使用**静态存储**，跨 PIE 会话保留（关闭 PIE 不清空，方便多次测试对比）
- 点击 **清空** 按钮手动清除历史条目
- Debug 模式（右上角切换）显示额外字段：伤害来源 AbilityTag、GE 实例 GUID、命中骨骼
- 时间戳精度为毫秒，适合检查 DoT 跳伤间隔是否符合 GE Period 配置
