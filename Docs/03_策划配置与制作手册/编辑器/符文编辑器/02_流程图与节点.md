# 流程图（Flow Graph）与节点使用说明

> 中间面板 → 切换到 **流程图** Tab  
> 底部面板 → 切换到 **节点库** Tab

---

## 一、流程图用途

Flow Graph 描述符文的**效果逻辑**：何时触发、触发后做什么、条件分支怎么走。所有运行时行为都在这里配置，与数值表里的 Key 配合使用。

---

## 二、节点库分类

| 分类 | 包含节点（举例） |
|---|---|
| **技能** | 触发伤害/暴击/击杀/冲刺/事件、WaitGameplayEvent |
| **效果** | 属性修改、施加状态（GE）、范围伤害、伤害计算 |
| **任务** | 搜索目标、结束符文（FinishBuff）、播放动画 |
| **生成** | 远程弹幕、路径效果、区域效果、飞行物配置 |
| **条件** | 属性比较、拥有 Tag、概率、只执行一次、距离判断 |
| **表现** | Niagara 特效、GameplayCue、序列帧 |
| **生命周期** | 延迟（Delay）、结束符文（FinishBuff） |

点击节点名 → 自动插入到当前流程图中。

## 三、常用节点一览

### 效果类

| 节点 | 用途 |
|---|---|
| **属性修改** `EffectAttributeModify` | 直接修改属性值（Attack / Health / MoveSpeed…），支持瞬发/持续/HasDuration |
| **施加状态** `EffectApplyState` | 施加 GameplayEffect（燃烧 GE / 中毒 GE / 诅咒 GE 等） |
| **范围伤害** `EffectAreaDamage` | 对范围内敌人造成伤害 |

### 生成类

| 节点 | 用途 |
|---|---|
| **远程弹幕** `SpawnRangedProjectiles` | 生成投射物，支持伤害/命中事件/多发/散射 |
| **地面路径效果** `SpawnRuneGroundPathEffect` | 生成地面区域（矩形/扇形），持续检测并施加 GE |
| **读取飞行物模块** `GetProjectileModule` | 从符文 DA 飞行物模块读取弹数/锥角/速度，输出数据引脚 |
| **读取光环模块** `GetAuraModule` | 从符文 DA 光环模块读取长宽高/时间/间隔，输出数据引脚 |

### 技能类

| 节点 | 用途 |
|---|---|
| **触发伤害事件** `OnDamageDealt` | 监听攻击命中时触发（Passive FA 内用） |
| **等待事件** `WaitGameplayEvent` | 等待指定 Gameplay Tag 事件（如命中回调） |
| **击杀触发** `OnKill` | 击杀目标时触发 |

### Pure 数据类（无执行引脚）

| 节点 | 用途 |
|---|---|
| **读取数值（Pure）** `BFNode_Pure_TuningValue` | 从数值表读取 Key 对应的值，输出 Float 数据引脚 |
| **连击段数（Pure）** `BFNode_Pure_ComboIndex` | 输出当前连击段数（Int），无执行引脚 |

## 四、数据引脚连线

节点属性中有两种输入字段：
- **普通字段**：直接填值
- **数据引脚**（圆形小点 ○）：可以连线从上游节点接收动态值

连线方式：
1. 在上游节点（如 Pure 节点）的输出引脚上**按住拖动**
2. 拖到下游节点的输入数据引脚上释放
3. 连线后节点属性面板里该字段显示为已连接状态

> 数据引脚连线优先级高于字段内填写的固定值。

## 五、常见 Flow 模板

### 被动持续型（如攻击力加成）

```
Entry → 属性修改(Attack + Value, Infinite) → 结束符文(可选)
```

### 命中触发型（如燃烧/中毒）

```
Entry → 触发伤害事件 → 施加状态(GE_Burn / GE_Poison)
                     → 播放特效
```

### 主动投射物型（如月光）

```
Entry ──→ 远程弹幕 ──→ [Out 不连]
       └──→ 等待事件(HitEvent) → 播放命中特效 → 结束符文
```

### 地面路径型（如燃烧地面）

```
Entry → 地面路径效果
          ├── Effect = GE_Burn
          ├── Duration ← 读取数值(Burn.Duration.01)
          └── SetByCallerValue ← 读取数值(Burn.Damage.01)
```

## 六、调试方式

1. 顶部工具栏 → **运行符文** → 底部切换到 **运行日志** 查看输出
2. 底部 **选中节点** Tab 查看当前选中节点的属性（只读）
3. 节点执行时 `LogTemp Warning` 会打印来源角色 / 数值 / GUID 等字段
4. 底部 **校验** Tab 查看连接错误和缺失配置警告

---

## 七、注意事项

- FA（FlowAsset）资产路径建议统一放在 `/Game/YogRuneEditor/Flows/`
- 不同符文的 FA 不应相互引用（除非明确设计为共用 LinkFlow）
- `FinishBuff` 节点终止 FA；Passive 符文通常不需要主动结束，FA 会在离开激活区时自动停止
- Pure 节点没有 `In` 执行引脚，不要试图给它连执行链
