# 卡牌设计准绳

## 目标

核心方案卡牌系统不追求无限法术拼装，而是追求可预测、可解释、能服务战斗爽感的组合。

玩家应当感到：

> 我能通过卡牌顺序和攻击选择打出更强效果。

制作侧必须保证：

> 所有效果都来自有限字段、有限效果形态和有限特征。

## 核心模型

```text
```

```text
状态 = 默认效果 + 标志性特征
```

- 默认效果：动作不匹配时也会执行。
- 标志性特征：只有动作匹配时才激活，并可被连携读取。

## 制作红线

1. 卡牌不能阻止玩家攻击。无卡、洗牌、动作不匹配时，攻击仍正常成立。
2. 保底效果不携带特征。动作不匹配时只执行默认效果。
3. 连携默认只读取上一张输出的 `TraitContext`，不读取完整历史。
4. 特征必须能被当前效果形态解释。无法解释时，降级为数值增强或默认效果。
6. 暂不考虑热度。后续接入时只强化数值、范围、表现，不改变触发资格。

## 判断标准

每个卡牌效果都必须能用一句话解释：

```text
因为上一张卡提供了【特征】，所以当前卡的【效果形态】发生了【可见变化】。
```

如果解释不清，就说明设计需要收窄。
