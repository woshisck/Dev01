# 512 教程弹窗与 HUD 提示文案

> 更新：2026-05-05
> 适用：`WBP_TutorialPopup`、`TutorialRegistryDA -> DialogContentDA`、卡组 HUD
> 目标：用少量暂停弹窗讲清核心规则，用战斗中 HUD 提示承接装填、消耗和连携反馈。

## 程序方案判断

当前方案可以继续使用，不需要重做教程 UI：

| 模块 | 判断 | 512 调整 |
|---|---|---|
| `TutorialRegistryDA -> DialogContentDA` | 好用，适合集中管理教程文案和多页弹窗。 | 保留兼容键，批量更新 512 文案。 |
| `LevelFlow -> ShowTutorial` | 适合初始房、整理期等安全时机的一次性教学。 | 战斗中不新增需要确认的中心弹窗。 |
| `TutorialManager` 状态机 | 可继续使用，但旧阶段语义需要改写。 | 不重排存档枚举，只改阶段含义和触发边界。 |
| 卡组 HUD | 已能显示洗牌、奖励入组、卡牌消耗等战斗反馈。 | 装填和连携原因优先由 HUD/短提示承载。 |

## 512 教程矩阵

| EventID | 触发时机 | 展示方式 | 文案 |
|---|---|---|---|
| `tutorial_weapon_pickup` | 初始房靠近第一把武器 | 暂停弹窗，2 页 | 标题 1：`拾起武器`；正文：`靠近发光武器，按 [E] 拾取。武器会装备到角色身上，并把它自带的初始卡牌装入下方 1D 卡组。` 标题 2：`武器自带初始卡牌`；正文：`每把武器都有一组起始卡。拾取后，卡组会按武器配置顺序显示；轻/重攻击不会被卡组阻止，但命中会按顺序消耗卡牌并触发效果。` |
| `tutorial_first_rune` | 首次三选一选择战斗卡牌后 | 暂停或整理期弹窗，1 页 | 标题：`新的卡牌已入组`；正文：`选择奖励后，它会进入后台背包，并追加到当前战斗卡组末尾。下一轮装填或下一房间，你会在 1D 卡组条里看到它。`；副文：`跳过奖励等于放弃这次成长。` |
| `tutorial_backpack` | 第一次打开背包/卡组编排界面 | 暂停弹窗，2 页 | 标题 1：`调整卡牌顺序`；正文：`整理阶段可以在卡组编排列表里拖动卡牌排序。放下后会触发一次短装填，HUD 会按新顺序刷新。`；副文：`鼠标拖动卡牌；手柄 A 拿起/放下，方向键移动位置。` 标题 2：`切换连携方向`；正文：`Link 卡可以正向读取前一张，也可以反向赋能下一张。选中 Link 卡按 R 或手柄左键切换方向。` |
| `tutorial_card_link` | 第一次进入卡组编排的连携说明，或首次成功触发连携后的安全时机 | 暂停弹窗或整理期弹窗，1 页 | 标题：`连携卡`；正文：`Link 卡会读取相邻卡牌的标签和方向。条件满足时，它会触发连携效果；HUD 会显示连携原因，让你知道为什么这次攻击被强化。`；副文：`例：月光放在不同方向，可以选择不同连携效果。` |
| `shuffle HUD hint` | 首次卡组打空进入洗牌 | HUD/卡组条短提示，不弹中心窗 | `卡组装填中：这段空窗仍可攻击，但暂时不会触发卡牌效果。` |

## 兼容键说明

| 旧键/阶段 | 512 含义 |
|---|---|
| `tutorial_weapon_pickup` | 武器拾取 + 武器初始 1D 卡组。 |
| `tutorial_first_rune` | 首次奖励战斗卡牌入组。保留名称兼容旧调用，不再表达旧符文成长。 |
| `tutorial_backpack` | 卡组排序 + Link 方向切换。 |
| `NeedHeatPhaseTutorial` | 作为首次连携教学阶段使用，不再教学热度激活区。 |
| `NeedPostCombatTutorial` | 旧存档兼容阶段，不再强制打开背包。 |

## 触发原则

- 初始武器、奖励入组、第一次卡组编排使用中心弹窗。
- 战斗命中瞬间不弹需要确认的中心教程。
- 洗牌/装填、奖励入组、卡牌消耗、连携原因优先放在卡组 HUD。
- 512 版不再教学旧“热度扩展激活区”；热度只作为连携/终结强化背景，由 HUD 表现说明。

## 资产生成

运行：

```powershell
UnrealEditor-Cmd.exe DevKit.uproject -run=Tutorial512Setup -DryRun
UnrealEditor-Cmd.exe DevKit.uproject -run=Tutorial512Setup -Apply
```

命令会创建/更新 `/Game/Docs/UI/Tutorial` 下的教程 `DialogContentDA`，并更新 `DA_TutorialRegistry`。
