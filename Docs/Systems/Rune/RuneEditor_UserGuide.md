## 二、创建符文

1. 在左上角填写 **名称**、**符文 Tag**（格式 `Rune.ID.xxx`）、**文件夹**。
2. 选择 **类型**（Buff / Debuff / Passive…）、**稀有度**、**触发方式**（Passive / Active…）。
3. 点击 **新建符文** → 编辑器自动在对应文件夹创建 `DA_Rune_xxx`。
4. 新符文默认没有 Flow 资产，需要在右侧 **基本信息 → 流程资产** 处关联一个已有的 `UYogRuneFlowAsset`（或复制模板）。

## 三、数值表（Value Table）

在中间面板切换到 **数值表** 标签。

### 3.1 添加数值行

点击 **添加数值行** → 在弹出行中填写：

| 字段 | 说明 |
|---|---|
| **Key** | Flow 图中引用的名称，全局唯一，建议格式 `符文.维度`，如 `Burn.Damage` |
| **Value** | 基础值（数字） |
| **Category** | 分组筛选用，如 `Damage` / `Stack` / `Duration` |
| **Display Name** | 策划可读名称，显示在编辑器中 |
| **Value Source** | Literal（直接用 Value）/ Formula（填表达式）/ MMC（用 C++ 类计算） |

### 3.2 连招加成（Combo Bonus）

每行可以开启 **连招模式**，让数值随连招段数缩放：

| 字段 | 说明 |
|---|---|
| **连招模式** | 不缩放 / 加算 / 乘算 |
| **每段奖励** | 每增加一段连招的增量 |
| **奖励上限** | 最大加成（0 = 无上限） |
| **取整** | 无 / 向下 / 四舍五入 / 向上 |

> **计算公式**：`ComboStacks = max(0, ComboIndex - 1)`  
> 加算：`Final = Base + min(Cap, Stacks × PerStack)`  
> 乘算：`Final = Base × (1 + min(Cap, Stacks × PerStack))`

筛选按钮 **「连招」** 只显示开启了连招加成的行，方便集中调整。

### 3.3 删除与导出

- 行末红色 **✕** 删除单行。  
- 工具条 **导出/导入** 可将数值表保存为 CSV 供外部对比。

## 四、流程图（Flow Graph）

在中间面板切换到 **流程图** 标签。

### 4.1 节点库（底部面板）

底部切换到 **节点库**，按分类筛选查找节点：

| 分类 | 说明 |
|---|---|
| 技能 | 流程控制、触发事件（伤害/暴击/击杀/冲刺/事件） |
| 效果节点 | 瞬时（伤害/治疗/消耗）/ 持续（属性修改）/ 状态（施加GE/Tag）/ 范围 |
| 任务节点 | 搜索目标 / 结束技能 / 播放动画 |
| 生成节点 | 投射物配置 / 区域配置 / 地面路径 / 远程弹幕 |
| 条件节点 | 属性比较 / 拥有Tag / 概率 / 只执行一次 / 距离判断 |
| 表现节点 | 特效 / GameplayCue / 序列帧 / VFX配置 |
| 生命周期 | 延迟 / 结束符文 |

点击节点名称 → 自动插入到当前 Flow 图中，并与 Entry 节点连接。

### 4.2 节点属性

选中节点后，底部切换到 **选中节点** 或在右侧 **详情面板** 查看属性。

### 4.3 常见 Flow 模板

**被动触发型（如燃烧）：**
```
Entry → TriggerDamageDealt → EffectApplyState(燃烧GE)
                           └→ PresentationPlayVFX
```

**主动投射物型（如月光）：**
```
Entry → SpawnProjectileProfile → [命中回调] → EffectDamage
                                            └→ EffectApplyState
```

**条件分支型：**
```
Entry → TriggerDamageDealt → ConditionHasTag(Tag.X)
                               ├─ Yes → EffectDamage(高伤害)
                               └─ No  → EffectDamage(基础伤害)
```

**定时 DoT 型（如中毒）：**
```
Entry → EffectApplyState(中毒GE, Periodic)
             └→ [GE 内部] Periodic Tick → 伤害
```

## 五、右侧详情面板

### 5.1 基本信息

填写符文的名称、Rune Tag、摘要文本、类型、稀有度、触发类型，以及关联的流程资产。填写完毕后点击 **保存基本信息**。

### 5.2 战斗卡（Combat Card）

若此符文需要参与连携系统，勾选 **启用战斗卡**。

| 字段 | 说明 |
|---|---|
| Card ID Tag | 格式 `Card.ID.xxx`，连携配方中邻居符文用此 Tag 匹配 |
| 效果 Tags | 格式 `Card.Effect.xxx`，可选多个；连携配方支持按效果 Tag 匹配 |
| 卡类型 | Attack / Defense / Utility |
| 显示名称 | 战斗卡 HUD 显示名 |

## 六、连携配方（Combo Recipe）

在中间面板切换到 **连携配方** 标签。

连携配方定义当前符文卡与**相邻卡**之间的加成关系：

| 方向 | 含义 |
|---|---|
| **正向（Forward）** | 当前符文排在邻居卡**后面**（邻居→本卡） |
| **反向（Backward）** | 当前符文排在邻居卡**前面**（本卡→邻居） |

每条配方配置：
- **邻居 Tag**：邻居卡的 Card.ID.xxx 或 Card.Effect.xxx
- **匹配方式**：按 Effect Tag 匹配（推荐）或按 ID Tag 精确匹配
- **倍率（Multiplier）**：对本次触发的输出乘以该系数
- **理由文本**：显示在 HUD 上的连携名称说明

填写完毕后点击 **保存连携配方**。

---

## 七、校验与调试

- 底部切换 **校验** 标签：查看 Flow 图的连接错误和数值异常警告。  
- 底部切换 **运行日志** 标签：在编辑器内点击顶部 **运行符文** 查看执行输出。  
- 顶部 **打开符文 / 打开流程** 按钮直接跳转到 Content Browser 中的对应资产。

---

## 八、工作流总结

```
新建符文 → 填写基本信息 → 关联 Flow 资产
         → 数值表：添加所有调参 Key
         → 流程图：连节点描述逻辑
         → [可选] 战斗卡 + 连携配方
         → 运行调试 → 保存
```
