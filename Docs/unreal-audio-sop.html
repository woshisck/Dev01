Unreal Audio SOP

虚幻引擎音效管理指南

从命名、响度检测、UE 配置到验收归档的一页式工作流。

核心原则 命名规范 响度数据 分类路由 UE 接口 声音工作流 资产模板 目录结构 SOP 流程

让项目里的每一个声音都可追踪、可配置、可验收。

不要只管理 WAV 文件。更稳的方式是管理声音资产的完整生命周期：制作、响度、导入、路由、触发、调试、归档。

命名统一类型、系统、动作、变体、版本都能从文件名看出来。
响度可测记录 LUFS、True Peak、RMS、时长和动态范围。
配置复用Attenuation、Concurrency、Submix、Sound Class 模板化。
验收闭环用 Audio Insights 和资产表检查播放、性能和混音。

核心原则

从“能响”升级到“能长期维护”。

推荐生命周期：原始素材 -> 处理与响度校准 -> UE 导入 -> 分类路由 -> 空间化/衰减/并发/混音配置 -> 触发逻辑 -> QA 检查 -> 版本归档

项目规模越大，越要把音效当成系统资产。命名、响度、配置、触发、验收都应该有规则。

可以从最简单的免费工具开始：先学会看 LUFS 和 True Peak，再把结果填进资产表。

响度参考起点

类型建议范围注意
Master 输出不超过 -1 dBTP避免平台转换或压缩后削波。
UI 音效-20 到 -16 LUFS-S短、清楚、不要刺耳。
交互 SFX-18 到 -12 LUFS-S按重要性分层。
脚步 / Foley-28 到 -20 LUFS-S高频播放，避免疲劳。
环境 AMB-34 到 -24 LUFS-I长时间播放，宁可稳一点。
BGM-26 到 -18 LUFS-I给 VO 和关键 SFX 留空间。
VO-24 到 -18 LUFS-S优先保证清晰度。
零基础教程

用 Youlean Loudness Meter 做单个声音验收

  1. 安装免费版 Youlean Loudness Meter，在 Reaper、Nuendo、Pro Tools 或任意支持 VST 的软件里作为插件打开。
  2. 把要检查的 WAV 放到一条音轨上，音轨音量保持 0 dB，不要额外加压缩、限制或自动音量。
  3. 点击插件里的 Reset，从头播放完整声音；Loop 类声音至少播放 20 到 30 秒。
  4. 记录 Integrated LUFS、Short Term LUFS、True Peak Max。
  5. 如果 True Peak 高于 -1 dBTP，先降素材增益；如果同类声音听感差很多，再按目标范围微调增益。
  6. 把数值填进资产登记表，状态先标记为 Measured，进 UE 场景听感确认后再标记 Approved。

最小团队可以先不做复杂母带，只要求每个声音都有响度记录、不过载、同类型听感接近。

UE 其他系统接口建议

音频不要散落在各个蓝图节点里。推荐提供一个统一 Audio Facade，让玩法、动画、UI、关卡和剧情都从同一入口请求播放。

统一入口

建议做一个 Audio Manager / Subsystem

接口用途
Play2D(AudioId)UI、系统提示、非空间化声音。
PlayAtLocation(AudioId, Location)一次性世界声音，例如开门、拾取、爆炸。
PlayAttached(AudioId, Actor/Component)跟随角色或物体的声音，例如机器运转、火焰。
SetState(Tag)切换音乐、环境、战斗状态。
SetRTPC/Parameter(Name, Value)把速度、血量、距离、天气强度传给 MetaSound 或混音系统。
StopGroup(GroupId)停止一类声音，例如关闭某区域环境音。
数据驱动

不要让业务直接引用 WAV

玩法系统只传 AudioId 或 GameplayTag，Audio Manager 去查 Data Asset，再决定实际播放哪个 MetaSound、Sound Cue、Attenuation、Concurrency 和 Submix。

Gameplay Event
  -> AudioId / GameplayTag
  -> Audio Data Asset
  -> Sound Asset + ATT + CON + Submix
  -> Audio Manager Play
接入位置

各系统应该在哪里接音频

系统推荐接入点典型声音注意
角色动画Anim Notify / Anim Notify State脚步、落地、挥砍、衣料高频声音要并发限制，脚步材质可从 Physical Material 取。
玩法交互交互组件、Ability、Gameplay Event开门、拾取、宝箱、机关只发事件和 AudioId，不直接播放具体 WAV。
UI统一 UI Sound Router 或 Common UI Action点击、确认、返回、错误提示2D 播放，独立 UI Sound Class，支持玩家关闭或调低。
关卡环境Audio Volume、Ambient Sound Actor、Level Script 只做区域开关风、雨、河流、城市底噪区域进入退出用 Fade，避免突然切断。
音乐系统Game State、Music Manager、Gameplay Tags探索、战斗、Boss、胜利 Stinger用状态切换，不在单个敌人蓝图里直接控制 BGM。
剧情过场Sequencer Audio Track 或剧情控制器对白、镜头音效、过场音乐要和 VO、BGM Ducking 规则一致。
物理碰撞Hit Event / Impact Resolver撞击、滚动、破碎按材质、速度、质量选择量级，避免每帧触发。

三类核心工作流

环境音、交互音、背景音乐要分开设计，它们的目标和配置完全不同。

环境音 AMB

按层组织

AMB_Forest_Wind_Loop
AMB_Forest_Birds_Random

  • 用 Audio Volume 控制室内外、混响和过渡。
  • 循环必须检查无缝，区域切换要 Fade。
交互音 SFX

随机、并发、空间化

  • 一个行为至少 3 个变体。
  • 高频声音必须设置 Concurrency。
  • 3D 世界声音必须设置 Attenuation。
  • 用 MetaSound 或 Sound Cue 做随机 Pitch / Volume。
  • 不要在蓝图里散落硬引用，优先统一 Data Asset。

SOP 总流程

从需求到归档的完整闭环，可以直接复制进团队文档。

需求登记明确声音类型、触发场景、是否循环、是否 3D、优先级。
素材制作在 DAW 中完成剪辑、降噪、响度处理、Loop 处理、命名。
响度检测批量测量 LUFS、True Peak、RMS、时长，输出记录表。
导入 UE按目录导入 Sound Wave，检查采样率、声道、Loop、压缩设置。
创建播放资产创建 Sound Cue 或 MetaSound，添加随机变体、Pitch/Volume 随机、参数控制。
配置空间与并发设置 Attenuation、Concurrency、Priority、Virtualization。
路由混音分配 Sound Class、Submix，确认玩家设置能控制对应分类。
玩法接入蓝图、C++、Anim Notify、Sequencer 接入，避免硬编码散落引用。
场景测试检查响度、触发时机、重复感、遮挡、混响、室内外过渡。
性能检查用 Audio Insights 检查活动声音数量、并发裁剪、CPU、内存、流式加载。
验收归档更新音频资产表，标记 Approved，锁定版本。
一句话版：虚幻音效 SOP 的关键不是“把 WAV 放进去能响”，而是建立一套可追踪的资产系统：统一命名、外部响度检测、UE 内分类路由、环境/交互/BGM 分工作流配置，最后用 Audio Insights 和响度表做验收。
