# CustomInput 数据引脚 — 子图参数传递

> 更新：2026-04-24
> 上级文档：[BuffFlow_NodeReference.md](BuffFlow_NodeReference.md) · [BuffFlow_NodeUsageGuide.md](BuffFlow_NodeUsageGuide.md)

---

## 概述

Custom Input 节点现在支持声明 **数据引脚**（Data Pin Properties），使得父图的 SubGraph 节点可以向子图传递参数值。

**解决的问题**：以前基础效果 FA 的数值（伤害量、持续时间、层数上限等）只能硬编码在子图节点里。现在可以在 Custom Input 上声明参数，由父图（符文 FA）在 SubGraph 节点上填入具体值，实现"一个基础效果 FA，不同符文传不同参数"。

---

## 数据流

```
┌─ 父图（符文 FA）──────────────────────────────┐
│                                                │
│  SubGraph (Asset = FA_Effect_Poison)           │
│    ├── "Apply" 执行引脚（触发 Custom Input）    │
│    ├── DamagePercent = 0.07  ← 数据输入引脚     │
│    └── Duration = 8.0       ← 数据输入引脚     │
│                                                │
└────────────────────────────────────────────────┘
        │ 执行 + 数据
        ▼
┌─ 子图（FA_Effect_Poison）─────────────────────┐
│                                                │
│  Custom Input "Apply"                          │
│    Properties:                                 │
│      DamagePercent (Float, 默认 0.07)          │
│      Duration      (Float, 默认 8.0)           │
│    ↓ 执行引脚                                   │
│    ↓ DamagePercent 数据引脚 ─→ CalcDamage.BaseValue
│    ↓ Duration 数据引脚 ─────→ ApplyAttributeModifier.Duration
│                                                │
│  Custom Output "Done"                          │
│                                                │
└────────────────────────────────────────────────┘
```

**优先级**：父图 SubGraph 填的值 > Custom Input 本地默认值。父图不填时回退到默认值。

## 使用步骤

### 1. 在子图的 Custom Input 上声明参数

1. 打开子图 FA（如 FA_Effect_Poison）
2. 选中 Custom Input "Apply" 节点
3. 在 Details 面板找到 **DataPins → Properties**
4. 点击 `+` 添加条目：
   - **Name**：参数名（如 `DamagePercent`）
   - **DataPinProperty**：选择类型（如 `Float`），填入默认值（如 `0.07`）
5. 节点底部会自动生成对应的 **数据输出引脚**

### 2. 将数据引脚连到下游节点

从 Custom Input 的数据输出引脚（如 `DamagePercent`）拉线连接到下游节点的数据输入引脚（如 `CalcDamage.BaseValue`）。

### 3. 在父图 SubGraph 上填值

1. 打开父图（如 FA_Rune_PoisonBlade）
2. 选中 SubGraph 节点（引用子图 FA_Effect_Poison）
3. SubGraph 节点会 **自动生成** 与 Custom Input Properties 匹配的数据输入引脚
4. 直接在 SubGraph 节点上填入具体值，或从父图其他节点的数据引脚连入

## 支持的数据类型

| 类型 | 典型用途 |
|------|---------|
| Bool | 开关标志 |
| Int | 层数上限、计数 |
| Float | 伤害值、百分比、持续时间 |
| Name | 标识符 |
| String | 文本 |
| Text | 本地化文本 |
| Enum | 枚举选项 |
| Vector | 位置、方向 |
| Rotator | 旋转 |
| Transform | 完整变换 |
| GameplayTag | 单个 Tag |
| GameplayTagContainer | Tag 集合 |
| InstancedStruct | 自定义结构体 |
| Object | UObject 引用 |
| Class | UClass 引用 |

## 示例：可配参数的中毒效果

### 子图 FA_Effect_Poison

Custom Input "Apply" 声明两个参数：

| Name | Type | Default |
|------|------|---------|
| DamagePercent | Float | 0.07 |
| Duration | Float | 8.0 |

```
[Custom Input "Apply"]
    Properties: DamagePercent(Float,0.07), Duration(Float,8.0)
    ↓ 执行                ↓ DamagePercent        ↓ Duration
[CalcDamage ←───── BaseValue]
    BaseMode = PercentMaxHP
    Target   = LastDamageTarget
    ↓ Out                 ↓ FinalDamage
[ApplyAttributeModifier ← Value=FinalDamage, Duration=Duration
    Attribute       = DamageAttributeSet.DamageBuff
    ModOp           = Additive
    DurationType    = HasDuration
    Period          = 2.0
    FireImmediately = false
    GrantedTags     = Buff.Status.Poisoned
    StackMode       = Unique
    DurationRefresh = RefreshOnSuccessfulApplication
    Target          = LastDamageTarget]
    ↓ Out
[Custom Output "Done"]
```

### 父图调用方式

**符文 A — 轻度中毒**（5% MaxHP，6 秒）：
```
[Start] → [SubGraph FA_Effect_Poison]
              DamagePercent = 0.05
              Duration      = 6.0
```

**符文 B — 剧毒**（12% MaxHP，10 秒）：
```
[Start] → [Probability 0.3] → [SubGraph FA_Effect_Poison]
                                  DamagePercent = 0.12
                                  Duration      = 10.0
```

> 同一个 FA_Effect_Poison 通过不同参数实现不同强度，无需复制子图。

## 与 Start 节点的对比

| 对比项 | Start 节点 | Custom Input 节点 |
|--------|-----------|------------------|
| 触发时机 | SubGraph 执行引脚为 "Start" 时 | SubGraph 执行引脚为自定义名称时 |
| 数据引脚 | 支持 | 支持（本次新增） |
| 多入口 | 不支持（一个子图只有一个 Start） | 支持（可声明多个 Custom Input） |
| 适用场景 | 单入口子图 | 多入口子图（如 Apply / Cancel） |

> 对于基础效果 FA（只有 Apply → Done 的单入口子图），Start 和 Custom Input 都可以用。推荐继续使用 Custom Input "Apply" + Custom Output "Done" 的模式，因为语义更清晰。

## 技术实现

改动范围（FlowGraph 插件，3 个文件）：

| 文件 | 改动 |
|------|------|
| `FlowNode_CustomInput.h` | 新增 `IFlowNodeWithExternalDataPinSupplierInterface` + `IFlowDataPinGeneratorNodeInterface` 接口；添加 `OutputProperties` 数组和外部数据供应商成员 |
| `FlowNode_CustomInput.cpp` | 实现 `TryAppendExternalInputPins`（SubGraph 自动生成引脚）、`AutoGenerateDataPins`（节点自身生成数据输出引脚）、`TryFindPropertyByRemappedPinName`（本地默认值查找）、15 种 `TrySupplyDataPinAs*`（外部供应商优先，本地回退） |
| `FlowAsset.cpp` | 移除两处 "Custom Input 不支持数据引脚" 的过时注释（Cast 逻辑已就绪，现在 CustomInput 实现了接口，Cast 成功） |

**数据解析优先级**：

```
下游节点 TryResolveDataPinAsFloat("DamagePercent")
    → 向上查找 CustomInput
        → CustomInput.TrySupplyDataPinAsFloat("DamagePercent")
            → 1. 检查外部供应商（SubGraph）→ 有值则返回
            → 2. 回退到本地 OutputProperties 默认值
```
