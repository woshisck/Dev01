# Obsidian 引用与文档规范

## 链接格式

优先使用相对 Markdown 链接：

```md
[文档名](../04_开发实现与系统文档/系统/Rune/README.md)
```

可以使用 Obsidian Backlinks、Outgoing Links 和 Graph 查看关系，但正文链接保持 GitHub 和编辑器兼容。

## 状态标记

关键入口文档建议使用 YAML frontmatter：

```yaml
---
status: current
type: guide
owner: engineering
updated: 2026-06-06
---
```

状态含义：

- `current`：当前依据。
- `archived`：历史记录，仅追溯。
- `deprecated`：废弃内容，不作为依据。

## 移动文档必须做

1. 更新 [路径迁移表](../00_入口与规范/路径迁移表.md)。
2. 修复所有能自动确定的新路径链接。
3. 无法确定的链接写入 [缺失引用记录](../00_入口与规范/缺失引用记录.md)。
4. 如果文档被归档或废弃，在正文开头标注状态。

## 关系图谱配置

Vault 的全局关系图谱配置在 `Docs/.obsidian/graph.json`。

默认图谱用于查看当前工作文档关系，因此默认：

- 隐藏 unresolved 节点，避免源码文件名、旧路径和未创建页面污染图谱。
- 隐藏孤立节点，避免未连接文档压缩主图可读性。
- 过滤 `90_自动生成报告/`、`98_废弃/`、`99_归档/`，让当前依据更突出。
- 按顶层目录分组上色：
  - `01_策划需求与版本方案/`
  - `02_玩法设计与调研/`
  - `03_策划配置与制作手册/`
  - `04_开发实现与系统文档/`
  - `05_完成记录与任务看板/`
  - `06_AI协作与VibeCoding/`

如果要追溯历史方案，可以在 Obsidian 关系图谱筛选框里临时删除 `-path:"99_归档"`；如果要检查废弃内容是否被误引用，可以临时删除 `-path:"98_废弃"`。
