# 测试与验证要求

## 验证层级

1. 静态检查：确认引用路径、Tag、配置名、资产路径。
2. 编译检查：涉及 C++ 时必须给出真实编译结果。
3. 自动化测试：已有测试覆盖时优先运行目标测试。
4. PIE/包体验证：涉及流程、UI、教程、战斗手感时必须说明运行路径。
5. 文档验证：涉及设计或配置变更时同步更新 Docs。

## 输出格式

实现完成后说明：

- 验证命令。
- 验证结果。
- 未能验证的原因。
- 仍需人工检查的点。

## 注意

`Target is up to date` 不等于完整编译验证。需要证明重新编译时，应使用明确的 clean build 或足够可靠的增量编译证据。
