# AI 协作与 Vibe Coding 总入口

> 本目录是 Codex、Claude 等 AI 助手参与项目开发前的必读入口。AI 写代码前必须先确认当前系统依据、修改边界和验证方式。

## 必读顺序

1. [AI 修改边界与禁区](AI修改边界与禁区.md)
2. [引擎版本与项目依据](引擎版本与项目依据.md)
3. [AI 代码编写规范](AI代码编写规范.md)
4. [常用源码入口](常用源码入口.md)
5. [测试与验证要求](测试与验证要求.md)
6. [Obsidian 引用与文档规范](Obsidian引用与文档规范.md)

## 任务交接

新任务交给 AI 前，优先使用 [AI 任务交接模板](AI任务交接模板.md)。模板必须明确：

- 背景与目标。
- 必读文档。
- 允许修改范围。
- 禁止修改范围。
- 验收方式。
- 回滚点。

## 当前依据优先级

1. 代码和配置中的真实运行路径。
2. `01_长期系统文档/` 中的长期系统文档。
3. `02_版本计划与需求/` 中的当前核心方案。
4. `03_配置与编辑器手册/` 中的制作和配置说明。
5. `99_归档/` 仅作历史背景。
6. `98_废弃/` 不得作为实现依据。
