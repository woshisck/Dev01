# 512版本项目计划 - 任务清单

## 版本目标

512 版本的目标是把 429 反馈迭代方案落成可开发、可分工、可验收的任务清单。

本版本的核心体验为：快速战斗 + 1D 攻击卡组 + 后台背包 + 洗牌装弹。玩家主要在战斗中通过轻重攻击顺序触发卡牌，旧 2D 背包退为后台存储、奖励接收和存档系统。

## 分工原则

| 方向 | 负责人 | 说明 |
| --- | --- | --- |
| 背包、符文、卡组底座 | Codex | 负责 CombatDeckComponent、卡牌数据、洗牌逻辑、后台背包兼容、卡组 UI 事件 |
| 战斗输入与连招 | 战斗同事 | 负责轻重攻击、连招终结、DashSave、攻击事件派发 |
| 关卡奖励接入 | Codex + 关卡同事 | 负责清场奖励进入后台背包与卡组，不自动打开旧背包 |
| 联调验收 | 共同联调 | 验证攻击消耗卡牌、打空后洗牌、洗牌期间仍可攻击 |
| 表现与文案 | 策划/美术 | 负责卡牌名称、HUD 提示、连携触发说明、洗牌表现 |

## 阶段 1：接口冻结

| 任务 | 负责人 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| 冻结战斗到卡组的调用接口 | 共同联调 | `ResolveAttackCard(ActionType, bIsComboFinisher, bFromDashSave)` | 战斗侧和卡组侧都按同一参数理解攻击事件 |
| 冻结卡组返回结果 | Codex | 卡牌消耗、动作匹配、基础效果、匹配奖励、连携、洗牌状态 | 战斗侧无需读取卡组内部数据即可播放表现 |
| 冻结攻击动作枚举 | 战斗同事 | Light / Heavy / Any | 卡牌需求和战斗输入使用同一套动作概念 |
| 冻结 DashSave 上下文需求 | 共同联调 | DashSave 是否保留卡牌连携上下文 | DashSave 后继续攻击时，卡牌链路不会意外丢失 |

## 阶段 2：背包符文卡组底座

| 任务 | 负责人 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| 创建 CombatDeckComponent 方案 | Codex | 组件职责、状态、事件、接口文档 | 文档能指导后续直接开发组件 |
| 定义卡牌数据结构 | Codex | CardType、RequiredAction、BaseFlow、MatchedFlow、LinkMode、FinisherRequirement | 卡牌能表达基础触发、匹配奖励、连携、终结技 |
| 定义洗牌装弹逻辑 | Codex | 固定顺序、打空、冷却、回填规则 | 4 张卡连续攻击 4 次后进入洗牌，冷却结束后按固定顺序回填 |
| 定义后台背包兼容方式 | Codex | 旧 BackpackGridComponent 的后台职责 | 旧背包不再默认打开，但仍能保存和接收奖励 |
| 定义卡组 UI 事件 | Codex + UI 同事 | 卡牌消耗、匹配、连携、洗牌开始、洗牌进度、洗牌完成 | UI 不需要主动查询卡组内部结构 |

## 阶段 3：战斗侧事件接入

| 任务 | 负责人 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| 轻攻击事件派发 | 战斗同事 | Light 攻击完成或命中时调用卡组接口 | Light 卡被 Light 攻击消耗时触发匹配奖励 |
| 重攻击事件派发 | 战斗同事 | Heavy 攻击完成或命中时调用卡组接口 | Light 卡被 Heavy 攻击消耗时只触发基础效果 |
| 连招终结标记 | 战斗同事 | Combo Finisher 标记 | Finisher 卡只在终结击触发额外效果 |
| DashSave 衔接 | 战斗同事 | DashSave 窗口和上下文保留 | DashSave 后连招与卡牌连携上下文按规则保留 |

## 阶段 4：关卡奖励接入

| 任务 | 负责人 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| 清场主奖励三选一 | 关卡同事 | 清场后出现主奖励选择 | 玩家选完奖励后进入后台背包和卡组 |
| 小怪掉落金币或小资源 | 关卡同事 | 普通掉落规则 | 小怪掉落不打断战斗节奏 |
| 奖励自动加入卡组 | Codex | 奖励到卡组的接入规则 | 选择奖励后不自动打开旧背包 UI |
| 传送门保留 | 关卡同事 | 清场后开启传送门 | 玩家可不拾取奖励直接进门，但等于主动放弃成长 |

## 阶段 5：联调验收

| 场景 | 负责人 | 验收标准 |
| --- | --- | --- |
| 4 张卡连续攻击 4 次 | 共同联调 | 第 4 张消耗后立即进入洗牌，1D UI 清空 |
| 洗牌期间继续攻击 | 共同联调 | 玩家攻击照常播放和造成基础伤害，但不触发卡牌 |
| 洗牌结束 | 共同联调 | UI 立即按固定顺序显示卡组，下一次攻击可触发卡牌 |
| 动作不匹配 | 共同联调 | 卡牌仍消耗，只触发 BaseFlow，不触发 MatchedFlow 和连携 |
| 终结技卡 | 共同联调 | 只有 Combo Finisher 命中时触发终结效果 |
| 关卡奖励 | 共同联调 | 奖励选择后进入后台背包和卡组，不自动打开旧背包 |

## 本版本不做

- 不删除旧 2D 背包。
- 不做完整鬼泣式复杂连招树。
- 不做随机洗牌。
- 不把热度继续作为符文激活区扩展逻辑。
- 不在第一版强制玩家进入背包整理流程。
