# 战斗与卡组接口契约

## 目标

这个文档用于冻结战斗同事和 Codex 之间的协作边界。双方只通过明确接口交换信息，不互相读取内部状态。

## 战斗侧负责

- 判断玩家本次攻击是 Light 还是 Heavy。
- 判断本次攻击是否命中或是否进入消耗时机。
- 判断本次攻击是否为 Combo Finisher。
- 判断本次攻击是否来自 DashSave 衔接。
- 在稳定时机调用卡组接口。

## 卡组侧负责

- 判断当前是否有可用卡牌。
- 消耗当前卡牌。
- 判断动作是否匹配。
- 触发 BaseFlow 和 MatchedFlow。
- 判断 Link 和 Finisher。
- 判断是否进入洗牌。
- 派发 UI 事件。

## 冻结接口

```cpp
FCombatCardResolveResult ResolveAttackCard(
    ECardRequiredAction ActionType,
    bool bIsComboFinisher,
    bool bFromDashSave
);
```

## 输入约定

| 输入 | 约定 |
| --- | --- |
| `ActionType = Light` | 轻攻击事件 |
| `ActionType = Heavy` | 重攻击事件 |
| `ActionType = Any` | 仅用于特殊攻击或无差别卡牌，普通输入不优先使用 |
| `bIsComboFinisher = true` | 本次攻击是当前连招最后一击 |
| `bFromDashSave = true` | 本次攻击来自 DashSave 衔接 |

## 输出约定

| 输出 | 战斗侧用途 |
| --- | --- |
| `bHadCard` | 决定是否播放卡牌触发反馈 |
| `bActionMatched` | 决定是否播放匹配反馈 |
| `bTriggeredBaseFlow` | 决定是否执行基础效果表现 |
| `bTriggeredMatchedFlow` | 决定是否执行奖励效果表现 |
| `bTriggeredLink` | 决定是否播放连携提示 |
| `bTriggeredFinisher` | 决定是否播放终结技表现 |
| `bStartedShuffle` | 决定是否播放装填开始反馈 |

## 关键约束

- 卡组系统不能阻止玩家攻击。
- 战斗系统不能跳过卡组的 CurrentIndex 自行选择卡牌。
- 洗牌期间战斗侧仍可调用接口，但卡组侧返回无卡结果。
- 动作不匹配时卡牌仍消耗，只是不触发匹配奖励和连携。
- Finisher 卡必须依赖 `bIsComboFinisher`，不能只靠卡牌位置触发。

## 验收标准

- 双方只依赖本接口即可完成第一次联调。
- 洗牌期间攻击不会报错，也不会触发卡牌。
- DashSave 场景能通过 `bFromDashSave` 被卡组侧识别。
- 战斗侧不需要读取 DeckList、ActiveSequence、CurrentIndex。
