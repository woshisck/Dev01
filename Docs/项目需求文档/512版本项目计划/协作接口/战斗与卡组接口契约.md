# 战斗与卡组接口契约

## 目标

这个文档用于冻结战斗同事和 Codex 之间的协作边界。双方只通过明确接口交换信息，不互相读取内部状态。

## 战斗侧负责

- 判断本次攻击是否命中或是否进入消耗时机。
- 提供玩家本次攻击的动作类型，供连携卡条件使用。
- 提供当前连招是否继续、是否退出连招状态、连击段数或 ComboTag。
- 当玩家退出连招状态时，向卡组侧发送明确通知；卡组侧不自行用时间超时猜测。
- 后续终结技阶段再提供 Combo Finisher。
- 判断本次攻击是否来自 DashSave 衔接。
- 在稳定时机调用卡组接口。

## 卡组侧负责

- 判断当前是否有可用卡牌。
- 消耗当前卡牌。
- 普通卡攻击即触发，不再用 Light / Heavy 判断是否激活。
- 触发普通释放 Flow。
- 根据战斗侧提供的连击段数读取全局倍率配置。
- 判断正向连携、反向连携和断链释放。
- 判断是否进入洗牌。
- 派发 UI 事件。

## 当前兼容接口

```cpp
FCombatCardResolveResult ResolveAttackCard(
    ECardRequiredAction ActionType,
    bool bIsComboFinisher,
    bool bFromDashSave
);
```

该接口可以继续作为过渡接口使用，但已经不足以完整表达新连携规则。后续建议替换为：

```cpp
FCombatCardResolveResult ResolveAttackCard(const FCombatDeckActionContext& Context);
void NotifyComboPaused();
void NotifyComboStateExited();
void ClearPendingLinkContext(ECombatLinkBreakReason Reason);
```

## 输入约定

| 输入 | 约定 |
| --- | --- |
| `ActionType = Light / Heavy / Any` | 不再决定普通卡是否激活，只作为连携卡条件 |
| `bIsComboFinisher = true` | 过渡字段；本阶段暂不扩展终结技 |
| `bFromDashSave = true` | 本次攻击来自 DashSave 衔接，后续可进入连携条件 |
| `bExitedComboState = true` | 后续字段；如果存在等待中的反向连携，需要触发断链处理 |
| `ComboIndex / ComboTags` | 后续字段；用于全局倍率和少数连携条件 |

## 输出约定

| 输出 | 战斗侧用途 |
| --- | --- |
| `bHadCard` | 决定是否播放卡牌触发反馈 |
| `bActionMatched` | 旧字段；普通卡不再依赖动作匹配 |
| `bTriggeredBaseFlow` | 决定是否执行普通释放表现 |
| `bTriggeredMatchedFlow` | 旧字段；后续拆分为正向连携、反向赋能或断链 |
| `bTriggeredLink` | 旧字段；后续拆分为 `bTriggeredForwardLink` / `bTriggeredBackwardLink` |
| `bTriggeredFinisher` | 终结技阶段再使用 |
| `bStartedShuffle` | 决定是否播放装填开始反馈 |

## 关键约束

- 卡组系统不能阻止玩家攻击。
- 战斗系统不能跳过卡组的 CurrentIndex 自行选择卡牌。
- 洗牌期间战斗侧仍可调用接口，但卡组侧返回无卡结果。
- 普通卡不再存在动作不匹配导致奖励失效的问题；攻击出牌即正常触发。
- 动作类型只影响少数连携卡的条件。
- 当战斗侧通知退出连招状态且存在等待中的反向连携时，卡组侧需要执行断链清理。
- 普通卡倍率来自全局配置，不在每张普通卡上单独配置。
- 反向连携的效果 B 是独立配置的新效果，不复用后一张卡自身的普通释放 Flow。
- 三连携和终结技本阶段不进入接口冻结。

## 验收标准

- 双方只依赖本接口即可完成第一次联调。
- 洗牌期间攻击不会报错，也不会触发卡牌。
- DashSave 场景能通过 `bFromDashSave` 被卡组侧识别。
- 战斗侧不需要读取 DeckList、ActiveSequence、CurrentIndex。
- 普通卡 Light / Heavy 都会正常触发。
- 连携卡可以区分正向连携、反向赋能和断链释放。
