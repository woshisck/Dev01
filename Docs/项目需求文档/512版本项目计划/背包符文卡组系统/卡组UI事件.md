# 卡组 UI 事件

## 目标

1D 卡组 UI 只展示玩家需要立即理解的信息：当前序列、下一张卡、匹配结果、连携触发、洗牌状态。

UI 不直接读取卡组内部数组，而是监听 `CombatDeckComponent` 派发的状态事件。

## 事件列表

| 事件 | 触发时机 | UI 表现 |
| --- | --- | --- |
| `OnDeckLoaded` | 战斗开始或洗牌完成 | 显示完整 ActiveSequence |
| `OnCardConsumed` | 卡牌被攻击消耗 | 当前卡移出或淡出 |
| `OnActionMatched` | 攻击动作符合 RequiredAction | 当前卡闪光或播放匹配反馈 |
| `OnLinkTriggered` | 连携成功 | 显示触发原因文案 |
| `OnFinisherTriggered` | 终结技卡成功触发 | 播放更强的终结反馈 |
| `OnShuffleStarted` | 最后一张卡被消耗 | UI 清空，显示装填状态 |
| `OnShuffleProgress` | 洗牌冷却中 | 更新进度条或倒计时 |
| `OnShuffleCompleted` | 洗牌结束 | 重新显示固定顺序卡组 |

## 触发原因文案

高级连携必须告诉玩家为什么触发，避免玩家觉得效果随机出现。

示例：

| 场景 | HUD 文案 |
| --- | --- |
| 读取上一张攻击卡 | 月光读取上一张攻击牌：伤害提升 |
| 传递给下一张卡 | 火种已传递：下一张卡获得爆发 |
| 终结击触发 | 终结击命中：卡牌效果强化 |
| 热度强化连携 | 热度加压：连携倍率提升 |

## 洗牌期间表现

洗牌期间 UI 可以采用以下任一表现：

- 卡槽暂时清空。
- 显示“装填中”进度。
- 显示剩余冷却时间。

第一版优先保证信息清楚，不追求复杂动画。

## 验收标准

- 卡牌消耗后 UI 能同步移除当前卡。
- 洗牌开始时 UI 清空或进入装填状态。
- 洗牌结束后 UI 按固定顺序回填。
- 连携和终结技触发时有明确原因提示。
