# 卡牌数据结构

## 目标

第一版卡牌数据要足够表达攻击触发、动作匹配、连携、终结技，同时不要把旧 2D 背包的复杂空间规则带回来。

## 建议枚举

```cpp
enum class ECombatCardType
{
    Attack,
    Link,
    Finisher,
    Passive
};

enum class ECardRequiredAction
{
    Light,
    Heavy,
    Any
};

enum class ECardLinkMode
{
    None,
    ReadPrevious,
    PassToNext
};

enum class EDeckState
{
    Ready,
    EmptyShuffling
};
```

## 卡牌配置字段

| 字段 | 说明 |
| --- | --- |
| `CardType` | Attack / Link / Finisher / Passive |
| `RequiredAction` | Light / Heavy / Any |
| `BaseFlow` | 不符合动作时也触发的基础效果 |
| `MatchedFlow` | 符合动作时触发的奖励效果 |
| `LinkMode` | 是否读取上一张或传递给下一张 |
| `FinisherRequirement` | 是否要求本次攻击为 Combo Finisher |
| `DisplayName` | UI 显示名称 |
| `HUDReasonText` | 连携触发时显示给玩家的原因 |

## 运行时实例

运行时建议使用 `FCombatCardInstance`，不要直接修改 DataAsset。

| 字段 | 说明 |
| --- | --- |
| `InstanceGuid` | 本张卡的运行时唯一标识 |
| `SourceData` | 来源卡牌 DataAsset |
| `Level` | 当前等级 |
| `OwnerSource` | 来源：初始武器、关卡奖励、商店、事件 |
| `RuntimeState` | 当前运行时状态 |

## 第一版限制

- 不做随机洗牌。
- 不做 2D 格子旋转、裁剪、拼接。
- Passive 卡可以先进入数据结构，但第一版可暂不作为主战斗触发重点。
- Link 卡作为高级奖励，不作为新手必须理解的第一层规则。

## 验收标准

- 一张卡能表达 Light / Heavy / Any 的动作需求。
- 动作不匹配时仍可触发 BaseFlow。
- 动作匹配时能触发 MatchedFlow。
- Finisher 卡能声明自己需要 Combo Finisher。
- Link 卡能声明读取上一张或传递给下一张。
