# 洗牌装弹逻辑

## 目标

卡组不是无限触发。玩家把当前卡组打空后，会进入一个短暂的洗牌或装弹空窗。空窗期间玩家依然可以攻击，只是不会触发卡牌效果。

这个节奏类似射击游戏中的装弹：不是禁止行动，而是让玩家感受到爆发和空窗的交替。

## 基础规则

| 规则 | 说明 |
| --- | --- |
| 固定顺序 | DeckList 按玩家当前卡组顺序排列，不随机 |
| 按次消耗 | 每次有效攻击事件尝试消耗当前 1 张卡 |
| 打空进入冷却 | 最后一张卡被消耗后立即进入 EmptyShuffling |
| UI 清空 | 洗牌期间 1D UI 暂时为空 |
| 玩家可攻击 | 洗牌期间攻击照常播放和造成基础伤害 |
| 不触发卡牌 | 洗牌期间不会触发 BaseFlow、MatchedFlow、Link、Finisher |
| 冷却回填 | 冷却结束后按 DeckList 固定顺序重新填充 ActiveSequence |

## 时间配置

洗牌时间默认 1 秒，但不能写死在组件里。

建议由武器 DA 或等价武器配置提供：

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `ShuffleCooldownDuration` | 1.0 | 卡组打空后的洗牌时间 |
| `InitialDeckList` | 按武器配置 | 初始卡组 |
| `MaxActiveSequenceSize` | 取 DeckList 长度 | 第一版可直接展示完整卡组 |

## 例子：4 张卡组

1. 玩家有 4 张卡：A、B、C、D。
2. 第一次攻击消耗 A。
3. 第二次攻击消耗 B。
4. 第三次攻击消耗 C。
5. 第四次攻击消耗 D。
6. D 消耗后立刻进入洗牌，1D UI 清空。
7. 洗牌 1 秒内，玩家仍可攻击，但不触发卡牌。
8. 1 秒结束后，UI 重新显示 A、B、C、D。
9. 下一次攻击从 A 重新开始。

## 验收标准

- 固定顺序卡组每轮洗牌后顺序一致。
- 4 张卡连续攻击 4 次后进入洗牌。
- 洗牌期间 UI 为空或显示装填进度。
- 洗牌期间继续攻击不会触发卡牌，也不会阻止冷却。
- 冷却结束后 UI 立即回填，下一次攻击可以触发卡牌。
