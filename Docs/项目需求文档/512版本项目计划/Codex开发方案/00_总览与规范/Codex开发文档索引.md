# Codex 开发文档索引

## 作用

本索引用于归纳 `Codex开发方案` 下的 AI 开发文档，避免后续开发方案、配置说明、规则说明混在一起难以查找。

后续新增文档时，优先参考：

`00_总览与规范/AI开发文档编写与分类规范.md`

## 配置说明目录

配置说明统一放在：

`配置说明/`

当前已有分类：

| 文件夹 | 用途 |
| --- | --- |
| `配置说明/符文/` | 符文、卡牌、BuffFlow、连携卡相关配置说明 |
| `配置说明/武器/` | 武器 DA、初始卡组、武器连招相关配置说明 |
| `配置说明/UI/` | WBP、HUD、菜单、界面绑定相关配置说明 |
| `配置说明/奖励/` | 三选一、奖励池、掉落池相关配置说明 |
| `配置说明/敌人/` | 敌人 BP、EnemyData、AI 行为树、动作命中相关配置说明 |
| `配置说明/关卡/` | 刷怪难度分、房间敌人上限、关卡节奏相关配置说明 |

新增配置文档时，必须先按功能种类进入对应文件夹，再按配置对象拆文件，例如：

```text
配置说明/符文/战斗卡牌符文DA配置说明.md
配置说明/符文/符文WBP配置说明.md
配置说明/战斗手感/FreezeHitStop配置说明.md
配置说明/表现/命中特效Niagara配置说明.md
```

## 设计文档目录

非代码执行方案、玩法规则、字段规范、策划填写模板应放在：

`../设计/`

示例：

```text
设计/符文/卡牌设计总纲.md
设计/符文/卡牌字段规范.md
设计/战斗/打击感规则说明.md
```

## 后续归档规则

- 总览、上下文、规范和索引放在 `Codex开发方案/00_总览与规范/`。
- AI 要执行的系统开发计划放在 `Codex开发方案/01_系统开发方案/`。
- 联调和验收方案放在 `Codex开发方案/02_联调验收/`。
- 策划照着配置的说明放在 `Codex开发方案/配置说明/<功能种类>/`。
- 玩法规则、字段规范、设计填写表放在 `设计/<功能种类>/`。
- 长期技术说明放在 `Docs/Systems/`。
- 临时过程记录放在 `Docs/WorkSession/`。

如果一个新功能同时涉及代码和配置，需要同时产出：

1. `01_系统开发方案/功能名开发方案.md`
2. `配置说明/<功能种类>/<配置对象>配置说明.md`

如果涉及 WBP、材质 Custom Node、Niagara、DataTable，必须按对象拆分配置说明。
