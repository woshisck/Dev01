## 目标

512 版本开发按多个 Codex 小组拆分。每个小组只负责自己的系统边界，通过冻结接口联调，避免多人同时修改同一批文件。

当前我们所在的小组是“背包符文卡组组”。除此之外，还需要战斗系统组、关卡奖励组、UI 表现组、热度 BuffFlow 组、联调验收组。

## Codex 小组划分

| 小组 | 主要职责 | 主要产出 | 不负责 |
| --- | --- | --- | --- |
| Codex-背包符文卡组组 | 后台背包、卡牌数据、卡组运行时、洗牌装弹 | CombatDeckComponent、卡牌 DA、背包兼容、奖励入卡组 | 轻重攻击动作、连招动画 |
| Codex-战斗系统组 | 轻重攻击、连招终结、DashSave、攻击事件派发 | 攻击事件接口接入、终结击标记、冲刺窗口 | 卡组内部状态、奖励选择 UI |
| Codex-关卡奖励组 | 房间节奏、清场奖励、小怪掉落、传送门 | 清场后三选一、奖励跳过、下一房节奏 | 卡牌效果实现、攻击判定 |
| Codex-UI表现组 | 1D 卡组 HUD、洗牌表现、连携提示、奖励提示 | 卡组条、装填进度、触发原因文本 | 卡组判定逻辑、战斗伤害 |
| Codex-热度 BuffFlow 组 | 热度收益、Flow 执行、倍率和表现强度 | Heat 与 BuffFlow 的新职责 | 背包格子激活区 |
| Codex-联调验收组 | 跨系统验证、测试清单、问题回收 | 联调场景、验收记录、缺陷清单 | 单独拥有系统实现 |

## 推荐开发顺序

1. Codex-背包符文卡组组先冻结数据结构和 `ResolveAttackCard` 返回结果。
2. Codex-战斗系统组接入 Light / Heavy / Finisher / DashSave 调用。
3. Codex-UI表现组接入卡组事件，不直接读卡组内部数组。
4. Codex-关卡奖励组接入清场奖励到后台背包和 DeckList。
5. Codex-热度 BuffFlow 组调整热度定位，只强化连携、终结技和表现。
6. Codex-联调验收组按清单跑通最小闭环。

## 共享接口原则

- 战斗侧只调用卡组接口，不读取 DeckList、ActiveSequence、CurrentIndex。
- 卡组侧不阻止玩家攻击，只返回本次是否触发卡牌。
- UI 侧只监听事件，不反向修改卡组状态。
- 关卡侧只负责发奖励，不直接执行卡牌效果。
- 热度侧只影响倍率、表现和连携强度，不再控制背包激活区。
