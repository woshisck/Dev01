# Codex-关卡奖励组开发方案

## 组定位

本组负责房间节奏、清场奖励、小怪掉落、传送门和奖励接入节奏。关卡奖励组的目标是让玩家每个房间都能快速验证卡组构筑，而不是清场后被迫整理背包。

## 负责范围

| 模块 | 说明 |
| --- | --- |
| 普通房节奏 | 45-90 秒，1-3 波敌人 |
| 精英房节奏 | 测试连招、洗牌空窗、热度强化 |
| 小怪掉落 | 金币、小资源、临时补给 |
| 清场主奖励 | 清场后生成三选一奖励 |
| 奖励跳过 | 玩家可直接进门，但放弃成长 |
| 传送门 | 清场后开启，和奖励流程不互相阻塞 |

## 不负责范围

- 不实现卡牌内部效果。
- 不实现 Light / Heavy 攻击。
- 不维护卡组运行时状态。
- 不控制卡组 UI 内部动画。

## 奖励流程

```text
清场 -> 生成主奖励 -> 三选一 -> 玩家选择 -> 后台背包记录 -> DeckList 更新 -> 下一房间验证
```

## 开发步骤

1. 梳理现有 LevelFlow、RewardPickup、LootSelectionWidget、传送门流程。
2. 保留清场后生成主奖励的流程。
3. 调整奖励选择完成后的行为：不自动打开旧背包。
4. 选择奖励后通知背包符文卡组组，将奖励写入后台背包和 DeckList。
5. 小怪掉落只放金币或小资源，不直接掉复杂卡牌。
6. 传送门在清场后开启，允许玩家跳过奖励。
7. 调整普通房敌人数量，让玩家至少能打完一轮卡组。
8. 调整精英房压力，让玩家至少经历一次洗牌空窗。

## 验收标准

- 普通房能让玩家至少打完一轮 4 张卡。
- 精英房能让玩家体验至少一次卡组打空和回填。
- 清场后出现主奖励。
- 选择奖励后不自动打开旧背包。
- 奖励能进入后台背包和 DeckList。
- 玩家不拾取奖励直接进门时流程不阻塞。
