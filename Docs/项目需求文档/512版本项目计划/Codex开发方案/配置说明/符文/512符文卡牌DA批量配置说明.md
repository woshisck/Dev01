# 512 符文卡牌 DA 批量配置说明

## 作用

用编辑器工具批量生成 512 符文卡牌 DA、基础 Flow 模板副本、图标引用和检查报告。

## 生成位置

| 类型 | 路径 |
| --- | --- |
| DA | `/Game/Docs/BuffDocs/V2-RuneCard/512Generated` |
| Flow | `/Game/Docs/BuffDocs/V2-RuneCard/512Generated/Flow` |
| 图标 | `/Game/Docs/BuffDocs/V2-RuneCard/Icons` |
| 报告 | `Saved/512RuneCardBatchReport.md` |

## 使用步骤

1. 先 dry run：
   ```powershell
   & "Z:\GZA_Software\RealityCapture\UE_5.4\Engine\Binaries\Win64\UnrealEditor-Cmd.exe" "X:\Project\Dev01\DevKit.uproject" -run=RuneCardBatchGenerator -DryRun -unattended -nop4 -nosplash
   ```
2. 查看 `Saved/512RuneCardBatchReport.md`，确认将生成的 DA、Flow、图标和手工待办。
3. 正式生成：
   ```powershell
   & "Z:\GZA_Software\RealityCapture\UE_5.4\Engine\Binaries\Win64\UnrealEditor-Cmd.exe" "X:\Project\Dev01\DevKit.uproject" -run=RuneCardBatchGenerator -Apply -unattended -nop4 -nosplash
   ```
4. 打开生成的 DA，重点检查 `RuneInfo -> CombatCard`。

## DA 字段

| 字段 | 生成值 |
| --- | --- |
| `RuneInfo -> RuneConfig -> RuneName` | 卡牌名 |
| `RuneInfo -> RuneConfig -> RuneIcon` | `T_Rune512_*` |
| `RuneInfo -> RuneConfig -> RuneDescription` | Excel 归纳描述 |
| `RuneInfo -> RuneConfig -> HUDSummaryText` | 关卡/传送门紧凑 HUD 摘要，可手写 1-2 行；为空时自动从描述截取 |
| `RuneInfo -> Flow -> FlowAsset` | 对应基础 Flow 模板副本 |
| `RuneInfo -> CombatCard -> bIsCombatCard` | 勾选 |
| `RuneInfo -> CombatCard -> CardType` | 普通卡为 `Normal`，月光为 `Link` |
| `RuneInfo -> CombatCard -> CardIdTag` | `Card.ID.*` |
| `RuneInfo -> CombatCard -> CardEffectTags` | `Card.Effect.*` |
| `RuneInfo -> CombatCard -> BaseFlow` | 对应基础 Flow 模板副本 |
| `RuneInfo -> CombatCard -> LinkRecipes` | 仅月光卡填写 |
| `RuneInfo -> CombatCard -> DefaultLinkOrientation` | `Moonlight_Forward` 为 `Forward`，`Moonlight_Reversed` 为 `Reversed` |

## Generic Rune 基础效果接入

`Burn / Poison / Bleed / Rend / Wound / Knockback / Fear / Freeze / Stun / Curse` 会作为普通卡生成，`BaseFlow` 来自：

`/Game/Docs/BuffDocs/Playtest_GA/RuneBaseEffect/FA_Effect_*`

这些卡牌只负责“命中 AN 后打出卡牌并调用基础 FA”。流血、中毒、燃烧等状态本身的持续时间、伤害、护甲交互，应修改对应 Generic Rune 的 GA/GE，不要在每张 512 卡牌里重复写一套逻辑。

当前月光连携只覆盖已设计的有限配方，不会自动给所有新增状态卡生成月光 LinkFlow。

## 验收

1. 所有 DA 的 `bIsCombatCard` 已勾选。
2. 普通卡有 `BaseFlow` 和 `CardEffectTags`。
3. 月光两张测试卡都有 Forward/Reversed 两组 `LinkRecipes`。
4. 新增状态卡 `CardType=Normal`，`TriggerTiming=OnHit`。
5. 图标引用不为空。
6. 报告中没有 `Missing GameplayTag`、`Missing icon`、`Failed to duplicate`。
