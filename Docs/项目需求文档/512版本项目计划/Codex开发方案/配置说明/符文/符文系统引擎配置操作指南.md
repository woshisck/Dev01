# 符文系统引擎配置操作指南

## 当前配置目标

本阶段先配置可运行的普通战斗卡，并准备少量连携卡设计样例。不要大批量配置连携卡，因为 `LinkConfig`、反向赋能 Flow、断链事件还需要后续开发落地。

配置优先级：

1. 普通战斗卡。
2. 武器初始卡组。
3. 三选一奖励入卡组。
4. 少量连携卡设计记录。

## 普通战斗卡配置

打开目标 `RuneDataAsset`，配置：

`RuneInfo -> CombatCard`

| 字段 | 当前应该怎么填 |
| --- | --- |
| `bIsCombatCard` | 勾选 |
| `CardType` | `Attack` |
| `RequiredAction` | `Any` |
| `BaseFlow` | 填该卡正常释放的 `FlowAsset` |
| `MatchedFlow` | 留空 |
| `LinkMode` | `None` |
| `bRequiresComboFinisher` | 不勾选 |
| `DisplayName` | 填 HUD 显示名 |
| `HUDReasonText` | 可填简短效果说明 |

同时建议填写：

`RuneInfo -> RuneConfig -> RuneName / RuneIcon / RuneDescription`

## 普通卡配置规则

- 普通卡不再区分 Light / Heavy 是否匹配。
- 玩家任意攻击命中并出牌后，普通卡都应该执行 `BaseFlow`。
- 普通卡倍率来自后续全局连击倍率配置，不在每张普通卡里单独配置。
- 不要用 `MatchedFlow` 做普通卡奖励，否则后续迁移会增加返工。

## 武器初始卡组配置

打开目标 `WeaponDefinition`，找到：

`Combat Deck`

| 字段 | 当前应该怎么填 |
| --- | --- |
| `InitialCombatDeck` | 按顺序放入已开启 `bIsCombatCard` 的 `RuneDataAsset` |
| `ShuffleCooldownDuration` | 第一版建议 1.0 |
| `MaxActiveSequenceSize` | 第一版建议 0，表示展示完整卡组 |

规则：

- `InitialCombatDeck` 有配置时，优先使用它作为 1D 战斗卡组。
- `InitialCombatDeck` 为空时，会从旧 `InitialRunes` 中筛选开启了 `bIsCombatCard` 的符文。
- 建议现在直接配置 `InitialCombatDeck`，顺序更可控。

示例：

```text
InitialCombatDeck:
1. 火刃
2. 冰刺
3. 雷击
4. 破甲

ShuffleCooldownDuration = 1.0
MaxActiveSequenceSize = 0
```

## 三选一奖励配置

奖励仍然使用现有奖励池和 `RuneDataAsset`。

如果希望奖励进入战斗卡组：

1. 奖励符文必须勾选 `RuneInfo -> CombatCard -> bIsCombatCard`。
2. `CardType` 当前建议先配 `Attack`。
3. `BaseFlow` 必须能独立工作。
4. 选择奖励后，该符文会进入旧背包，并追加到战斗卡组末尾。

如果奖励符文没有勾选 `bIsCombatCard`：

- 它仍可以进入旧背包。
- 它不会进入战斗卡组。

## 连携卡当前怎么准备

当前不要大量配置连携卡资产。可以先准备 1-2 张设计样例，用于后续开发和验收。

打开目标 `RuneDataAsset`，可以先临时记录：

| 字段 | 当前建议 |
| --- | --- |
| `bIsCombatCard` | 勾选 |
| `CardType` | `Link` |
| `RequiredAction` | `Any` |
| `BaseFlow` | 该连携卡断链或普通释放时的 Flow |
| `MatchedFlow` | 暂时留空 |
| `LinkMode` | 暂时不要依赖 |
| `DisplayName` | 填卡名 |
| `HUDReasonText` | 写清楚连携意图 |

同时在备注、表格或命名文档中记录三项设计：

| 设计项 | 需要记录什么 |
| --- | --- |
| 正向连携 | 前一张满足什么条件时，本卡触发什么独立效果 |
| 反向连携 | 后一张满足什么条件时，后一张额外触发什么独立效果 |
| 断链释放 | 战斗系统通知退出连招状态时，本卡触发 BaseFlow、BreakFlow，还是失效 |

## 连携卡示例：月光

| 项 | 配置或设计 |
| --- | --- |
| `CardType` | Link |
| `BaseFlow` | 月光普通释放 Flow |
| 正向连携 | 前一张是 Attack 卡时，月光触发效果 A |
| 反向连携 | 后一张是 Attack 卡时，后一张额外触发效果 B |
| 断链释放 | 退出连招状态时，月光正常释放 BaseFlow |

注意：效果 B 后续会是单独配置的新 Flow，不复用后一张卡自己的 `BaseFlow`。

## 当前不要配置的内容

- 不要依赖 `MatchedFlow` 做普通卡动作匹配奖励。
- 不要配置三连携。
- 不要配置终结技卡。
- 不要尝试让所有卡牌都拥有连携效果。
- 不要把普通卡倍率写到每张卡里，倍率后续使用全局配置。
- 不要依赖卡组系统自己判断连招暂停；退出连招状态由战斗系统通知。

## 验收清单

### 普通卡验收

1. 武器 `InitialCombatDeck` 中放入 4 张普通卡。
2. 进入战斗，HUD 显示 4 张卡。
3. 任意 Light / Heavy 攻击命中后，当前卡被消耗。
4. 每张卡都执行自己的 `BaseFlow`。
5. 4 张卡消耗完后进入洗牌。
6. 洗牌结束后按固定顺序回填。

### 奖励入卡组验收

1. 奖励池中放入一张开启 `bIsCombatCard` 的普通卡。
2. 清场后选择该奖励。
3. 奖励进入旧背包。
4. 奖励追加到战斗卡组末尾。
5. 下一轮战斗能打出这张新增卡。

### 连携卡设计验收

1. 至少准备 1 张 Link 卡设计样例。
2. 明确它的正向连携条件和效果。
3. 明确它的反向连携条件和独立效果 B。
4. 明确退出连招状态时的断链释放策略。
5. 不要求当前版本在引擎内完全跑通连携逻辑。
