# 512 敌人与关卡符文配置说明

## 结论

敌人可以使用 `RuneDataAsset`，但不要把 512 关卡/敌人 Buff 继续配置成旧的 `Content/Docs/BuffDocs/Playtest_GA/.../DA_Rune_*`。

敌人实际使用的是：

```text
RuneDataAsset.RuneInfo.Flow.FlowAsset
RuneDataAsset.RuneInfo.RuneConfig.TriggerType
```

也就是说，只要符文 DA 有可用 Flow，敌人生成时就能被 `YogGameMode` 挂上；`Passive` 会立刻启动，`OnAttackHit` 会注册到敌人的命中事件后再触发。

## 新资产路径

本轮已经在 512 符文同级目录下创建了敌人/关卡专用资产：

```text
Content/Docs/BuffDocs/V2-RuneCard/512Generated/EnemyRoom/
```

这些资产复用 512 Flow，但关闭了 `CombatCard.bIsCombatCard`，避免把“玩家卡牌资产”和“敌人/关卡 Buff 资产”混用。

## 配置到关卡

在 `DA_Room.BuffPool` 中添加条目：

```text
RuneDA = /Game/Docs/BuffDocs/V2-RuneCard/512Generated/EnemyRoom/DA_Rune512_EnemyRoom_Attack
DifficultyScore = 2
ApplyChance = 1.0
```

然后在对应难度档位设置：

```text
BuffCount = 1
```

关卡开始时会从 `BuffPool` 抽取 `BuffCount` 个符文，挂到本关所有刷出的敌人身上。当前关卡 HUD 会显示实际抽中的符文。

## 配置到敌人

在 `EnemyData.EnemyBuffPool` 中添加条目，例如死亡守卫概率护盾：

```text
RuneDA = /Game/Docs/BuffDocs/V2-RuneCard/512Generated/EnemyRoom/DA_Rune512_EnemyRoom_Shield
DifficultyScore = 2
ApplyChance = 0.35
```

这个池子是敌人专属 Buff。只有该敌人被计划刷出时才会按概率判定并扣除难度分。

## 触发注意

- `Passive` 符文：敌人生成后立即启动 Flow。
- `OnAttackHit` 符文：依赖敌人攻击命中事件。敌人的攻击 Notify 必须正常广播命中事件，否则符文只会注册，不会触发。
- `Moonlight` / `Split` / `Pierce` 当前表现更重，优先放到精英房、测试房或少量强敌，不建议直接塞进所有普通房。
