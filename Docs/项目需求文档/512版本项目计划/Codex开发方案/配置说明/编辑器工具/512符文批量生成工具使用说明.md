# 512 符文批量生成工具使用说明

## 工具

Commandlet：`RuneCardBatchGenerator`

源码：

- `Source/DevKitEditor/Rune/RuneCardBatchGeneratorCommandlet.h`
- `Source/DevKitEditor/Rune/RuneCardBatchGeneratorCommandlet.cpp`

## 参数

| 参数 | 作用 |
| --- | --- |
| `-DryRun` | 默认模式，只生成报告，不创建或修改资产 |
| `-Apply` | 正式导入图标、复制模板资产、写入 DA 字段 |
| `-NoImportIcons` | 跳过 PNG 导入，只处理 DA/Flow |

## 命令

Dry run：

```powershell
& "Z:\GZA_Software\RealityCapture\UE_5.4\Engine\Binaries\Win64\UnrealEditor-Cmd.exe" "X:\Project\Dev01\DevKit.uproject" -run=RuneCardBatchGenerator -DryRun -unattended -nop4 -nosplash
```

正式生成：

```powershell
& "Z:\GZA_Software\RealityCapture\UE_5.4\Engine\Binaries\Win64\UnrealEditor-Cmd.exe" "X:\Project\Dev01\DevKit.uproject" -run=RuneCardBatchGenerator -Apply -unattended -nop4 -nosplash
```

## 输出

- `Saved/512RuneCardBatchReport.md`
- `/Game/Docs/BuffDocs/V2-RuneCard/512Generated`
- `/Game/Docs/BuffDocs/V2-RuneCard/512Generated/Flow`
- `/Game/Docs/BuffDocs/V2-RuneCard/Icons`

## 报告检查

重点搜索：

- `Missing GameplayTag`
- `Missing icon`
- `Missing template`
- `Failed to duplicate`
- `Manual check`

`Manual check` 不是失败，表示脚本已复制模板，但复杂 Flow/Niagara 仍需策划或技术美术在编辑器中确认节点连接与参数。

## 已知项目启动错误

当前项目在命令行启动编辑器时会先报既有缺失美术资源和动画 Skeleton 错误，例如 `/Game/Art/...`、`ABP_CommonLayer`。这些错误会导致 `UnrealEditor-Cmd` 返回非 0，但 `RuneCardBatchGenerator` 仍会执行并写出报告。判断本工具是否成功，以 `Saved/512RuneCardBatchReport.md` 和生成资产是否存在为准。

## 验收

1. Dry run 能列出 8 个图标、9 个 DA、基础 Flow 和月光 14 个 Link Flow。
2. Apply 后 Content Browser 中能看到生成资产。
3. 再次运行 Apply 不重复创建资产，而是复用并更新现有资产。
