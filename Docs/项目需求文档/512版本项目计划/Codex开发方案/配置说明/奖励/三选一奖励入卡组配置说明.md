# 三选一奖励入卡组配置说明

## 作用
玩家选择三选一奖励后，奖励会先进入旧背包持有物，再追加到战斗卡组末尾。选择奖励后不再自动打开旧背包。

## 配置位置
奖励本身仍使用现有奖励池和 `RuneDataAsset`。

想让奖励进入战斗卡组，需要在该奖励符文中配置：

`RuneInfo -> CombatCard -> bIsCombatCard = true`

如果没有勾选，奖励仍可作为旧背包符文存在，但不会作为战斗卡牌加入卡组。

## 规则
- 加入卡组顺序：追加到 `DeckList` 末尾。
- 选择奖励后：关闭三选一界面，恢复游戏输入。
- 背包预览按钮：继续保留，可以手动打开旧背包查看。
- 跳过奖励：保持原流程，不强制加入卡组。

## 策划检查点
| 情况 | 预期结果 |
| --- | --- |
| 奖励符文开启 Combat Card | 选择后加入旧背包，并追加进卡组 |
| 奖励符文未开启 Combat Card | 只进入旧背包，不加入卡组 |
| 点击背包预览 | 可以正常打开旧背包 |
| 点击跳过 | 不添加奖励，不破坏传送门流程 |

## 测试方式
1. 在奖励池中放入一张开启 Combat Card 的符文。
2. 进入奖励房并选择该符文。
3. 确认不会自动打开旧背包。
4. 确认 HUD 出现 `Added` 提示。
5. 下一轮战斗中打完原有卡组后，确认新奖励卡能出现在后续卡组顺序中。
