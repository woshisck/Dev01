# WBP_CombatDeckEditCardSlot配置说明

## 1. 作用
背包卡组编辑列表中的单张卡牌行。显示卡牌类型、名称、连携方向，并提供悬停/焦点选中与按下即拖拽排序。反转由 C++ 输入处理：选中卡牌后键盘按 `R`，手柄按 `X`。

## 2. 创建
| 项            | 配置                                       |
| ------------ | ---------------------------------------- |
| Widget 名称    | `WBP_CombatDeckEditCardSlot`             |
| Parent Class | `CombatDeckEditCardSlotWidget`           |
| 使用位置         | `WBP_CombatDeckEditWidget.CardSlotClass` |

## 3. 推荐层级
```text
Root: Border
  HorizontalBox
    CardIcon: Image
    TypeText: YogCommonUI Text 或 TextBlock
    CardNameText: YogCommonUI Text 或 TextBlock
    SelectedMark: Border/Image
    DirectionText: YogCommonUI Text 或 TextBlock
```

## 4. 控件参数
| 控件 | 关键属性 |
| --- | --- |
| `Root` | Height Override: 44-52；Padding: 6；Color: 深色 `(0.06,0.06,0.08,0.9)` |
| `CardIcon` | Size: 36x36；无图标时 C++ 自动隐藏 |
| `TypeText` | Font Size: 12；显示 `普通` / `连携` / `终结技` |
| `CardNameText` | Font Size: 15；Bold；读取卡牌 `DisplayName`，为空时读取 DA 符文名 |
| `SelectedMark` | 可选；Width: 4-6；选中时显示，未选中自动隐藏。即使不做这个控件，C++ 也会给选中行自动高亮 |
| `DirectionText` | Font Size: 20；连携卡显示 `↑` 或 `↓`，普通/终结技自动隐藏 |
| `ReverseButton` | 可删除；旧版兼容控件。如果保留，点击仍可反转；新版建议不做按钮，改用 `R` / 手柄 `X` |

## 5. 方向规则
| 显示 | 含义 |
| --- | --- |
| `↑` | 正向连携，读取上一张卡 |
| `↓` | 反向连携，等待下一张卡触发 |

拖动换位由 C++ 处理。只要该行 Widget 是 `CombatDeckEditCardSlotWidget` 子类，就可以把一行拖到另一行上调整顺序。

## 6. 交互规则
| 操作 | 效果 |
| --- | --- |
| 鼠标悬停卡牌行 | 自动选中该卡牌 |
| 手柄十字键上/下 | 选择上一张/下一张卡牌，普通卡、连携卡、终结技都会被选中 |
| 鼠标左键按下卡牌行 | 立即拿起该卡牌，进入拖拽排序 |
| 鼠标松开左键 | 把卡牌放到黄色提示条位置；原位松开不会改变排序 |
| 手柄按下 `A` | 立即拿起当前选中卡牌，进入手柄拖拽排序 |
| 手柄拖拽中按十字键上/下 | 移动黄色放置提示条 |
| 手柄松开 `A` | 把卡牌放到黄色提示条位置 |
| 鼠标拖动时离开卡牌行 | C++ 仍会读取鼠标纵向位置并更新放置提示条 |
| 拖到另一张卡牌上半区松开 | 插入到目标卡牌前面 |
| 拖到另一张卡牌下半区松开 | 插入到目标卡牌后面 |
| 键盘按 `R` | 选中连携卡切换 `↑` / `↓` |
| 手柄按 `X` | 选中连携卡切换 `↑` / `↓` |

不要在这个 WBP 上做上下移动按钮。卡牌位置只通过拖拽改变。
新版也不需要做反转按钮；`ReverseButton` 只用于兼容旧 WBP。
左键和手柄 `A` 不再作为选择按钮使用。选择由鼠标悬停、手柄十字键/摇杆移动和焦点移动完成；按下左键或 `A` 的含义是“拿起当前卡牌”。
拖起卡牌后，该卡牌会被 C++ 临时从列表移出，其他卡牌自动补位；放置位置由 `WBP_CombatDeckEditWidget` 自动生成黄色提示条，不需要在这个 WBP 里额外制作提示条控件。

## 7. 功能边界
这个 WBP 只负责创建 Designer 组件和基础样式，不需要在蓝图里写时间、拖拽、排序、反转、锁定判断。

| 功能 | 处理位置 |
| --- | --- |
| 左键/手柄 `A` 按下立即进入拖拽 | C++ 自动处理 |
| 拖拽浮空透明效果 | C++ 自动处理 |
| 拖起后列表自动补位 | C++ 自动处理 |
| 战斗中禁止操作的红色闪烁/位移反馈 | C++ 自动处理 |
| 插入到上半区/下半区 | C++ 根据鼠标位置判断 |
| 悬停、手柄焦点选中 | C++ 自动处理 |
| 选中高亮 | C++ 自动处理 |
| 手柄 `A` 按下拖拽、松开放下 | C++ 自动处理 |
| 鼠标拖拽离开卡牌行后的插入位置刷新 | C++ 自动处理 |
| 键盘 `R` / 手柄 `X` 反转 | C++ 自动处理 |

WBP 蓝图里不需要实现 `Delay`、`Timer`、`Tick`、拖拽事件或排序逻辑。只要控件名称正确，功能会自动生效。
