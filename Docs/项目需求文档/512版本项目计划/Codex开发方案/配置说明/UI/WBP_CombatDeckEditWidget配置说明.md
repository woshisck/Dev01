# WBP_CombatDeckEditWidget配置说明

## 1. 作用
用于背包中编辑当前武器的战斗卡组：纵向显示卡牌、选择卡牌、拖动换位、反转连携方向，并把选中卡牌信息显示到 `WBP_RuneInfoCard`。

## 2. 创建
| 项            | 配置                                                                  |
| ------------ | ------------------------------------------------------------------- |
| Widget 名称    | `WBP_CombatDeckEditWidget`                                          |
| Parent Class | `CombatDeckEditWidget`                                              |
| 挂载位置         | 放到 `WBP_BackpackScreen` 中                                           |
| 子控件命名        | 根背包中该控件命名为 `CombatDeckEditWidget`，C++ 会自动绑定玩家 `CombatDeckComponent` |

## 3. 推荐层级
```text
Root: Border 或 SizeBox
  HorizontalBox
    CardListPanel: ScrollBox
      CardListBox: VerticalBox
    RuneInfoCard: WBP_RuneInfoCard（已有信息卡，可选）
```

说明：这里不是新建一个叫 `RuneInfoCard` 的 WBP。这里是把已有的 `WBP_RuneInfoCard` 放进 `WBP_CombatDeckEditWidget`，并把这个子控件实例命名为 `RuneInfoCard`，C++ 才能自动把选中卡牌信息传给它。

## 4. 控件参数
| 控件              | 关键属性                                                                                    |
| --------------- | --------------------------------------------------------------------------------------- |
| `Root`          | Width Override: 520-640；Height Override: 720；Padding: 8                                 |
| `CardListPanel` | Width Override: 300-360；Scroll Bar Visibility: `Visible`。如果不想常驻显示滚动条，可填 `Hidden`，仍然可以滚动 |
| `CardListBox`   | 必须命名为 `CardListBox`，类型 `VerticalBox`                                                    |
| `RuneInfoCard`  | 可选；类型用已有 `WBP_RuneInfoCard`，子控件实例命名为 `RuneInfoCard`，用于显示选中卡牌 DA 信息                      |
| `CardSlotClass` | 填 `WBP_CombatDeckEditCardSlot`                                                          |
| `GamepadDragHoldSeconds` | 旧版兼容字段；新版手柄 `A` 按下立即拿起卡牌，不再读取长按时间 |
| `GamepadFloatingDragOpacity` | 鼠标/手柄拖拽浮空卡牌透明度；推荐 `0.9-1.0`，当前默认 `0.92` |
| `GamepadFloatingDragScale` | 鼠标/手柄拖拽浮空卡牌缩放；C++ 会限制到自然范围，当前默认 `1.06` |
| `GamepadFloatingDragOffset` | 鼠标/手柄拖拽浮空卡牌相对列表的位置偏移；C++ 会压缩到自然范围，当前默认 `X=18, Y=-24` |

## 5. 使用
把 `WBP_CombatDeckEditWidget` 放入现有背包界面后，不需要蓝图手动绑定。打开背包时会读取玩家当前武器的战斗卡组。

策划操作：
| 操作 | 效果 |
| --- | --- |
| 鼠标悬停/手柄焦点移动到卡牌 | 选中并在右侧显示卡牌信息 |
| 手柄十字键上/下 | 选择上一张/下一张卡牌，普通卡、连携卡、终结技都会被选中 |
| 手柄按下 `A` | 立即拿起当前选中卡牌，进入手柄拖拽排序，并显示浮空卡牌 |
| 手柄拖拽中按十字键上/下 | 移动黄色放置提示条 |
| 手柄松开 `A` | 把卡牌放到黄色提示条位置 |
| 鼠标左键按下卡牌 | 立即拿起该卡牌，使用和手柄一致的列表内部浮空卡牌 |
| 鼠标拖动卡牌 | C++ 持续读取鼠标纵向位置并移动黄色放置提示条 |
| 鼠标拖到列表上方/下方 | 插入位置会自动夹到最上/最下，即使鼠标不压在某张卡牌上也能移动 |
| 鼠标松开左键 | 把卡牌放到黄色提示条位置；原位松开不会改变排序 |
| 鼠标拖动卡牌到另一张卡牌上半区 | 插入到目标卡牌前面 |
| 鼠标拖动卡牌到另一张卡牌下半区 | 插入到目标卡牌后面 |
| 键盘按 `R` | 选中连携卡正向/反向切换 |
| 手柄按 `X` | 选中连携卡正向/反向切换 |
| 键盘按 `F` | 显示/隐藏右侧详情预览，默认显示 |
| 手柄按 `Y` | 显示/隐藏右侧详情预览，默认显示 |

卡牌排序只通过拖拽完成，不需要在卡牌行上配置上下移动按钮。列表按卡组数组顺序从上到下显示，先生成/先配置的卡牌排在上面。

左键和手柄 `A` 不再作为选择按钮使用。选择由鼠标悬停、手柄十字键/摇杆移动和焦点移动完成；按下左键或 `A` 的含义是“拿起当前卡牌”。

反转按钮可以取消。当前正式交互是先选中卡牌，再按 `R` 或手柄 `X` 反转。旧版 `ReverseButton` 如果还留在 `WBP_CombatDeckEditCardSlot` 中，C++ 仍会兼容点击反转。

拖起卡牌后，C++ 会把该卡牌临时从列表中移出，剩余卡牌自动补位；拖到两张卡牌之间时，会自动生成一条粗黄色放置提示条。松开后，提示条前面的卡牌位置不变，提示条下面的卡牌整体向下移动，为拖拽卡牌让出位置。

战斗中或只读预览状态下，C++ 会锁定编辑交互，并自动播放基础红色反馈。

非战斗阶段成功调整卡组顺序后，当前装备卡组会进入一次短装弹流程：装弹时间为武器 `ShuffleCooldownDuration` 的一半。装弹结束后，HUD 会按新的卡组顺序刷新可用卡牌。

## 6. 功能边界
`WBP_CombatDeckEditWidget` 只负责 Designer 层级和组件命名。以下功能都由 C++ 完成，不需要在蓝图里写逻辑：

| 功能 | 处理位置 |
| --- | --- |
| 打开背包自动绑定玩家战斗卡组 | C++ |
| 生成卡牌列表 | C++ |
| 悬停/手柄焦点自动选中 | C++ |
| 左键/手柄 `A` 按下立即拿起卡牌 | C++ |
| 鼠标/手柄拖拽浮空卡牌 | C++ 使用 `CardSlotClass` 在 `CardListBox` 内部自动生成，不需要 WBP 创建额外控件 |
| 鼠标拖拽时离开卡牌行仍更新插入位置 | C++ 每帧读取鼠标纵向位置 |
| 非战斗阶段调整顺序后短装弹并刷新装备卡组 | C++ |
| 手柄拖拽中十字键移动放置位置 | C++ |
| 拖拽时临时移除源卡并补位 | C++ |
| 黄色放置提示条 | C++ 自动生成，不需要 WBP 创建 |
| 反转连携方向 | C++ |
| 键盘 `R` / 手柄 `X` 输入 | C++ |
| 键盘 `F` / 手柄 `Y` 切换详情预览 | C++ |
| 选中卡牌高亮 | C++ |
| `WBP_BackpackScreen` 焦点转发十字键到卡组列表 | C++ |
| 战斗中或预览状态锁定交互 | C++ |
| 锁定时红色闪烁/位移反馈 | C++ |

WBP 不需要配置时间参数，也不要写 `Delay`、`Timer`、`Tick` 或拖拽排序蓝图。
