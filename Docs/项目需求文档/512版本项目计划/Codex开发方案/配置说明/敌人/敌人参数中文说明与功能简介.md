# 敌人参数中文说明与功能简介

## 已有功能

- 敌人可通过 `EnemyData` 配置行为树、感知、移动风格、攻击配置、难度分、霸体和专属 Buff。
- `AYogAIController` 已接管 Crowd 寻路避让、DA 行为树启动、警戒/进攻状态写入和战斗移动目标计算。
- 默认行为树资产可由 `EnemyAITemplateGenerator` 生成，不需要手动搭巡逻、警戒、进攻节点。
- 行为树可通过 `Update Enemy Awareness` 维护巡逻、警戒、进攻三态。
- 行为树可通过 `Update Enemy Combat Move` 获取移动点、距离和攻击距离判断。
- 行为树可通过 `Enemy Attack By Profile` 从 DA 读取攻击距离、权重、冷却和 GA Tag。
- 关卡刷怪已支持整关敌人上限、单波上限、高分敌人优先和 Buff 概率生效。

## 待做功能

- 运行默认 AI 生成器，确认 `BB_Enemy_DefaultMelee`、`BT_Enemy_DefaultMelee` 被创建。
- 在老鼠、死亡守卫 BP 和 DA 中按本文填写参数。
- 在攻击蒙太奇中配置流血、击退和霸体窗口。
- 如果后续需要固定巡逻路线，再新增 PatrolRoute 资产；当前 V1 是出生点周围随机巡逻。

## 1. 推荐配置顺序

```text
敌人 BP
  -> CharacterDataComponent 指向 EnemyData
  -> EnemyData 填 AI / 感知 / 移动 / 攻击 / 分数 / 霸体 / Buff
  -> 运行 EnemyAITemplateGenerator 生成默认 BT / BB
  -> DA_Room EnemyPool 和难度档位
  -> 攻击蒙太奇 Notify
```

## 2. 敌人 BP 参数

| 参数 | 中文说明 | 功能简介 |
|---|---|---|
| `AI Controller Class` | 敌人使用哪个 AI 控制器 | 用 `AYogAIController` 或其蓝图子类；不需要主动创建蓝图子类 |
| `Auto Possess AI` | 敌人何时自动生成 AIController | 建议 `Placed in World or Spawned`，保证预放和刷出的敌人都会启动 AI |
| `CharacterDataComponent` | 敌人读取哪个角色数据资产 | 指向对应 `EnemyData`，否则 AI 参数、行为树和霸体参数无法生效 |
| `Orient Rotation to Movement` | 移动时是否朝向移动方向 | 建议开启，让移动朝向更自然 |
| `Use Controller Desired Rotation` | 是否由 Controller 强制旋转 | 建议关闭，避免和 CharacterMovement / 行为树旋转冲突 |
| `Use RVO Avoidance` | 是否开启 CharacterMovement 自带 RVO | 建议关闭，避让交给 `AYogAIController` 的 Crowd 系统 |

## 3. EnemyData 基础参数

| 参数 | 中文说明 | 功能简介 |
|---|---|---|
| `EnemyClass` | 实际刷出的敌人 BP 类 | 刷怪系统用它生成敌人 |
| `DifficultyScore` | 敌人的基础难度分 | 分越高越贵，刷怪算法会优先刷高分敌人 |
| `BehaviorTree` | 敌人行为树 | 推荐使用 `/Game/Code/Enemy/AI/Behaviour/BT_Enemy_DefaultMelee` |
| `MovementTuning` | 移动和停距参数 | 决定敌人如何靠近、绕侧、停距和避让 |
| `AwarenessTuning` | 感知和巡逻参数 | 决定发现玩家、进攻、丢失目标、广播警戒和巡逻半径 |
| `AttackProfile` | 敌人攻击列表 | `Enemy Attack By Profile` 会按这里的距离、权重、冷却和 Tag 选招 |
| `EnemyBuffPool` | 敌人专属 Buff 池 | 死亡守卫概率护甲等敌人专属强化放这里 |
| `PreSpawnFX` | 预生成特效 | 刷怪前播放的 Niagara 特效 |
| `PreSpawnFXDuration` | 预生成等待时间 | 大于 0 时，特效播放后才真正生成敌人 |
| `SuperArmorThreshold` | 连续受击几次后触发霸体 | `0` 表示不会通过受击计数触发霸体 |
| `SuperArmorDuration` | 霸体持续时间 | 受击计数触发霸体后维持多少秒 |

## 4. MovementTuning 参数

| 参数 | 中文说明 | 功能简介 |
|---|---|---|
| `ApproachStyle` | 接近玩家的风格 | 决定敌人是直线接近、绕侧包围，还是压到中距离读招 |
| `PreferredRange` | 理想停距 | 敌人会尝试停在这个距离附近 |
| `AttackRange` | 攻击距离 | 小于等于这个距离时，黑板 `bInAttackRange` 会变为 true |
| `AcceptanceRadius` | MoveTo 接受半径 | 移动到目标点时允许的误差范围 |
| `RepathInterval` | 重新算移动目标的间隔 | 数值越小，敌人越频繁调整路线 |
| `FlankDistance` | 绕侧偏移距离 | `SwarmFlank` 下影响敌人往玩家侧面 / 背后绕多远 |
| `StrafeChance` | 侧移倾向 | 数值越高，越偏向侧向接近 |
| `CrowdSeparationWeight` | 群体分离权重 | 数值越高，敌人越不容易挤在同一点 |

`ApproachStyle` 可选：

| 值 | 中文说明 | 适用敌人 |
|---|---|---|
| `Direct` | 直接接近 | 简单敌人或临时测试 |
| `SwarmFlank` | 群体绕侧 | 老鼠这类数量多的敌人 |
| `BruiserHold` | 中距离压迫 | 死亡守卫这类大体型敌人 |

## 5. AwarenessTuning 参数

| 参数 | 默认值 | 中文说明 | 功能简介 |
|---|---:|---|---|
| `DetectionRadius` | 900 | 发现半径 | 敌人在该半径内且有视线时会发现玩家 |
| `CombatEnterRadius` | 650 | 进攻半径 | 发现玩家且距离小于该值时直接进入进攻状态 |
| `LoseTargetDelay` | 2.0 | 丢失目标延迟 | 进攻状态下看不见目标超过该时间后转警戒 |
| `AlertDuration` | 4.0 | 警戒持续时间 | 警戒状态搜索多久后回到巡逻 |
| `AlertBroadcastRadius` | 1200 | 警戒广播半径 | 发现玩家或被攻击后，通知该半径内附近敌人 |
| `PatrolRadius` | 600 | 巡逻半径 | 没有固定巡逻路线时，在出生点附近该半径内随机找点 |
| `PatrolWaitMin` | 0.6 | 最短巡逻停留 | 默认行为树当前使用 0.6 秒基础等待 |
| `PatrolWaitMax` | 1.5 | 最长巡逻停留 | 默认行为树当前使用 0.6 + 0.9 随机偏移 |

状态含义：

| 状态 | 中文说明 | 功能简介 |
|---|---|---|
| `Patrol` | 巡逻 | 默认非战斗逻辑，在出生点附近随机巡逻 |
| `Alert` | 警戒 | 搜索最后发现玩家的位置，也会响应附近敌人通知 |
| `Combat` | 进攻 | 追击、停距、选招；被攻击会直接进入该状态 |

## 6. AttackProfile 参数

| 参数 | 中文说明 | 功能简介 |
|---|---|---|
| `AttackProfile.Attacks` | 攻击项列表 | 每个条目代表一种可被 C++ 任务选择的攻击 |
| `AttackName` | 攻击备注名 | 方便策划识别，例如 `QuickBite`、`Heavy` |
| `AbilityTags` | 攻击 GA Tag 列表 | 必须和此敌人 `AbilityData` 中的攻击 Tag 对上；代码会过滤无效 Tag |
| `MinRange` | 最近可用距离 | 敌人与目标距离小于该值时不会选这招 |
| `MaxRange` | 最远可用距离 | 敌人与目标距离大于该值时不会选这招；`0` 表示不限制上限 |
| `Weight` | 随机权重 | 多个攻击同时可用时，权重越高越容易被选中 |
| `Cooldown` | 攻击项冷却 | 这条攻击成功激活后，多少秒内不会再次被选中 |
| `bPreAttackFlash` | 前摇红光 | 是否在 GA 持续期间播放攻击前摇提示 |

## 7. 默认行为树和黑板

| 参数 / 资产 | 中文说明 | 功能简介 |
|---|---|---|
| `BB_Enemy_DefaultMelee` | 默认小怪黑板 | 包含状态、目标、巡逻、战斗移动和计时 Key |
| `BT_Enemy_DefaultMelee` | 默认小怪行为树 | 巡逻、警戒、进攻三态通用树 |
| `Update Enemy Awareness` | 感知服务 | 发现玩家、切状态、处理丢失目标和警戒超时 |
| `Update Enemy Combat Move` | 战斗移动服务 | 写入 `MoveTargetLocation`、`DistanceToTarget`、`bInAttackRange` |
| `Update Enemy Patrol Target` | 巡逻点任务 | 在出生点附近随机找巡逻目标 |
| `Enemy Attack By Profile` | 按 DA 攻击配置选招 | 进入攻击前确保状态为 `Combat`，再按距离、权重、冷却选招 |

黑板 Key：

| Key | 类型 | 功能简介 |
|---|---|---|
| `EnemyAIState` | Enum | 当前状态：巡逻、警戒、进攻 |
| `TargetActor` | Object / Actor | 当前目标 |
| `LastKnownTargetLocation` | Vector | 最后一次看见或收到通知的位置 |
| `PatrolOriginLocation` | Vector | 巡逻中心点 |
| `PatrolTargetLocation` | Vector | 本次巡逻目标点 |
| `MoveTargetLocation` | Vector | 战斗移动目标点 |
| `DistanceToTarget` | Float | 到目标的 2D 距离 |
| `bInAttackRange` | Bool | 是否进入攻击距离 |
| `AcceptanceRadius` | Float | 推荐接受半径 |
| `AlertExpireTime` | Float | 警戒结束时间 |
| `LastSeenTargetTime` | Float | 最近看见目标时间 |

## 8. 刷怪参数

| 参数 | 中文说明 | 功能简介 |
|---|---|---|
| `DA_Campaign.FloorTable.TotalDifficultyScore` | 本关总难度分 | 决定本关整体刷怪预算 |
| `DA_Room.EnemyPool` | 房间敌人池 | 决定这个房间可能刷哪些敌人 |
| `EnemyPool.EnemyData` | 候选敌人 DA | 指向老鼠、死亡守卫等敌人数据 |
| `EnemyPool.MaxCountPerLevel` | 单类敌人整关上限 | 控制老鼠最多几只、死亡守卫最多几只 |
| `MaxWaveCount` | 最大波次数 | 程序会在 1 到该值之间随机实际波数 |
| `MaxEnemiesPerWave` | 单波同屏上限 | 控制同一波最多刷出多少敌人 |
| `MaxEnemiesPerLevel` | 整关敌人总上限 | 小房间建议 10，大房间建议 20 |
| `BuffCount` | 房间 Buff 数量 | 从房间 Buff 池抽几个 Buff 加到敌人身上 |

## 9. Buff 参数

| 参数 | 中文说明 | 功能简介 |
|---|---|---|
| `RuneDA` | 要施加的符文 / Buff | 敌人生成后会启动对应 BuffFlow |
| `DifficultyScore` | Buff 难度分 | Buff 生效后额外占用刷怪预算 |
| `ApplyChance` | 生效概率 | `1` 必定生效，`0.3` 表示 30% 概率生效 |

死亡守卫概率护甲建议放在 `DA_RottenGuard.EnemyBuffPool`：

| 字段 | 推荐 |
|---|---|
| `RuneDA` | `Content/Docs/BuffDocs/Playtest_GA/IronArmor/DA_Rune_IronArmor` |
| `DifficultyScore` | 2-3 |
| `ApplyChance` | 0.3-0.4 |

当前未看到专门命名为 `DA_EnemyBuff_IronArmor` 的资产；现有可直接使用的是 `DA_Rune_IronArmor`。

## 10. 动作通知参数

| 参数 / 通知 | 中文说明 | 功能简介 |
|---|---|---|
| `AN_MeleeDamage.ActDamage` | 命中伤害 | 控制本段攻击造成多少伤害 |
| `AN_MeleeDamage.ActResilience` | 动作韧性 | 攻击动作期间的抗打断能力 |
| `AN_MeleeDamage.OnHitEventTags` | 命中事件 Tag | 命中后广播给符文 / BuffFlow 使用 |
| `AN_MeleeDamage.AdditionalRuneEffects` | 命中附加效果 | 老鼠流血、死亡守卫击退等效果放这里 |
