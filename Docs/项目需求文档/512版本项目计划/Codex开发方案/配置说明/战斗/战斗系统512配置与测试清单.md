# 战斗系统 512 配置与测试清单

## 作用

用于汇总 512 战斗系统本轮需要策划配置和验收的内容。单项配置细节以本目录下对应说明文档为准。

## 需要配置的内容

| 类型 | 配置位置 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| 武器连招树 | `WeaponDefinition.WeaponComboConfig` | 近战必须 | 决定 L/H 输入怎么走到具体连招节点 |
| 连招节点 | `WeaponComboConfigDA.Nodes` | 必须 | 配 `NodeId`、父节点、输入、GA Tag、蒙太奇配置 |
| 节点终结击 | `WeaponComboConfigDA.Nodes.bIsComboFinisher` | 按需求 | 终结击不再只看第四段 |
| 闪避保存 | `WeaponComboConfigDA.Nodes.bAllowDashSave` | 按需求 | 决定该节点是否允许 DashSave 后继续 |
| 蒙太奇配置 | `MontageConfigDA` | 必须 | 配动画、总帧数、Entries |
| 连击窗口 | `MontageConfigDA.Entries.Combo Window` | 必须 | 决定下一段输入接收区间 |
| 命中窗口 | `MontageConfigDA.Entries.Hit Detection Window` | 必须 | 决定什么时候开判定 |
| 攻击参数 | `MontageAttackDataAsset` | 必须 | 配伤害、距离、判定框、HitStop |
| HitStop | `MontageAttackDataAsset.HitStop` | 按需求 | 配命中停顿和慢动作反馈 |
| 卡牌触发时机 | `CombatCard.TriggerTiming` | 按需求 | `OnCommit` 出招触发，`OnHit` 命中触发；不要用连招节点字段覆盖 |

## 最小配置流程

1. 建 `MontageAttackDataAsset`，先填伤害、攻击距离、判定框和 HitStop。
2. 建 `MontageConfigDA`，填 `Montage`、`TotalFrames`、`Combo Window`、`Hit Detection Window`，并引用 AttackData。
3. 建 `WeaponComboConfigDA`，配置 `L1/L2/L3`、`L2H3`、`H1/H2` 等节点。
4. 在 `WeaponDefinition` 上指定 `WeaponComboConfig`。
5. 如卡牌需要命中才消耗，在对应卡牌配置中把 `TriggerTiming` 改成 `OnHit`。

## 需要测试的内容

| 测试项 | 验收标准 |
| --- | --- |
| 连招路由 | `L-L-H` 能进入独立节点，不走 `L-L-L` 的第三击 |
| 根节点回退 | 当前节点找不到子节点时，下一次输入能回到对应根节点 |
| 同蒙太奇不同数据 | 两个节点复用同一个蒙太奇时，可得到不同伤害/距离/HitStop |
| 终结击 | `bIsComboFinisher=true` 的节点被识别为终结击 |
| DashSave | 允许保存的节点闪避后，下一击能从保存节点继续 |
| OnCommit | 出招时消耗卡牌，空挥也消耗 |
| OnHit | 空挥不消耗，命中后消耗 |
| 防重复消费 | 同一次攻击不能 OnCommit 和 OnHit 各消耗一次 |
| 旧武器兼容 | 未配置 `WeaponComboConfig` 的武器仍使用旧蒙太奇映射 |
| HitStop | 只在命中时触发，轻/重/终结击手感有差异 |

## 自动化测试建议

- `L-L-H` 节点路由和终结击判断。
- `OnCommit / OnHit` 消耗时机。
- 同一 `AttackInstanceGuid` 防重复消耗。
- DashSave 后恢复一次连招节点。
- 缺失父节点、重复 `NodeId`、缺失 `MontageConfig` 的配置校验 Warning。

## 注意事项

- 本清单不覆盖具体符文 Flow、Buff、Link 效果配置。
- 战斗侧只负责输出事实：输入类型、连招段数、节点 ID、是否终结击、是否 DashSave、触发时机。
- 策划验收时至少需要一把测试剑覆盖 `L-L-L`、`L-L-H`、`H-H` 三条路径。
