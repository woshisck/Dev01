# 武器连招与攻击参数配置说明

## 作用

用于配置玩家近战武器的轻/重攻击连招、每个连招节点使用的蒙太奇、攻击参数，以及战斗侧给卡牌/符文系统提供的触发时机和连招上下文。

本功能只负责战斗逻辑配置，不负责具体符文效果配置。

## 配置位置

| 配置对象 | 配置位置 |
| --- | --- |
| 武器连招树 | `WeaponDefinition` 的 `GameplayAbilityComboGraph`，旧资源可继续用 `WeaponComboConfig` |
| 连招节点 | `WeaponComboConfigDA` |
| 蒙太奇与命中窗口 | `MontageConfigDA` |
| 连招输入窗口 | 实际 `AnimMontage` 上的 `ANS_AddGameplayTag`，Tags 填 `PlayerState.AbilityCast.CanCombo` |
| 攻击伤害/距离/判定框/HitStop | `MontageAttackDataAsset` |
| 卡牌触发时机 | `RuneDataAsset` 的 `CombatCard.TriggerTiming` |

## 字段说明

### WeaponDefinition

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `GameplayAbilityComboGraph` | 当前推荐使用的连招图配置 | 近战武器优先配置，例如 `CG_THSword_Test` |
| `WeaponComboConfig` | 旧版数组连招配置 DA | 兼容字段；使用 `GameplayAbilityComboGraph` 时可留空 |

如果两个字段都不配置，系统会回退旧的 `AbilityData.MontageMap` 逻辑。使用战斗卡牌连携时建议配置 `GameplayAbilityComboGraph`，这样卡牌能读取 `ComboIndex`、`ComboNodeId`、`AbilityTag`、是否连续连招等上下文。

### WeaponComboConfigDA

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `RootNodes` | 根节点 ID 数组 | 配第一段轻击和第一段重击 |
| `Nodes` | 连招节点数组，用数组表达连招树 | 每个攻击动作一个节点 |
| `NodeId` | 节点唯一 ID | 使用 `L1`、`L2`、`L2H3` 这类可读命名 |
| `ParentNodeId` | 父节点 ID，空表示根节点 | 根节点留空，后续节点填上一个节点 |
| `InputAction` | 进入该节点需要的输入，`Light` 或 `Heavy` | 按实际按键配置 |
| `AbilityTag` | 激活的 GA Tag | 复用现有轻/重攻击 GA Tag |
| `MontageConfig` | 该节点使用的蒙太奇配置 | 必填 |
| `AttackDataOverride` | 当前节点覆盖使用的攻击参数 | 同一蒙太奇在不同分支有不同伤害/范围时填写 |
| `bIsComboFinisher` | 是否是终结击 | 只在连招最后一击勾选 |
| `bAllowDashSave` | 是否允许闪避保存当前连招节点 | 默认按旧规则开启需要保留的节点 |
| `CardTriggerTiming` | 节点侧兼容字段 | 当前不建议作为主配置，实际消耗时机以卡牌 `CombatCard.TriggerTiming` 为准 |

### GameplayAbilityComboGraph 节点窗口

当前阶段连招开窗临时以蒙太奇 Notify 为准，Graph 节点里的窗口帧只保留为配置展示和后续编辑器工具使用。

| 字段 | 当前用法 | 推荐 |
| --- | --- | --- |
| `Use Node Combo Window` | 当前不作为运行时开窗依据 | 暂时关闭 |
| `ComboWindowStartFrame` / `ComboWindowEndFrame` / `ComboWindowTotalFrames` | 仅保留数据，不驱动运行时 | 后续做 Graph 编辑器开窗工具时再启用 |

### MontageConfigDA

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `Montage` | 实际播放的动画蒙太奇 | 必填 |
| `ConfigTags` | 蒙太奇配置标签，用于工具或分支选择 | 只填需要被规则读取的标签 |
| `TotalFrames` | 蒙太奇总帧数 | 按动画帧数填写 |
| `Entries` | 命中窗口、Combo Window、事件等配置 | 命中窗口至少配置一次 |

`Entries` 常用类型：

| 类型 | 说明 | 推荐 |
| --- | --- | --- |
| `Combo Window` | 连击输入窗口配置数据 | 当前暂不作为运行时开窗依据；实际开窗用蒙太奇 `ANS_AddGameplayTag` |
| `Hit Detection Window` | 命中判定窗口 | 每段攻击至少配置一次 |
| `Gameplay Event at Frame` | 单帧 GameplayEvent | 只在需要额外事件时配置 |

`Hit Detection Window` 可在 `AttackDataCandidates` 中配置候选攻击参数。需要按标签区分时，使用 `RequiredTags`、`BlockedTags` 和 `Priority` 选择最终 `AttackData`。

### MontageAttackDataAsset

| 字段 | 说明 | 推荐 |
| --- | --- | --- |
| `EventTag` | 命中事件标签 | 按当前 GA/事件约定填写 |
| `ActDamage` | 攻击伤害 | 按武器和连招段数配置 |
| `ActRange` | 攻击距离 | 与实际动画位移匹配 |
| `ActResilience` | 韧性相关数值 | 按敌人受击规则配置 |
| `ActDmgReduce` | 减伤相关数值 | 默认 `0` |
| `HitboxTypes` | 判定框数组 | 普通近战至少配置一个 |
| `HitStopMode` | 命中停顿模式 | 轻攻击 `Freeze`，重击可用 `Freeze` 或 `Slow` |
| `OnHitEventTags` | 命中后抛出的事件标签 | 只填需要系统监听的标签 |
| `AdditionalRuneEffects` | 额外符文效果入口 | 当前战斗侧不配置具体效果 |

攻击参数优先级：

1. 连招节点 `AttackDataOverride`
2. `Hit Detection Window.AttackDataCandidates` 选出的 `AttackData`
3. 旧 `AN_MeleeDamage` 自身字段

## 推荐配置

- 普通近战武器优先配置 `GameplayAbilityComboGraph`。
- `L-L-H`、`H-H-H` 即使播放同一个蒙太奇，也要拆成不同 `NodeId`，这样可以配置不同伤害、范围和 HitStop。
- 需要继续接下一段的蒙太奇，在实际 `AnimMontage` 上添加 `ANS_AddGameplayTag`，Tags 填 `PlayerState.AbilityCast.CanCombo`；终结击通常不需要配置。
- 512 战斗卡牌建议使用 `CombatCard.TriggerTiming = OnHit`，表示 `AN_MeleeDamage` 生效后才消耗和触发。
- 终结击不要再依赖固定第四段，使用节点上的 `bIsComboFinisher`。
- 512 版本暂时不要求把所有 AN/ANS 迁移进 DA，旧蒙太奇可以继续使用。

## 配置示例

一把测试剑配置三条连招：

| 输入 | NodeId | ParentNodeId | InputAction | MontageConfig | bIsComboFinisher |
| --- | --- | --- | --- | --- | --- |
| L | `L1` | 空 | Light | `DA_MC_Sword_L1` | false |
| L-L | `L2` | `L1` | Light | `DA_MC_Sword_L2` | false |
| L-L-L | `L3` | `L2` | Light | `DA_MC_Sword_L3` | true |
| L-L-H | `L2H3` | `L2` | Heavy | `DA_MC_Sword_L2H3` | true |
| H | `H1` | 空 | Heavy | `DA_MC_Sword_H1` | false |
| H-H | `H2` | `H1` | Heavy | `DA_MC_Sword_H2` | true |

## 验收方式

1. 按 `L-L-H`，第三击进入 `L2H3`，并识别为终结击。
2. `L-L-H` 和 `H-H-H` 复用同一蒙太奇时，伤害和攻击距离可以不同。
3. 配置 `OnCommit` 的卡牌空挥也会消耗。
4. 配置 `OnHit` 的卡牌空挥不消耗，命中后才消耗。
5. 闪避保存后继续攻击，能从保存的连招节点继续。
6. 未配置 `GameplayAbilityComboGraph` / `WeaponComboConfig` 的武器仍能使用旧蒙太奇映射，但卡牌连携上下文会不完整。

## 注意事项

- 当前生产武器优先使用 `GameplayAbilityComboGraph`。旧 `WeaponComboConfigDA` 仍保留兼容。
- `NodeId` 必须唯一，`ParentNodeId` 必须能找到对应节点。
- 节点缺少 `MontageConfig` 会导致该节点无法正确播放攻击。
- 如果同一个蒙太奇同时被“可继续连招节点”和“终结节点”复用，蒙太奇上的 `CanCombo` Notify 会同时影响这些分支；需要分支差异时应拆不同蒙太奇，或后续重新启用 Graph 节点窗口。
- `RequiredAction` 是旧卡牌字段，普通攻击卡不再依赖 Light/Heavy 匹配才触发。
- 符文 Flow、具体 Buff、Link 效果由符文系统配置文档说明；战斗侧只提供 `ComboIndex`、`ComboNodeId`、`AbilityTag`、`TriggerTiming` 等上下文。
