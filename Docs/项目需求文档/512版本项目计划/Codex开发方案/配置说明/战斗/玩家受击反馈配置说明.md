# 玩家受击反馈配置说明

## 作用

玩家受到有效伤害后触发本地反馈：屏幕边缘红光、玩家身体白光、手柄震动、短时间全局慢动作。逻辑接在玩家 ASC 的 `ReceivedDamage` 上，不影响伤害计算、符文效果或敌人攻击逻辑。

## 配置位置

打开玩家角色蓝图或其 Class Defaults，查找 `Damage Feedback` 分类。

## 字段

- `bEnableDamageReceivedFeedback`：总开关。
- `DamageScreenFlashColor`：屏幕边缘闪光颜色，默认红色。
- `DamageScreenFlashAlpha`：边缘红光强度，默认 `0.28`，建议 `0.2-0.35`。
- `DamageScreenFlashDuration`：边缘红光淡出时间，默认 `0.12` 秒。
- `DamageScreenEdgeThickness`：边缘红光厚度，默认 `120`，数值越大越靠近屏幕中心。
- `DamagePlayerOverlayMaterial`：玩家受击白光 Overlay 材质。为空时复用热度升阶 Overlay 材质。
- `DamagePlayerGlowColor`：身体白光颜色，默认高亮白。
- `DamagePlayerGlowDuration`：身体白光持续时间，建议 `0.12-0.2` 秒。
- `DamagePlayerGlowIridIntensity`：白光材质的彩虹/Fresnel 强度，默认 `0`。
- `DamageReceivedForceFeedback`：受击手柄震动资产。
- `DamageDynamicForceFeedbackIntensity`：未配置震动资产时的动态震动强度，默认 `0.7`。
- `DamageDynamicForceFeedbackDuration`：未配置震动资产时的动态震动时长，默认 `0.12` 秒。
- `bEnableDamageTimeDilation`：是否启用受击慢动作。
- `DamageTimeDilationScale`：全局时间倍率，默认 `0.08`。
- `DamageTimeDilationDuration`：慢动作真实持续时间，默认 `0.15` 秒。

## 材质要求

`DamagePlayerOverlayMaterial` 需要支持以下参数：

- `EmissiveColor`
- `GlowAlpha`
- `SweepProgress`
- `IridIntensity`

当前实现会把 `SweepProgress` 固定为 `1`，表现为瞬时全身白光淡出。

## 测试

1. 可选：给玩家角色配置 `DamageReceivedForceFeedback`。不配置时会使用动态震动兜底。
2. 确认 `DamagePlayerOverlayMaterial` 已配置，或确认热度升阶 Overlay 材质可复用。
3. PIE 中让玩家被敌人实际造成伤害。
4. 验收：屏幕只有边缘红光，不再整屏红闪；角色身体白光、手柄震动、约 `0.15` 秒慢动作均出现。
5. 连续受击时，慢动作计时会刷新，结束后恢复到受击前的全局时间倍率。
