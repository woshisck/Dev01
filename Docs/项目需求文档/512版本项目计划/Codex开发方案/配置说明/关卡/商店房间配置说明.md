# 商店房间配置说明

## 已生成的默认资产

| 类型 | 路径 |
| --- | --- |
| 商店房间 DA | `/Game/Docs/Map/Shop/DA_Room_512_Shop` |
| 默认商店库存 | `/Game/Docs/Map/Shop/DA_ShopInventory_512_Default` |
| 商店选择 WBP | `/Game/UI/Playtest_UI/Event/WBP_ShopSelection` |
| 默认占位地图 | `ShopRoom`，当前由代码解析到 `/Game/Art/Map/Map_Data/L1_InitialRoom/InitialRoom` |

## 运行流程

`DA_Room_512_Shop` 带有 `Room.Type.Shop` 和 `Room.Layer.L1` 标签。

进入该房间后：

1. `YogGameMode` 识别为商店房间。
2. 跳过刷怪、跳过关卡敌人符文 HUD。
3. 自动配置场景中预放的 `AShopActor`；如果场景中没有商店 Actor，则在玩家前方生成默认商店 Actor。
4. 玩家进入商店交互范围后按交互键打开 `WBP_ShopSelection`。
5. 购买时消耗金币，获得对应符文，并把战斗卡符文加入 CombatDeck。
6. 房间传送门直接开启，玩家可购买后离开。

## 你在有美术效果的房间里需要做什么

1. 准备或选择一张商店房间 `.umap`，放好 `PlayerStart`。
2. 放置传送门 Actor，并确保 `Portal.Index` 与 `DA_Room_512_Shop.PortalDestinations` 中的 index 对得上。
3. 如果已有商人、柜台、光效等美术，建议做一个 `AShopActor` 的蓝图子类，把美术组件挂到这个蓝图上。
4. 在 `DA_Room_512_Shop.ShopActorClass` 中填入这个蓝图子类；若不填，程序会生成默认方块占位商店。
5. 如果你已经在地图里手动摆好了商店 Actor，程序会优先配置场景中已有的商店 Actor，不会再生成新的默认商店。
6. 在 `DA_Room_512_Shop.RoomName` 中把 `ShopRoom` 改成实际商店地图的长包路径，或在 `YogGameMode::ResolveRoomLevelNameForOpen` 中新增别名解析。
7. 在 `DA_ShopInventory_512_Default.StockPool` 中调整商品符文和价格；商品默认购买后售罄。

## 默认商品

默认库存暂时使用 512 已有符文：

- `DA_Rune512_Attack`
- `DA_Rune512_Burn`
- `DA_Rune512_Shield`
- `DA_Rune512_Poison`

价格优先读 `FShopRuneEntry.OverrideGoldCost`；当该值为 `-1` 时才读取符文本身的 `RuneConfig.GoldCost`。

## 注意

商店房间当前定义为非战斗房。后续如果需要“黑市遭遇”“击败敌人后开店”等变体，建议另开 `Room.Type.Event` 或商店子类型，不要把当前基础商店房改成战斗房。
