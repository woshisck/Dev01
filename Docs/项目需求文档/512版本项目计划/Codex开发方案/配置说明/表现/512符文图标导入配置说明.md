# 512 符文图标导入配置说明

## 作用

记录 512 符文卡牌图标的源文件、导入位置和 DA 引用方式。策划正常只需要确认 DA 的 `Rune Icon` 是否正确，不需要手动导入。

## 源文件

源 PNG 位于：

```text
X:\Project\Dev01\SourceArt\512RuneIcons
```

| 文件 | 用途 |
| --- | --- |
| `T_Rune512_Attack.png` | 攻击 |
| `T_Rune512_Burn.png` | 燃烧 |
| `T_Rune512_Poison.png` | 中毒 |
| `T_Rune512_Moonlight.png` | 月光 |
| `T_Rune512_Splash.png` | 溅射，近战卡牌 |
| `T_Rune512_Split.png` | 分裂，远程卡牌 |
| `T_Rune512_Shield.png` | 护盾 |
| `T_Rune512_Pierce.png` | 穿透 |
| `T_Rune512_ReduceDamage.png` | 减伤 |

## 风格规范

图标统一参考关卡/敌人符文信息的显示需求：

| 项目 | 规范 |
| --- | --- |
| 画布 | `256x256` PNG |
| 背景 | 暗色方形底，保留外框 |
| 边框 | 黑色厚边 + 暗金内边 |
| 主体 | 中央高对比符号，避免过细线条 |
| 颜色 | 按效果区分主色，整体不要离开暗底金边体系 |
| 禁忌 | 普通“分裂”图标不要出现月亮/月刃元素 |

## UE 导入位置

```text
/Game/Docs/BuffDocs/V2-RuneCard/Icons
```

导入后资产名保持 `T_Rune512_*`。

## DA 引用位置

打开生成的 `DA_Rune512_*`：

```text
Rune Config -> Rune Icon
```

批量工具 `RuneCardBatchGenerator -Apply` 会自动导入/覆盖图标，并把图标引用写回 DA。重新修改源 PNG 后，重新运行 `-Apply` 即可同步到引擎资产。

## 验收

| 检查项 | 结果 |
| --- | --- |
| `Icons` 文件夹下存在对应 `T_Rune512_*.uasset` | 必须 |
| 每张 `DA_Rune512_*` 的 `Rune Icon` 不为空 | 必须 |
| 分裂图标无月亮/月刃元素 | 必须 |
| 背包/卡组 UI 缩略图不裁切关键符号 | 必须 |
