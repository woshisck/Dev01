# 当前上下文

## 来源

该计划来自 429 反馈迭代后的方向确认。

核心反馈包括：

- 热度系统能理解意图，但存在感不强，且有强迫攻击感。
- 旧 2D 背包太复杂，玩家很少停下来整理。
- 玩家更希望战斗是快速、爽快、连续验证构筑的体验。
- 关卡结算和奖励拾取流程偏生硬。

## 已确认方向

游戏战斗节奏采用“快速动作战斗”为主，参考黑帝斯的爽快战斗节奏，同时保留少量魂系的出招承诺和读招压力。

系统方向为：

- 攻击卡组成为前台主系统。
- 旧 2D 背包退为后台存储。
- 轻重攻击顺序决定卡牌匹配收益。
- 卡牌打空后进入洗牌或装弹空窗。
- 洗牌期间玩家仍可攻击，但不触发卡牌。
- 热度用于强化连携、终结技、表现和倍率，不再控制符文激活区。
- 关卡奖励选择后自动加入后台背包和卡组，不自动打开旧背包。

## 最小战斗循环

1. 玩家进入房间，看到当前 1D 卡组。
2. 玩家用 Light / Heavy 攻击敌人。
3. 每次攻击尝试消耗当前卡牌。
4. 动作匹配时触发基础效果和奖励效果。
5. 动作不匹配时卡牌仍消耗，只触发基础效果。
6. 连招最后一击可能触发 Finisher 卡。
7. 卡组打空后进入洗牌，UI 暂时为空。
8. 洗牌期间玩家继续攻击，但不触发卡牌。
9. 冷却结束后卡组按固定顺序回填。
10. 清场后选择奖励，奖励进入后台背包与卡组。
11. 下一房间立即验证新卡组。

## 当前协作拆分

Codex 负责：

- CombatDeckComponent。
- 卡牌数据结构。
- 洗牌装弹逻辑。
- 后台背包兼容。
- 卡组 UI 事件。
- 奖励接入卡组。

战斗同事负责：

- Light / Heavy 攻击。
- 连招终结击。
- DashSave。
- 攻击事件派发。

共同联调：

- 战斗侧调用卡组接口。
- 4 张卡打空后洗牌。
- 洗牌期间仍可攻击。
- 奖励选择后加入卡组。
